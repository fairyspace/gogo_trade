💧 流动性池管理 运营
Liquidity Pool — "多家 LP 报价聚合，取最优 Bid/Ask 或合并深度"
💧
剧情：大宗品种需要从 3 家 LP 同时取报价，选最优 Bid/Ask 对冲。运营创建一个 BEST_PRICE 模式的流动性池， 加入 3 个 LP 网关成员，设置各自权重和单笔上限。路由规则可直接将订单指向此池， 池聚合引擎实时过滤过期/偏离报价，保证最优价格。
POST /api/v1/liquidity-pools 创建流动性池

字段类型必填说明
pool_codestring必填池代码，唯一
pool_namestring必填池名称
aggregation_modestring必填BEST_PRICE / VWAP / DEPTH_MERGE
distribution_modestring可选STANDARD / HEDGING / MIXED，默认 STANDARD
min_active_sourcesinteger可选最低活跃源数，低于则池降级，默认 1
stale_quote_threshold_msinteger可选报价过期阈值毫秒，超过不参与聚合，默认 500
price_deviation_threshold_bpsinteger可选价格偏离阈值（基点），偏离中位数超过则过滤，null=不过滤
applicable_categoriesarray可选适用品种分类，如 ["FOREX","METALS"]，null=所有
aggregation_mode 含义：
BEST_PRICE 最优 Bid/Ask VWAP 量加权均价 DEPTH_MERGE 合并订单簿深度

POST /api/v1/route-rules 创建路由规则
📤 Body
字段类型必填说明
tenant_idinteger可选租户 ID，0=全局，默认 0
rule_codestring必填规则代码，同租户唯一，如 FOREX_VIP_ABOOK
rule_namestring必填规则名称
match_conditionsobject可选匹配条件树 JSON，null=匹配所有（仅默认规则可用）
execution_modestring必填A（A-Book STP）/ B（B-Book 内部）/ HYBRID（混合）
allocation_modestring必填PRIORITY（故障转移）/ RATIO（按比例）/ SPLIT（拆单）
execution_profile_idinteger可选执行参数 Profile ID，null=用默认
markup_profile_idinteger可选加价 Profile ID，null=不加价
sor_strategystring可选BEST_PRICE / BEST_EXECUTION / MINIMIZE_FEES / FUNDING_AWARE
sor_configobject可选SOR 配置 JSON，按 sor_strategy 区分
credit_check_modestring可选NONE / PRE_CHECK / POST_CHECK / REAL_TIME，默认 NONE
priorityinteger必填优先级，值越大越先匹配
is_defaultinteger可选0=普通规则 1=默认兜底，每租户最多一条
effective_startdatetime可选生效开始，null=立即
effective_enddatetime可选生效结束，null=永久
// match_conditions 条件树示例：外汇品种 + VIP账户组 + 订单量≤50手 { "logic": "AND", "conditions": [ { "dimension": "SYMBOL", "field": "category", "operator": "EQ", "value": "FOREX" }, { "dimension": "ACCOUNT", "field": "group_id", "operator": "IN", "value": [10, 20] }, { "dimension": "ORDER", "field": "volume", "operator": "LTE", "value": 50 } ] } // 可用维度：ORDER / SYMBOL / ACCOUNT / CLIENT / POSITION / REQUEST // 可用运算符：EQ NEQ GT GTE LT LTE IN NOT_IN BETWEEN CONTAINS REGEX // 支持 OR/NOT 嵌套，叶子节点可加 "negate":true 取反
⚠️ is_default=1 时每租户最多一条（数据库唯一索引保证）。match_conditions=null 只能用于默认兜底规则。

last_reset_atdatetime最近一次重置时间
data_sourcestringREDIS（实时）/ DB（降级）
⚠️ 数据源为 Redis 时近实时，为 DB 时为上次异步同步值（可能有几秒延迟）。风控决策请以 Redis 数据为准。
POST /api/v1/credit-limits/{limitId}/reset 手动重置信用占用
🚨 高危操作，需管理员权限（RBAC: credit.reset）。将 used_amount 和 reserved_amount 原子清零，同时更新 last_reset_at 并写 t_config_change_log 审计记录。操作不可撤销。

🚀 配置发布 & 版本管理 管理员
Config Publish — "草稿变更经发布原子刷新各节点内存快照"
🚀
剧情：一批配置改动（新增 LP、调整规则优先级、更新 Profile）在草稿态积累后， 管理员发起发布，系统打全量快照（SHA-256 校验）并写入新版本号； 所有路由节点收到版本变更通知后拉快照、校验后原子切换指针，切换期间旧版本继续服务， 零停机。发现问题可随时回滚到历史版本。
