# 交易系统路由网关配置 — MySQL 数据库设计文档

> **版本**: v2.2（精简版）  
> **日期**: 2026-06-09

---

## 一、设计总览

### 1.1 设计目标

设计一套可生产落地的路由网关配置体系，覆盖网关接入、品种映射、流动性聚合、路由决策与审计追踪。

**核心架构**：配置表全量加载入内存，运行时内存匹配，数据库不参与实时查询。

> **两类表的区分**：
> - **配置表**：低频变更、全量入内存（如网关、规则、Profile）。
> - **运行时/流水表**：高频读写、不入内存（如审计日志、指标、信用占用）。
> 两者在约束、索引、并发上的设计取舍不同，下文分别说明。

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| 配置与运行时分离 | 配置入内存匹配；运行时状态（连接状态、信用占用）独立存储，不混入配置表 |
| JSON 扩展字段 | 核心字段独立列存储，差异化/低频字段用 JSON 承载，避免字段膨胀 |
| 优先级匹配 | 规则按优先级排序，第一条命中生效，支持默认兜底 |
| 软删除 + 时间有效性 | `deleted_at` 软删除 + `effective_start/end` 时间窗口 |
| Profile 化复用 | 执行参数、加价参数抽为 Profile，多规则引用，减少冗余 |
| 分组与聚合 | 网关按组管理（故障转移/负载均衡），多 LP 按池聚合（最优价） |
| 信用约束感知 | 路由前检查信用额度，避免超限拒单 |
| 配置发布版本化 | 变更经"发布"递增全局版本号，节点据此原子刷新内存快照，保证多节点一致 |
| 完整性约束兜底 | 关键不变量由应用层校验 + 数据库 CHECK/唯一键双重保证 |
| 多租户隔离 | 多法律实体/品牌数据按 `tenant_id` 隔离 |

### 1.3 表清单

| # | 表名 | 用途 | 类型 |
|---|------|------|------|
| 1 | `t_gateway` | 网关定义 | 配置 |
| 2 | `t_gateway_group` | 网关组（故障转移/负载均衡） | 配置 |
| 3 | `t_gateway_group_member` | 网关组成员 | 配置 |
| 4 | `t_gateway_symbol_mapping` | 网关品种映射 | 配置 |
| 5 | `t_liquidity_pool` | 流动性池（多 LP 报价聚合） | 配置 |
| 6 | `t_liquidity_pool_member` | 流动性池成员 | 配置 |
| 7 | `t_execution_profile` | 执行参数 Profile | 配置 |
| 8 | `t_markup_profile` | 加价/费用 Profile | 配置 |
| 9 | `t_route_rule` | 路由规则 | 配置 |
| 10 | `t_route_rule_allocation` | 路由分配 | 配置 |
| 11 | `t_gateway_credit_limit` | 网关信用限额（静态） | 配置 |
| 12 | `t_gateway_credit_usage` | 信用额度实时占用 | 运行时 |
| 13 | `t_route_audit_log` | 路由审计日志 | 流水 |
| 14 | `t_gateway_metric` | 网关运行指标 | 流水 |
| 15 | `t_config_version` | 配置发布版本 | 配置 |
| 16 | `t_config_change_log` | 配置变更审计 | 流水 |

> **网关组 vs 流动性池**：组解决"用哪个网关"（选择/切换），池解决"多家报价怎么合成最优价再成交"（聚合）。简单场景仅用组即可，多 LP 做市才需要池。
>
> **交易时段不在本模块**：当前是否可交易、属于哪个时段（含节假日/DST/熔断）由独立的【时间配置模块】维护并输出，路由层只消费其结论，不重复建模。

### 1.4 表间关系

```
t_gateway_group (1) ──< (N) t_gateway_group_member (N) >── (1) t_gateway
t_liquidity_pool (1) ──< (N) t_liquidity_pool_member (N) >── (1) t_gateway
                                                                  |
                                                                  ↓ 映射
                                                       t_gateway_symbol_mapping

t_route_rule (1) ──< (N) t_route_rule_allocation ──> t_gateway / t_liquidity_pool / 内部
     |  引用
     ├──> t_execution_profile
     └──> t_markup_profile

t_gateway_credit_limit (1) ──< (1) t_gateway_credit_usage   [配置/运行时分离]
t_gateway / t_route_rule 等配置表 ── 记录变更 ──> t_config_change_log
t_config_version ── 驱动 ──> 各节点内存快照原子刷新
```

## 二、核心表定义

### 表1：`t_gateway`（网关定义）

每个网关代表一个外部连接：流动性提供商（LP）、报价源、交易所或内部 B-Book。

```sql
CREATE TABLE `t_gateway` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `gateway_code`              VARCHAR(32)         NOT NULL COMMENT '网关代码: LMAX_TRADE, REUTERS_QUOTE',
    `gateway_name`              VARCHAR(128)        NOT NULL COMMENT '网关名称',
    `description`               VARCHAR(512)        DEFAULT NULL COMMENT '描述',

    -- 类型: 功能维度 + 来源维度
    `gateway_type`              VARCHAR(16)         NOT NULL
        COMMENT '功能类型: TRADE-交易 / QUOTE-行情 / NEWS-新闻 / SWAP-隔夜利息 / SETTLEMENT-清算',
    `provider_type`             VARCHAR(16)         NOT NULL
        COMMENT '来源类型: LP-流动性提供商 / QUOTE_SOURCE-报价源 / EXCHANGE-交易所 / INTERNAL-内部',
    `gateway_group_id`          BIGINT UNSIGNED     DEFAULT NULL COMMENT '归属网关组ID, NULL=独立网关',

    -- 连接配置
    `protocol`                  VARCHAR(16)         NOT NULL
        COMMENT '协议: FIX44 / FIX50 / REST / WEBSOCKET / BRIDGE / INTERNAL',
    `protocol_version`          VARCHAR(32)         DEFAULT NULL COMMENT '协议版本: 用于兼容性判断',
    `auth_type`                 VARCHAR(16)         NOT NULL DEFAULT 'NONE'
        COMMENT '认证方式: NONE / PASSWORD / API_KEY / CERT / OAUTH',
    `primary_host`              VARCHAR(256)        DEFAULT NULL COMMENT '主连接地址, INTERNAL为NULL',
    `primary_port`              INT UNSIGNED        DEFAULT NULL COMMENT '主连接端口',
    `secondary_host`            VARCHAR(256)        DEFAULT NULL COMMENT '备连接地址(同一对手方的链路冗余)',
    `secondary_port`            INT UNSIGNED        DEFAULT NULL COMMENT '备连接端口',
    `connection_config`         JSON                DEFAULT NULL
        COMMENT '连接扩展配置(JSON): 含认证信息(敏感字段加密), 按auth_type区分, 结构见下方',
    `data_center`               VARCHAR(16)         DEFAULT NULL COMMENT '部署机房: LD5/NY2/NY4/HK3',

    -- 运行参数
    `max_retry`                 INT UNSIGNED        NOT NULL DEFAULT 3 COMMENT '断线重连最大次数, 0=不重连',
    `retry_interval_ms`         INT UNSIGNED        NOT NULL DEFAULT 5000 COMMENT '重连间隔(毫秒)',
    `heartbeat_interval_ms`     INT UNSIGNED        NOT NULL DEFAULT 30000 COMMENT '心跳间隔(毫秒)',
    `request_timeout_ms`        INT UNSIGNED        NOT NULL DEFAULT 5000 COMMENT '请求超时(毫秒)',
    `rate_limit_per_second`     INT UNSIGNED        DEFAULT NULL COMMENT '每秒请求数上限, NULL=不限',
    `circuit_breaker_config`    JSON                DEFAULT NULL COMMENT '熔断配置(JSON), 结构见下方',
    `health_check_config`       JSON                DEFAULT NULL COMMENT '健康检查配置(JSON), 结构见下方',
    `capabilities`              JSON                DEFAULT NULL COMMENT '网关能力声明(JSON), 结构见下方',
    `fee_schedule`              JSON                DEFAULT NULL COMMENT '费用结构(JSON), 结构见下方',

    -- 状态(管理意图; 运行时连接状态/心跳由监控层维护, 不入配置表)
    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 0 COMMENT '管理状态: 0-停用 1-启用',
    `priority`                  INT                 NOT NULL DEFAULT 0 COMMENT '优先级: 值越大越优先',

    -- 审计
    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `created_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '创建人ID',
    `updated_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '最后修改人ID',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳: 0-正常, 非0-删除时Unix毫秒',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_gateway_code` (`gateway_code`, `deleted_at`),
    KEY `idx_gateway_type` (`gateway_type`, `status`),
    KEY `idx_provider_type` (`provider_type`, `status`),
    KEY `idx_gateway_group` (`gateway_group_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin COMMENT='网关定义';

-- connection_config (按 auth_type):
--   PASSWORD: {"username":"enc:..","password":"enc:..","senderCompId":"..","targetCompId":"..","beginString":"FIX.4.4"}
--   API_KEY : {"apiKey":"enc:..","secretKey":"enc:..","baseUrl":"https://.."}
--   CERT    : {"certPath":"..","keyPath":"..","caPath":".."}
--   OAUTH   : {"clientId":"..","clientSecret":"enc:..","tokenUrl":"..","scope":".."}
-- circuit_breaker_config: {"enabled":true,"errorThreshold":5,"windowSeconds":60,"halfOpenAfterSeconds":30,"successThreshold":3}
-- health_check_config   : {"enabled":true,"intervalMs":60000,"timeoutMs":5000,"url":"/health","method":"GET"}
-- capabilities          : {"supportedOrderTypes":["MARKET","LIMIT","STOP"],"partialFill":true,"modifyOrder":false,"maxOrderSize":1000000}
-- fee_schedule          : {"makerFee":"0.02%","takerFee":"0.05%","commissionPerLot":5.00,"commissionType":"PER_LOT","minCommission":1.00,"currency":"USD"}
```

### 表2：`t_gateway_group`（网关组）

将同类网关归组，提供故障转移、负载均衡、轮盘重路由能力。路由规则可指向组，由组按策略选择具体网关。

> 报价聚合能力由流动性池（表5）负责，本表不含聚合类型，避免职责重叠。

```sql
CREATE TABLE `t_gateway_group` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `group_code`                VARCHAR(32)         NOT NULL COMMENT '组代码: FAILOVER_FOREX',
    `group_name`                VARCHAR(128)        NOT NULL COMMENT '组名称',
    `description`               VARCHAR(512)        DEFAULT NULL COMMENT '描述',

    `group_type`                VARCHAR(16)         NOT NULL
        COMMENT '组类型: FAILOVER-故障转移 / LOAD_BALANCE-负载均衡 / ROUTING_WHEEL-轮盘(优先级轮转+拒单重路由)',
    `applicable_gateway_type`   VARCHAR(16)         NOT NULL DEFAULT 'TRADE'
        COMMENT '适用网关类型: TRADE / QUOTE / ALL, 限制组内只能加入对应类型网关',
    `reroute_config`            JSON                DEFAULT NULL
        COMMENT '重路由配置(JSON): 仅FAILOVER/ROUTING_WHEEL有效, 结构见下方',

    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `created_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '创建人ID',
    `updated_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '最后修改人ID',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_group_code` (`group_code`, `deleted_at`),
    KEY `idx_group_type` (`group_type`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin COMMENT='网关组';

-- reroute_config:
-- {
--   "rerouteOnReject": true, "rerouteOnTimeout": true, "rerouteOnTimeoutMs": 3000,
--   "maxRerouteAttempts": 3, "fallbackToInternal": true,
--   "rejectCodeMapping": {"LP_REJECT":"NEXT_PRIORITY","CREDIT_EXCEEDED":"SKIP_AND_ALERT"}
-- }
```

### 表4：`t_gateway_symbol_mapping`（网关品种映射）

解决上游品种与本系统品种的代码映射、精度/手数转换、非对称加点、交易限制、报价过滤、费用与时段映射。

```sql
CREATE TABLE `t_gateway_symbol_mapping` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `gateway_id`                BIGINT UNSIGNED     NOT NULL COMMENT '网关ID',
    `symbol_id`                 BIGINT UNSIGNED     NOT NULL COMMENT '本系统品种ID',
    `remote_symbol`             VARCHAR(64)         NOT NULL COMMENT '上游品种代码: EUR/USD, BTCUSDT, AAPL.US',

    -- 报价/手数转换
    `price_digits_remote`       TINYINT UNSIGNED    DEFAULT NULL COMMENT '上游价格精度(小数位), NULL=与本系统一致',
    `price_scale_factor`        DECIMAL(18,8)       DEFAULT NULL COMMENT '价格缩放: 上游价×此值=本系统价, NULL=1',
    `volume_scale_factor`       DECIMAL(18,8)       DEFAULT NULL COMMENT '手数缩放: 本系统手数×此值=上游手数, NULL=1',

    -- 价格微调(买卖分开)
    `bid_markup_points`         INT                 NOT NULL DEFAULT 0 COMMENT 'Bid加点: 负=利于券商, 正=利于客户',
    `ask_markup_points`         INT                 NOT NULL DEFAULT 0 COMMENT 'Ask加点: 正=利于券商, 负=利于客户',

    -- 交易限制(来自LP)
    `min_volume`                DECIMAL(18,4)       DEFAULT NULL COMMENT '最小下单量(手), NULL=用品种默认',
    `max_volume`                DECIMAL(18,4)       DEFAULT NULL COMMENT '最大下单量(手), NULL=用品种默认',
    `volume_step`               DECIMAL(18,4)       DEFAULT NULL COMMENT '下单步长(手), NULL=用品种默认',

    -- 报价过滤
    `max_spread_points`         INT UNSIGNED        DEFAULT NULL COMMENT '最大允许点差: 超过则丢弃该报价, NULL=不限',
    `max_deviation_points`      INT UNSIGNED        DEFAULT NULL COMMENT '最大价格偏离: 超过则告警, NULL=不限',
    `quote_depth`               TINYINT UNSIGNED    DEFAULT NULL COMMENT '报价深度(档位): NULL=仅最优价',

    -- 费用/时段/结算
    `fee_config`                JSON                DEFAULT NULL COMMENT '品种级费用(JSON): 覆盖网关fee_schedule, 结构见下方',
    `session_mapping`           JSON                DEFAULT NULL COMMENT '时段映射(JSON): 上游时段↔本地时段, 结构见下方',
    `settlement_timing`         VARCHAR(8)          DEFAULT NULL COMMENT '结算时效: T+0 / T+1 / T+2, NULL=用品种默认',

    -- 用途控制
    `use_for_quote`             TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '用于行情报价: 0-否 1-是',
    `use_for_execution`         TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '用于订单执行: 0-否 1-是',

    `last_synced_at`            DATETIME(3)         DEFAULT NULL COMMENT '最后与上游同步时间',
    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `created_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '创建人ID',
    `updated_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '最后修改人ID',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_gateway_symbol` (`gateway_id`, `symbol_id`, `deleted_at`),
    KEY `idx_symbol_id` (`symbol_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin COMMENT='网关品种映射';

-- fee_config     : {"makerFee":"0.01%","takerFee":"0.03%","commissionPerLot":3.00,"minCommission":0.50}  // NULL字段=不覆盖
-- session_mapping: {"sessions":[{"remote":"CONTINUOUS","local":"CONTINUOUS"},{"remote":"PRE_OPEN","local":"PRE_MARKET"}],"timezone":"America/New_York"}
```

### 表5：`t_liquidity_pool`（流动性池）

从多个 LP 获取报价，聚合为最优 Bid/Ask 或合并深度，提升价格质量与成交能力。

```sql
CREATE TABLE `t_liquidity_pool` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `pool_code`                 VARCHAR(32)         NOT NULL COMMENT '池代码: LP_POOL_MAJOR',
    `pool_name`                 VARCHAR(128)        NOT NULL COMMENT '池名称',
    `description`               VARCHAR(512)        DEFAULT NULL COMMENT '描述',

    `aggregation_mode`          VARCHAR(16)         NOT NULL DEFAULT 'BEST_PRICE'
        COMMENT '聚合模式: BEST_PRICE-最优价 / VWAP-加权平均 / DEPTH_MERGE-深度合并',
    `distribution_mode`         VARCHAR(16)         NOT NULL DEFAULT 'STANDARD'
        COMMENT '分发模式: STANDARD-按比例 / HEDGING-净头寸对冲 / MIXED-混合',

    -- 报价验证
    `min_active_sources`        TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '最低活跃源数: 低于则池降级',
    `stale_quote_threshold_ms`  INT UNSIGNED        NOT NULL DEFAULT 500 COMMENT '报价过期阈值(毫秒): 超过不参与聚合',
    `price_deviation_threshold_bps` INT UNSIGNED    DEFAULT NULL COMMENT '价格偏离阈值(基点): 偏离中位数超过则过滤, NULL=不过滤',
    `applicable_categories`     JSON                DEFAULT NULL COMMENT '适用品种分类(JSON): ["FOREX","METALS"], NULL=所有',

    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `created_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '创建人ID',
    `updated_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '最后修改人ID',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_pool_code` (`pool_code`, `deleted_at`),
    KEY `idx_distribution_mode` (`distribution_mode`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin COMMENT='流动性池';
```

### 表9：`t_route_rule`（路由规则）

规则全量入内存，按 `priority DESC` 排序，逐条评估 `match_conditions`，第一条命中生效（`is_default=1` 为兜底）。命中后按 `execution_mode` + `allocation_mode` 决定路由目标与分配方式。

> **设计决策——条件全 JSON 化**：规则数通常 < 500 条，内存遍历微秒级，独立字段的"可索引"优势在全内存匹配下无意义，且双模式需维护两套匹配逻辑。故匹配条件统一用 `match_conditions` JSON 表达。

```sql
CREATE TABLE `t_route_rule` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `tenant_id`                 BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '租户ID: 0=全局, 多租户隔离',
    `rule_code`                 VARCHAR(32)         NOT NULL COMMENT '规则代码: FOREX_VIP_ABOOK',
    `rule_name`                 VARCHAR(128)        NOT NULL COMMENT '规则名称',
    `description`               VARCHAR(512)        DEFAULT NULL COMMENT '规则描述',

    `match_conditions`          JSON                DEFAULT NULL
        COMMENT '匹配条件(JSON): 支持AND/OR/NOT嵌套, NULL=匹配所有(仅默认规则), 结构见下方',

    -- 路由目标
    `execution_mode`            VARCHAR(8)          NOT NULL COMMENT '执行模式: A-A-Book(STP) / B-B-Book(内部) / HYBRID-混合',
    `allocation_mode`           VARCHAR(8)          NOT NULL DEFAULT 'PRIORITY'
        COMMENT '分配模式: RATIO-按比例 / PRIORITY-按优先级故障转移 / SPLIT-拆单',

    -- Profile引用
    `execution_profile_id`      BIGINT UNSIGNED     DEFAULT NULL COMMENT '执行Profile ID, NULL=用默认',
    `markup_profile_id`         BIGINT UNSIGNED     DEFAULT NULL COMMENT '加价Profile ID, NULL=不加价',

    -- SOR与信用
    `sor_strategy`              VARCHAR(16)         DEFAULT NULL
        COMMENT '智能路由策略: NULL / BEST_PRICE / BEST_EXECUTION / MINIMIZE_FEES / FUNDING_AWARE',
    `sor_config`                JSON                DEFAULT NULL COMMENT 'SOR配置(JSON), 结构见下方',
    `credit_check_mode`         VARCHAR(8)          NOT NULL DEFAULT 'NONE'
        COMMENT '信用检查: NONE / PRE_CHECK-路由前 / POST_CHECK-路由后 / REAL_TIME-实时',

    -- 优先级与生效
    `priority`                  INT                 NOT NULL DEFAULT 0 COMMENT '优先级: 值越大越先匹配',
    `is_default`                TINYINT UNSIGNED    NOT NULL DEFAULT 0 COMMENT '默认兜底规则: 每租户最多一条',
    `effective_start`           DATETIME(3)         DEFAULT NULL COMMENT '生效开始, NULL=立即',
    `effective_end`             DATETIME(3)         DEFAULT NULL COMMENT '生效结束, NULL=永久',
    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `version`                   BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '乐观锁版本号',

    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    `created_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '创建人ID',
    `updated_by`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '最后修改人ID',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_rule_code` (`tenant_id`, `rule_code`, `deleted_at`),
    KEY `idx_status_priority` (`status`, `priority` DESC)

-- 注: 「按交易时段路由」不在本表维护。当前时段/可否交易由【时间配置模块】(t_daily_session 等)
--    决定并随订单上下文传入; 路由按时段差异化用 match_conditions.REQUEST.session 表达,

-- ================================================================
-- match_conditions 结构: 逻辑节点(logic + conditions[])可无限嵌套, 叶子为条件节点
-- 示例:
-- {"logic":"AND","conditions":[

--   POSITION: total_lots, total_profit, pending_lots, order_count, net_lots
--   REQUEST: request_time, request_ip, ip_country, platform(PC/MOBILE/API/WEB), session(ASIA/EUROPE/AMERICA), weekday(1-7)
--
-- sor_config (按 sor_strategy):
--   BEST_PRICE    : {"allowedVenues":[1,2,3],"excludedVenues":[5],"maxSlippageBps":10,"maxVenueCount":3}
--   BEST_EXECUTION: {"allowedVenues":[1,2,3],"executionTimeoutMs":5000,"partialFillAllowed":true,"venueSelectionOrder":"PRICE_THEN_SPEED"}
--   MINIMIZE_FEES : {"feeBudgetMax":10.00,"feeCurrency":"USD","considerRebates":true}
--   FUNDING_AWARE : {"minBalanceByCurrency":{"USD":1000},"creditCheckEnabled":true,"nettingPreference":"SAME_VENUE"}
-- ================================================================
```

### 表10：`t_route_rule_allocation`（路由分配）

定义规则命中后的具体路由目标。目标三选一：单网关（A-Book）、流动性池、内部（B-Book）。

> **场景**：A-Book 100%→网关；B-Book 100%→内部；混合 60%→池 + 40%→内部；故障转移按优先级；大单按 `split_config` 拆分。重路由仅在分配级（链式）与组级（表2）表达，不在网关或规则级重复配置。

```sql
CREATE TABLE `t_route_rule_allocation` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `rule_id`                   BIGINT UNSIGNED     NOT NULL COMMENT '路由规则ID',

    -- 目标(三选一, 由target_type区分, CHECK保证一致)
    `target_type`               VARCHAR(8)          NOT NULL
        COMMENT '目标类型: GATEWAY-单网关 / POOL-流动性池 / INTERNAL-内部(B-Book)',
    `gateway_id`                BIGINT UNSIGNED     DEFAULT NULL COMMENT '目标网关ID: 仅target_type=GATEWAY时非空',
    `pool_id`                   BIGINT UNSIGNED     DEFAULT NULL COMMENT '目标流动性池ID: 仅target_type=POOL时非空',

    -- 分配参数
    `allocation_ratio`          TINYINT UNSIGNED    NOT NULL DEFAULT 100
        COMMENT '分配比例(0~100): RATIO模式有效, 同规则启用分配之和必须=100',
    `allocation_priority`       INT                 NOT NULL DEFAULT 0 COMMENT '分配优先级: PRIORITY模式有效, 值越大越优先',

    `split_config`              JSON                DEFAULT NULL COMMENT '拆单配置(JSON): SPLIT模式有效, 结构见下方',
    `reroute_to_allocation_id`  BIGINT UNSIGNED     DEFAULT NULL COMMENT '重路由目标分配ID: 当前拒单时路由到此, 形成链式重路由',
    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',

    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),

    PRIMARY KEY (`id`),
    KEY `idx_rule_id` (`rule_id`, `status`),
    KEY `idx_pool_id` (`pool_id`, `status`),
    CONSTRAINT `ck_alloc_target` CHECK (
        (`target_type`='GATEWAY'  AND `gateway_id` IS NOT NULL AND `pool_id` IS NULL) OR
        (`target_type`='POOL'     AND `pool_id`    IS NOT NULL AND `gateway_id` IS NULL) OR
        (`target_type`='INTERNAL' AND `gateway_id` IS NULL     AND `pool_id` IS NULL)
    )
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin COMMENT='路由分配';

-- split_config: {"splitThresholdVolume":50,"maxSplitCount":5,"splitMethod":"EQUAL","minSplitVolume":5,"splitDelayMs":100,"timeoutPerSplitMs":3000}
```

## 三、索引、分区与数据保留

### 3.1 索引原则

- 配置表：以唯一键 + 状态查询为主，索引精简；JSON 字段不建索引（全量入内存匹配）。
- 流水/指标表：以时间范围 + 维度查询为主，重点索引时间字段。
- 所有唯一索引包含 `deleted_at`（软删除并存）。

### 3.2 分区与保留

> MySQL 分区键必须包含在主键中，故 `t_route_audit_log` 主键为 `(id, created_at)`、`t_gateway_metric` 为 `(id, metric_time)`。建议建表即带分区，并自动化滚动维护未来分区。

```sql
ALTER TABLE `t_route_audit_log`
PARTITION BY RANGE (TO_DAYS(created_at)) (
    PARTITION p202601 VALUES LESS THAN (TO_DAYS('2026-02-01')),
    PARTITION p202602 VALUES LESS THAN (TO_DAYS('2026-03-01')),
    PARTITION p_future VALUES LESS THAN MAXVALUE
);
```

| 表 | 保留周期 | 归档策略 |
|----|---------|---------|
| `t_route_audit_log` | 6 个月在线，2 年归档 | 超期分区迁移归档库 |
| `t_gateway_metric` | 3 个月在线，1 年归档 | 超期分区删除 |
| `t_config_change_log` | 长期（合规常需 ≥5 年） | 按年归档 |
| 配置表 | 永久 | 软删除，不物理删除 |

## 五、数据流转与运算

本章把上述表"串起来"：一笔订单进入后，依次读取哪些表、做什么运算、产出什么，最后回写哪些表。

### 5.1 路由决策主流程（读路径）

**输入**：订单上下文（账户、品种、方向 `trade_cmd`、量 `volume`、客户属性、当前持仓、请求环境，以及由【时间配置模块】给出的当前时段/可交易状态），全部为内存对象，不查库。

```
订单上下文(含时段/可交易状态, 来自时间配置模块)
  │
  ① 规则匹配      t_route_rule              → 命中一条规则
  │
  ② 确定目标      t_route_rule_allocation   → 目标清单(网关/池/内部) + 分配方式
  │
  ③ 报价聚合      t_liquidity_pool(+member) → 最优 Bid/Ask (目标为池或启用SOR时)
  │              t_gateway_symbol_mapping   → 上游↔本地 代码/精度/手数 转换 + 一层加点
  │
  ④ 信用检查      t_gateway_credit_limit/usage → 可用额度判定，超限则跳过/拒单/重路由
  │
  ⑤ 定价计算      t_markup_profile          → 客户最终 Bid/Ask + 佣金 + 隔夜利息
  │
  ⑥ 执行下发      t_execution_profile        → 执行指令(成交方式/滑点/审核)，受 t_gateway.capabilities 约束
  │                                            交给执行器：A-Book→LP适配器 / B-Book→撮合引擎
  │
  ⑦ 记录          t_route_audit_log          → 异步写入全过程决策
```

每一步的表与运算：

| 步骤 | 用到的表/字段 | 运算 |
|------|-------------|------|
| ① 规则匹配 | `t_route_rule.match_conditions`、`priority`、`is_default`、`effective_*` | 取 `status=1` 且在生效期、租户匹配的规则，按 `priority` 降序；逐条对 `match_conditions` 条件树求值，**第一条命中即停**；都不中则用 `is_default=1` 兜底。按时段差异化路由时，条件树用 `REQUEST.session`（时段由时间模块带入） |
| ② 确定目标 | `t_route_rule.execution_mode/allocation_mode`、`t_route_rule_allocation` | 取该规则所有启用分配，按 `allocation_mode` 计算（见 5.2） |
| ③ 报价聚合+映射 | `t_liquidity_pool(_member)`、`t_gateway_symbol_mapping` | 池聚合最优价；品种代码/精度/手数转换 + 第一层加点（见 5.3、5.4） |
| ④ 信用检查 | `t_gateway_credit_limit` + `t_gateway_credit_usage` | `可用 = limit − used − reserved`，不足则按 `exceed_action` 处理（见 5.5） |
| ⑤ 定价计算 | `t_markup_profile`（规则的 `markup_profile_id`） | 第二层加点 + 佣金 + 隔夜利息，含时段加价 `session_overrides`（见 5.4） |
| ⑥ 执行下发 | `t_execution_profile`、`t_gateway.capabilities` | 滑点上限、人工审核判定；能力校验后交执行器 |
| ⑦ 记录 | `t_route_audit_log` | 命中规则、目标、各步明细（`sor_detail`/`credit_check_result`/`markup_detail`…）异步落库 |

### 5.2 分配策略运算（步骤②）

`allocation_mode` 决定如何从 `t_route_rule_allocation` 的多个目标中选择：

- **PRIORITY（故障转移）**：选 `allocation_priority` 最大且可用的目标；失败则按链 `reroute_to_allocation_id` 或组内 `priority` 顺序降级。
- **RATIO（按比例）**：按各分配 `allocation_ratio`（之和=100）做加权分流，单笔订单按权重命中一个目标，整体逼近比例。
- **SPLIT（拆单）**：当 `volume > split_config.splitThresholdVolume`，拆为 ≤ `maxSplitCount` 笔、每笔 ≥ `minSplitVolume`，按 `splitMethod`（等量/按比例）分配。

### 5.3 报价聚合运算（步骤③，目标为池时）

从 `t_liquidity_pool_member` 的各 LP 报价中：

1. **过滤**：丢弃过期报价（`报价龄 > stale_quote_threshold_ms`）和偏离过大报价（`|价 − 中位数| > price_deviation_threshold_bps`）；有效源数 < `min_active_sources` 则池降级。
2. **聚合**（按 `aggregation_mode`）：
   - `BEST_PRICE`：`bid = max(各有效bid)`，`ask = min(各有效ask)`（点差最优）。
   - `VWAP`：按量加权平均。
   - `DEPTH_MERGE`：合并各 LP 订单簿深度，支撑大单成交。

### 5.4 品种映射与定价运算（步骤③/⑤）

**映射转换**（`t_gateway_symbol_mapping`，仅在与某具体网关交互时）：

- 代码：`remote_symbol` ↔ 本系统品种
- 价格：`本系统价 = 上游价 × price_scale_factor`（NULL 视为 1）
- 手数：`上游手数 = 本系统手数 × volume_scale_factor`
- 报价过滤：`点差 = ask − bid`，若 `> max_spread_points` 则丢弃该报价

**两层加点**（点的符号约定：bid 负=利于券商，ask 正=利于券商）：

```
第一层(接入层, t_gateway_symbol_mapping)：与客户无关的基础点差
  bid1 = bid + bid_markup_points
  ask1 = ask + ask_markup_points

第二层(客户分层, t_markup_profile)：取值优先级 symbol_overrides > category_overrides > default
  最终bid = bid1 + profile_bid_markup + session_overrides.additionalBidMarkup(按当前时段)
  最终ask = ask1 + profile_ask_markup + session_overrides.additionalAskMarkup
```

**佣金**（`t_markup_profile`，按 `commission_type`）：

- `PER_LOT`：`佣金 = commission_per_lot × volume`
- `PERCENT`：`佣金 = 名义金额 × commission_percent`（名义 = 价 × volume × 合约规模）
- `PER_DEAL`：固定
- 最后 `佣金 = max(佣金, min_commission)`

### 5.5 信用额度运算（步骤④）

```
可用额度 = limit_amount − used_amount − reserved_amount
若 本单占用 > 可用额度 → 按 exceed_action: SKIP(跳过下一个) / REJECT(拒单) / ALERT(仅告警放行) / REROUTE(重路由)
```

`本单占用` 取决于 `credit_type`：EXPOSURE=名义敞口、VOLUME=手数、NETTING=净头寸增量。

### 5.6 状态回写（写路径）

| 触发时机 | 运算 | 回写表 |
|---------|------|-------|
| 路由成功提交 | `reserved_amount += 本单占用`（原子条件 UPDATE + `version` 乐观锁） | `t_gateway_credit_usage` |
| 成交回报 | `reserved -= 本单占用; used += 实际成交占用` | `t_gateway_credit_usage` |
| 网关拒单 | 按组 `reroute_config` + 成员 `can_reroute` + 分配 `reroute_to_allocation_id` 选下一个目标，追加一条重路由记录 | `t_route_audit_log.reroute_history` |
| 周期重置 | 到 `reset_cycle` 周期清零占用 | `t_gateway_credit_usage` |
| 定时聚合 | 按窗口统计延迟/成交率/滑点等 | `t_gateway_metric` |

### 5.7 配置如何进入内存（写配置 → 运行时）

配置表全量入内存匹配，变更不直接生效，经"发布"统一切换：

```
改配置(草稿) → 发布(写 t_config_version, 版本号+1) → 各节点拉快照 → 构建内存索引 → 指针原子切换
                                                                   ↑ 同时写 t_config_change_log 留痕
```

> **与撮合引擎的边界**：本设计到"算出路由决策 + 下发执行指令"为止。`execution_mode=B / target_type=INTERNAL` 时把订单与执行参数交给独立的撮合/Dealing 引擎；`A` 时交给 LP 网关适配器。订单簿、撮合算法、成交生成、持仓与盈亏不在本设计范围。

## 六、跨域契约对齐（v2.3 补充）

> 本节为 v2.3 对齐《平台跨域契约补充卷》v1.x 的补充章节。本文档其余章节凡与本节冲突的，以本节为准。

### 6.1 跨域契约引用声明

- **错误码**：遵循补充卷 §三，使用 `ROUTE_*` 前缀。
- **事件**：遵循补充卷 §四 CloudEvents envelope。本域新增生产 `revenue.realized.v1`（见 §6.6）。
- **角色**：遵循补充卷 §五，路由配置管理权限点引用 `PMS_*_MANAGER`。
- **ID**：遵循补充卷 §六，`route_rule_id`、`execution_link_id` 由本域生成。
- **API**：对外提供 `RouteResolver` gRPC（见补充卷 §9.1）。
- **容灾/容量/监控**：遵循补充卷 §十/§十一/§十二最低基线。

### 6.2 `t_markup_profile` 下沉（消除三角重复）

旧稿 `t_markup_profile` 同时含 markup、swap、commission 三类字段，与品种域 `t_symbol_swap_config`、交易域 `t_account_group_commission` 形成三角重复。v2.3 裁定：

| 字段类 | 旧位置 | 新权威 | 路由层职责 |
|--------|--------|--------|------------|
| 接入层加点（spread markup） | `t_markup_profile.markup_*` | **保留在 `t_markup_profile`** | 路由层产出 Client Quote 的第一层加点 |
| 隔夜利息加点（swap markup） | `t_markup_profile.swap_*` | **下沉至品种域 `t_symbol_swap_config`**（BASE + 覆盖合并公式见品种域） | 路由层不再持有 swap 字段 |
| 佣金加点（commission markup） | `t_markup_profile.commission_*` | **下沉至交易域 `t_account_group_commission`** | 路由层不再持有 commission 字段 |

> **迁移**：v2.3 起 `t_markup_profile` 的 `swap_*`、`commission_*` 字段标记 `status=DEPRECATED`，新接入层只读 `markup_*`。已有数据由 ETL 迁移至品种/交易域，过渡期双读。

### 6.3 运行时 API：`RouteResolver`

本域对外提供 gRPC `RouteResolver`（契约见补充卷 §9.1），输入输出：

```
RouteRequest {
  symbol_id, account_id, entity_id,
  session,            // ASIA/EUROPE/AMERICA（来自时间域映射，见 §6.4）
  order_volume, side,
  config_version
}
RouteResponse {
  route_rule_id,
  allocation_mode,    // PRIORITY/RATIO/SPLIT
  allocations[],      // [{gateway_id, ratio, execution_profile_id}]
  reject_reason       // 引用补充卷 §三
}
```

撮合引擎经此 RPC 获取路由决策，**禁止**在引擎侧重新实现 `match_conditions` 求值（求值器作为共享库被本域服务与引擎共用，确保一致）。

### 6.4 `REQUEST.session` 与时间域映射

时间域 `SessionQueryService.GetSessionState` 输出 `session_type`（REGULAR/PRE/POST/OVERNIGHT/QUOTE_ONLY）与 `runtime_state`（OPEN/CLOSE_ONLY/HALT）。本域路由 `match_conditions` 中的 `session` 字段取**平台时区映射后的区域枚举**：

| 时间域 session_type | 路由 session（按品种 primary_timezone 归类） |
|---------------------|---------------------------------------------|
| REGULAR（亚盘） | ASIA |
| REGULAR（欧盘） | EUROPE |
| REGULAR（美盘） | AMERICA |
| PRE/POST/OVERNIGHT | EXTENDED |
| QUOTE_ONLY | QUOTE_ONLY |
| HALT | HALT（路由直接拒绝，返回 `SESS_STATE_MARKET_CLOSED`） |

> 映射表由本域与时间域联合维护，作为补充卷附录登记。

### 6.6 `revenue.realized.v1` 生产方声明（解决代理域收入事件缺口）

本域在 A-Book / B-Book / Mixed 成交后异步产出 `revenue.realized.v1`（schema 见补充卷 §4.4），作为代理域 `NET_REVENUE_SHARE` 计算的**唯一收入事件来源**。

```
生产时机：
  - B-Book：成交 Commit 后异步计算 realized_markup_revenue（Core Book 价与 Client Quote 价差 × volume）
  - A-Book：LP 回报成交后，按 markup_profile.markup_* 计算
  - Mixed：各子单分别计算后聚合

distributable_revenue = realized_markup_revenue - lp_execution_cost
                     - exchange_and_regulatory_fees - non_shareable_tax
```

> 代理域**禁止**自行推算 `realized_markup_revenue`，必须消费本事件。

### 6.8 重路由状态机

```
[SUBMITTED] --成功--> [EXECUTING] --成交--> [FILLED]
     |                     |
     |拒单                 |拒单
     v                     v

```

### 6.9 池聚合降级矩阵

| 场景 | BEST_PRICE | VWAP | DEPTH_MERGE |
|------|------------|------|-------------|
| 全源 STALE | 降级：返回 `ROUTE_DEGRADED_POOL_DEGRADED` | 同左 | 同左 |
| 单源 STALE | 剔除该源，剩余继续 | 剔除 | 剔除该档深度 |
| 单源返回 | 用该源价 | 用该源价（VWAP 退化为单点） | 用该源深度 |
| 空池 | `ROUTE_NOTFOUND_RULE_MATCH` | 同左 | 同左 |

### 6.10 与行情 Core Book 职责裁定

- **Core Book**（行情域产出）：LP 原始报价归一化 + 池聚合前的原始价，供撮合引擎定价/估值/强平。
- **Client Quote**（本域产出）：Core Book + `markup_profile.markup_*`，供图表/下单展示/WS。
- **池聚合**：本域 `t_liquidity_pool` 用于 A-Book 多 LP 报价合成最优价；行情域 Core Book 不做 A-Book 池聚合（行情域只做归一化与品种级过滤）。
- **裁定**：A-Book 执行路径的池聚合由本域负责；行情域 Core Book 是 B-Book 与估值的权威。两者不重叠。
