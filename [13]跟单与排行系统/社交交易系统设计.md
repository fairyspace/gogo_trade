# 社交交易系统设计（权威版）

> **版本**：v1.0 Final  
> **定位**：零售 / 专业客户多资产 CFD·FX 经纪平台的**跟单（Copy）执行权威** + **信号源业绩与交易排行榜权威** + **跟单分成结算意图权威**。  
> **对标**：Spotware cTrader Copy、MetaQuotes MQL5 Signals、eToro CopyTrader（能力级，不照搬产品形态）。  
> **运行时分层**：Copy Orchestrator（订阅/放大/限速）+ Performance Projector（业绩/排行）+ 既有 Trading Engine（成交权威）。**方向设计不锁定实现语言。**  
> **域地图**：见 [`[总览]经纪平台域地图.md`](../[总览]经纪平台域地图.md)。  
> **命名**：本域 = **社交交易（Social Trading）** = 跟单 + 排行榜；完整 IM / 大厅广场见 《内容与社区系统设计》，营销活动见 《营销与推荐系统设计》。

## 一、系统定位与对标决策

### 1.1 系统定位

Social Copy Domain 负责：

1. **信号源（Provider）注册与可见性**（交易员账户 / 策略实例映射）  
2. **订阅关系与跟单参数**（比例、固定手、净值比、最大滑点、品种过滤）  
3. **信号生成与 1→N 放大下单**（经引擎 `SubmitOrder`，`source=SIGNAL`）  
4. **业绩统计与交易排行榜**（可审计、带风险披露）  
5. **跟单分成 / 订阅费结算意图**（入账走资金或 Partner 钱包模式契约，本域不算客户交易盈亏）  
6. **可选：信号源主页轻量动态与评论**（非群聊）

以下概念**不属于**本系统：

| 概念 | 归属 |
|------|------|
| Order/Deal/Position 写权威、PreTrade/强平 | 交易 / 风控 |
| 完整群聊、大厅广场、私聊 | 内容与社区 [15] |
| IB/Affiliate 归因与返佣账本 | 代理体系 |
| OpenAPI / 托管策略运行时 / 回测引擎 | 量化系统 |
| 出入金、Credit 发放规则 | 资金 / 营销与推荐活动 |
| 资讯头条 CMS | 内容与社区 [15] |

### 1.2 行业参考的正确使用方式

| # | 厂商 | 可借鉴 | 本设计采纳 / 取舍 |
|---|------|--------|------------------|
| 1 | **cTrader Copy** | 订阅、比例跟单、提供者业绩 | 采纳订阅+放大；不照搬其结算细节 |
| 2 | **MQL5 Signals** | 信号订阅、业绩展示、分成 | 采纳业绩披露与分成意图；不自研 MQL |
| 3 | **eToro** | Copy + 社交动态 | 采纳轻量动态；**完整 IM 不进本域** |
| 4 | **MetaQuotes** | magic 归因 | 与交易域 `magic_number` 对齐 |
| 5 | **Quant/API 平台** | 策略可作信号源 | 量化只生产可选信号，Copy 在本域 |

### 1.3 产品定位（拍板）

- **核心**：跟单订阅 + 信号放大 + 排行榜 + 合规披露。  
- **轻量社交**：信号源动态/评论可选。  
- **不做**：聊天室、自动解析自然语言下单、未授权晒全账户持仓。

## 二、十四条不可破坏决策

1. **成交只经引擎**：本域禁止直写 `t_order` / `t_deal` / `t_position` / `t_account_ledger`。  
2. **同一 PreTrade**：跟单订单与手工单同一风控/保证金/熔断；另叠加订阅级与账户级跟单限额。  
3. **幂等强制**：每笔放大单带账户内唯一 `client_order_id` + `request_id`；建议 `{subscription_id}:{signal_id}`。  
4. **强制归因**：`source=SIGNAL`，`magic_number` = 订阅级全局唯一 magic。  
5. **限速放大**：1 信号 → N 订阅者必须经独立队列与账户节流，保护引擎。  
6. **排行榜可复算**：业绩只基于已发生 Deal/权益曲线投影；禁止手填收益率。  
7. **合规披露强制**：排行与信号源页必须带风险声明；过去收益不保证未来。  
8. **`entity_id` 贯穿**：信号源、订阅、信号、分成均带实体；禁止跨实体跟单。  
9. **反向跟单默认关**：开启需二次确认与额外风险提示。  
10. **取消订阅不强制平仓**：默认保留已有仓；可选 `flatten_on_unsubscribe`。  
11. **事件经 Outbox**：订阅变更、信号、放大结果、分成意图禁止「写库后尽力发 MQ」。  
12. **IM 非本域**：群/大厅归内容与社区；本域最多深链落地页。  
13. **晒单非自动**：持仓展示给第三方须用户授权；IM 侧卡片由运营域编排。  
14. **不做的事**：凭聊天下单、把排行榜当营销刷数池、绕过引擎改仓。

## 三、边界与跨系统契约

### 3.1 领域边界

| 系统 | 权威数据 | 本域使用方式 |
|------|---------|--------------|
| 交易引擎 | Order/Deal/Position、SubmitOrder | **唯一写交易路径** |
| 行情 | Client Quote | 展示；成交价规则仍在引擎（Core） |
| 账户 | 账户归属、`allow_ea`/`allow_api` 类开关（可扩展 `allow_copy`） | 绑定跟随账户；校验 user↔account |
| 资金 | 入账 RPC / Partner 出款 | 分成结算意图落地 |
| 量化 | 策略实例、可选 `signal.generated` 适配 | 可作为 Provider 类型之一 |
| 代理 | 归因/佣金 | 弱耦合；跟单分成 ≠ IB 返佣 |
| 内容与社区 | IM、CMS | 深链与系统卡片；不拥有订阅 |
| 营销与推荐 | 活动、推荐 | 可深链本域 Provider；不拥有订阅 |
| 通知 | 模板送达 | 订阅成功/放大失败等事件 |
| 权限 | RBAC/SoD | 信号源上架审核、人工干预 |

### 3.2 下单契约

```text
Copy Orchestrator
  → 校验订阅有效、限额、品种过滤、账户 allow_copy
  → SubmitOrder(
       source=SIGNAL,
       magic_number=subscription.magic,
       client_order_id="{subscription_id}:{signal_id}",
       request_id=...,
       volume=sizing(provider_volume, mode, equity),
       ...
     )
```

| 入口 | `source` | `magic_number` | `client_order_id` 建议 |
|------|----------|----------------|------------------------|
| 跟单放大 | `SIGNAL` | `t_copy_subscription.magic_number` | `{subscription_id}:{signal_id}` |
| 信号源自成交（被跟） | `CLIENT`/`EA`/`API` | 其自身 magic 或 0 | 原路径不变 |

### 3.3 标准事件

| 事件 | 时机 | 消费者 |
|------|------|--------|
| `copy.provider.listed/suspended.v1` | 上架/停用 | 排行、通知、运营 |
| `copy.subscription.created/cancelled.v1` | 订阅变更 | 通知、运营卡片 |
| `signal.generated.v1` | 信号落定 | Copy 放大（本域） |
| `copy.execution.accepted/rejected.v1` | 单笔放大结果 | 通知、审计 |
| `copy.performance.snapshot.v1` | 日切/定时 | 排行投影 |
| `copy.fee.accrued.v1` | 分成计提意图 | 资金/结算适配 |

本域**消费**交易域 `deal.*` / `position.*` 用于信号检测（若 Provider 模式=镜像成交）与业绩计算，不反向改成交。

### 3.4 与内容与社区 IM 交叉契约

- IM 可深链：`/copy/providers/{id}`。  
- 跟单成功可由内容与社区发**系统卡片**到群（编排在 [15]，事实来自本域事件）。  
- **禁止**：IM 解析文本直接 `SubmitOrder`。

## 四、领域模型

### 4.1 聚合

```text
CopyProvider（信号源）
  ├── 绑定 account_id 或 strategy_instance_id
  ├── 可见性 / 上架状态 / 风险声明版本
  └── PerformanceSnapshot[]（投影）

CopySubscription（订阅）
  ├── follower_account_id
  ├── sizing_mode + params
  ├── magic_number（全局唯一）
  └── 状态：ACTIVE / PAUSED / CANCELLED

CopySignal（信号）
  ├── provider_id、symbol、side、volume 意图、价格参考
  └── 幂等键 provider_signal_key

CopyExecution（放大意图/结果）
  └── subscription_id + signal_id → 引擎 order_ticket / reject_reason

LeaderboardBoard（排行榜配置）
  └── 指标、窗口、过滤、披露模板
```

### 4.2 跟单比例模式

| `sizing_mode` | 语义 |
|---------------|------|
| `FIXED_LOT` | 固定手数 |
| `RATIO` | 相对信号源手数比例 |
| `EQUITY_RATIO` | 按净值比例缩放 |
| `MIRROR_BALANCE` | 按余额比镜像（慎用，需限额） |

均需：`max_lots`、`max_open_positions`、`max_daily_loss`、品种白/黑名单。

### 4.3 信号生成模式

| 模式 | 说明 |
|------|------|
| `MIRROR_DEAL` | 消费 Provider 账户 `deal.created` 生成信号 |
| `STRATEGY_HOOK` | 量化策略实例显式发信号（适配器） |
| `MANUAL_PROVIDER` | 提供者手动发信号（需审核开关，默认关） |

## 五、状态机

### 5.1 信号源

```text
DRAFT → PENDING_REVIEW → LISTED ⇄ SUSPENDED → DELISTED
```

上架须：风险声明签署、最低交易历史/披露字段齐全、实体合规开关。

### 5.2 订阅

```text
CREATED → ACTIVE ⇄ PAUSED → CANCELLED
```

`PAUSED`：不再放大新信号；已有仓按用户配置处理。

### 5.3 放大执行

```text
QUEUED → SUBMITTED → ACCEPTED | REJECTED | SKIPPED
```

`SKIPPED`：过滤规则（品种/限额/账户只平）命中，不计失败风暴。

---

## 六、排行榜

### 6.1 原则

1. 指标来自可复算投影：收益率、最大回撤、盈亏比、跟单人数、交易天数等。  
2. 窗口：`7D/30D/90D/ALL` 可配；防刷：最小成交次数、最小交易天数。  
3. 展示强制：风险声明、是否含模拟账户、是否跟单净值曲线。  
4. **禁止**运营手工改名次；仅可下架违规 Provider（审计）。

### 6.2 投影流

```text
deal.*/equity 快照
  → Performance Worker 增量更新
  → t_copy_performance_snapshot
  → Leaderboard 物化视图 / Redis 排行 ZSET
```

## 七、分成与费用（意图层）

| 类型 | 说明 | 落地 |
|------|------|------|
| 订阅费 | 跟随者周期付费 | 资金扣款/站内结算意图 |
| 利润分成 | 高水位利润抽成 | 日/周结算意图事件 |
| 平台抽成 | 从上述费用切分 | 公司 journal（资金域） |

本域产出 `copy.fee.accrued.v1`；**实际扣款/入账**走资金引擎契约。可复用代理「钱包入账」模式，但账本主体不是 IB Program。

## 十一、SLO 与风险

| 指标 | 目标 |
|------|------|
| 信号→首笔放大提交 P99 | 视规模，常态 &lt; 500ms（同机房，不含引擎成交） |
| 单信号放大扇出限速 | 可配；默认保护引擎热路径 |
| 排行延迟 | 分钟级可接受；交易路径零依赖排行 |
| 跟单故障影响手工交易 | **0**（松耦合） |

风控叠加：订阅级日损停跟、全站 Copy Kill Switch（暂停放大，不替代账户 StopOut）。

## 十二、明确不做与常见误区

| 误区 | 正确做法 |
|------|----------|
| 「跟单放在量化里一起做」 | 执行与排行在本域；量化只做可选信号源 |
| 「排行榜单独一个系统」 | 本域业绩投影 |
| 「群聊算社交交易」 | IM 在内容与社区 |
| 「聊天里说买入就下单」 | 禁止 |
| 「运营改排行名次冲活动」 | 禁止；活动发奖走营销与推荐+资金 |
| 「跟单绕过保证金」 | 同一 PreTrade |
