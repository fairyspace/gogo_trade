1
📥 事件接入 & 通知发送 INTERNAL
Business systems publish notification events via internal API
🏦 风控引擎检测到用户追保预警，调用通知中心内部 API 发布事件。通知中心幂等校验后，按策略编排 → 模板渲染 → 多通道投递。上游系统只需关心"发事件"，通道选择、降级、限频全部由通知中心负责。
POST /internal/v1/notification/events 业务事件上报
📋 Request Headers
Header类型必填说明
X-Entity-IdLong必填法律实体ID
X-Trace-IdString必填全链路追踪ID
Idempotency-KeyString必填幂等键，重复提交返回已处理结果
AuthorizationString必填mTLS 服务身份认证
📤 Request Body
字段类型必填说明
eventIdString必填事件唯一ID
sourceSystemString必填来源系统：RISK_ENGINE / TRADE_ENGINE / FUND_SYSTEM
eventTypeString必填事件类型：MARGIN_CALL / ORDER_FILLED / DEPOSIT
bizTypeString必填业务域：TRADE / FUND / RISK / ACCOUNT / ASSET / IB / MARKET / SERVICE / MARKETING
bizIdString必填业务单号
userIdLong必填目标用户ID
priorityString必填优先级：P0 / P1 / P2 / P3
localeString选填语言：zh-CN / zh-TW / en / ar / ja，默认 zh-CN
correlationIdString选填关联ID（同一业务流程多事件关联）
payloadObject必填事件内容（模板变量数据源）
overridesObject选填覆盖配置：templateCode / actionUrl / expireAt / channelPlanOverride
✅ Response (成功)
JSON { "code": 0, "message": "success", "data": { "accepted": true, "duplicate": false, "eventId": "evt_20250409_risk_001", "messageId": "msg_202504091029001", "traceId": "tr_abc123", "status": "RECEIVED", "estimatedDeliveryMs": 300 } }
🔁 Response (幂等命中)
IDEMPOTENT { "code": 0, "message": "idempotent_hit", "data": { "accepted": true, "duplicate": true, "eventId": "evt_20250409_risk_001", "messageId": "msg_202504091029001", "status": "SENT", "hint": "Event already processed" } }
📌 幂等机制：Redis SET NX (TTL=24h) 为第一道防线 + DB 唯一索引 (idempotency_key) 兜底。重复提交返回已处理结果，不会产生重复通知。

📬 站内信收件箱 CLIENT
User inbox: list, read, ack, delete, pin
👤 用户打开 App 站内信页面，看到未读消息列表。点击追保预警消息后标记已读，再点击"确认"按钮完成合规确认。系统记录确认时间戳，异步写入 WORM 合规存储。
GET /api/v1/inbox/messages?category={category}&status=unread&page=1&pageSize=20 查询收件箱
✅ Response
JSON { "code": 0, "data": { "unreadCount": 3, "unackedCount": 1, "total": 128, "page": 1, "pageSize": 20, "items": [ { "inboxId": "inbox_1001", "messageId": "msg_202504091029001", "category": "RISK", "priority": 0, "title": "【追保预警】您的账户维持担保比例已降至128.5%", "contentPreview": "请于T+1日14:30前补充担保物...", "actionUrl": "app://margin/collateral", "ackPolicy": "CONFIRM", "isRead": false, "isAcked": false, "isPinned": true, "createdAt": "2025-04-09T09:29:00Z" } ] } }
▼ ▼ ▼
POST /api/v1/inbox/messages/{inboxId}/read 标记已读
更新 is_read=1，read_at=NOW()。同步更新对应 ChannelTask 的 read_at。
POST /api/v1/messages/{messageId}/ack 显式确认
📤 Request Body
JSON { "clientTime": "2025-04-09T09:31:50Z", "deviceId": "iphone_abc123", "channel": "INBOX" }
🔒 P0 风控/合规类消息必须确认。确认后异步写入 WORM（消息体+确认时间+设备ID）。
▼ ▼ ▼
DELETE /api/v1/inbox/messages/{inboxId} 删除收件箱消息
⚠️ 软删除（is_deleted=1），不可物理删除审计记录。用户删除后前端不再展示，后端审计数据保留。

🔍 消息状态查询 INTERNAL CLIENT
Full lifecycle tracing: event → message → channel tasks
🔍 客服需要排查某条追保预警消息的投递情况。调用状态查询接口，可看到完整的通道任务列表：Push 失败（Token 失效）→ 降级 SMS 成功送达 → 用户在站内信确认。全链路可追踪。
GET /api/v1/messages/{messageId}/status 消息状态查询
✅ Response
JSON { "code": 0, "data": { "messageId": "msg_202504091029001", "eventId": "evt_20250409_risk_001", "msgType": "MARGIN_CALL_WARNING", "priority": "P0", "userId": 88001234, "status": "COMPLETED", "ackPolicy": "CONFIRM", "channelTasks": [ { "taskId": "task_push_001", "channel": "PUSH", "taskStatus": "FAILED", "isFallback": false, "lastErrorCode": "INVALID_TOKEN" }, { "taskId": "task_inbox_001", "channel": "INBOX", "taskStatus": "ACKED", "isFallback": false, "deliveredAt": "2025-04-09T09:29:00.050Z", "readAt": "2025-04-09T09:31:45Z", "ackAt": "2025-04-09T09:31:50Z" }, { "taskId": "task_sms_001", "channel": "SMS", "taskStatus": "DELIVERED", "isFallback": true, "deliveredAt": "2025-04-09T09:29:32Z" } ], "timeline": { "createdAt": "2025-04-09T09:29:00.000Z", "renderedAt": "2025-04-09T09:29:00.050Z", "completedAt": "2025-04-09T09:31:50.000Z" } } }
📌 channelTasks 展示所有通道任务（含降级通道），isFallback=true 标识降级触发的任务。

⚙️ 用户偏好 & 同意管理 CLIENT
Preference center + Consent management + Quiet hours
👤 用户进入通知设置页面，关闭营销类推送，设置静默时段 22:00-08:00。系统立即生效，后续营销消息仅走站内信。但风控/安全类消息不受偏好限制，始终强制发送。

✅ Response

▼ ▼ ▼
PUT /api/v1/users/{userId}/notification-preferences 更新通知偏好
📤 Request Body
JSON { "language": "zh-CN", "timezone": "Asia/Shanghai", "channelOpt": { "PUSH": true, "SMS": false, "EMAIL": true }, "categoryOpt": { "MARKETING": false }, "quietHours": { "enabled": true, "start": "22:00", "end": "08:00" } }
⚠️ complianceLocked 分类的偏好由系统忽略，即使前端传入也不会生效。P0/P1 合规消息始终强制发送。
▼ ▼ ▼
POST /api/v1/users/{userId}/consents/marketing/grant 营销授权
记录授权来源(APP/WEB)、时间、凭证引用。GDPR/个人信息保护法合规必备。
POST /api/v1/users/{userId}/consents/marketing/revoke 营销退订
退订后 24h 内生效。仅影响 P2/P3 营销类消息，P0/P1 合规类不受影响。
5
📱 Push 设备管理 CLIENT
Device token registration, update & deregistration
📱 用户安装 App 并登录后，客户端自动注册设备 Token。一个用户可绑定多个设备（手机+平板+Web），Push 时向所有有效设备发送。Token 失效时，通过回执自动标记 INVALID 并记录失败原因。
POST /api/v1/push/devices 注册 Push 设备
📤 Request Body
JSON { "appId": "broker-app", "platform": "IOS", // IOS / ANDROID / WEB "pushProvider": "APNS", // APNS / FCM / HMS / MIPUSH / OPPO / VIVO / WEBPUSH "deviceFingerprint": "fp_iphone15_abc123", "deviceToken": "apns_token_abc123xyz", "environment": "PROD", // PROD / UAT / DEV "appVersion": "5.2.1" }
📌 唯一键：(entity_id, user_id, app_id, device_fingerprint, push_provider)。重复注册则更新 Token。
▼ ▼ ▼
DELETE /api/v1/push/devices/{deviceId} 注销设备
用户退出登录时调用。标记 token_status=INVALID，后续不再向该设备推送。

📝 模板管理 ADMIN
Template versioning, approval flow, publish & retire
📋 运营团队需要修改追保预警的短信模板措辞。创建新版本 v4 → 提交审批 → 合规团队审批通过 → 发布上线。旧版本 v3 自动标记为 RETIRED。全程版本化追踪，任何时刻可回溯历史版本。
POST /internal/v1/templates 创建模板
📤 Request Body (核心字段)
字段类型说明
templateCodeString模板代码：MARGIN_CALL_WARNING
categoryString所属业务域
audienceTypeStringTRANSACTIONAL / REGULATORY / MARKETING / SERVICE
jurisdictionString司法辖区：CN / HK / SG / US / EU
defaultPriorityInteger默认优先级：0=P0 1=P1 2=P2 3=P3
allowUnsubscribeBoolean是否允许退订
approvalRequiredBoolean变更是否需审批
▼ ▼ ▼
POST /internal/v1/templates/{templateCode}/versions 创建模板版本
📤 Request Body (核心字段)
字段类型说明
versionString版本号：v4
localeString语言：zh-CN
channelContentJsonObject各通道内容模板（见下方示例）
variablesSchemaJsonArray变量定义：[{"name":"amount","type":"decimal","required":true}]
renderEngineString渲染引擎（白名单）：mustache / handlebars
changeDescString变更说明
channelContentJson 示例 { "push": { "title": "【追保预警】担保比降至{{ratio}}", "body": "请于{{deadline}}前补充{{amount}}元" }, "sms": { "body": "【XX证券】追保预警：担保比{{ratio}}，请于{{deadline}}前补充" }, "email": { "subject": "追保预警通知", "bodyHtml": "<p>尊敬的客户...</p>" }, "inbox": { "title": "追保预警", "body": "<p>...</p>" }, "voice": { "tts": "您好，您的账户担保比降至{{ratio}}..." } }
▼ ▼ ▼
POST /internal/v1/templates/{code}/versions/{ver}/approve 审批模板版本
交易/风控/合规类模板必须经合规团队审批。审批记录（审批人+时间）不可删除。
POST /internal/v1/templates/{code}/versions/{ver}/publish 发布模板版本
发布后即为 PUBLISHED 状态，新消息使用此版本渲染。旧版本自动 RETIRED。
▼ ▼ ▼
GET /internal/v1/templates/{templateCode}/versions/{version}?locale=zh-CN 查询模板版本
📌 状态流转：DRAFT → PENDING_APPROVAL → APPROVED → PUBLISHED → RETIRED。已发布版本不可修改，只能创建新版本。

📩 通道回执 & 回调 CALLBACK
Delivery receipt, open tracking, bounce & unsubscribe
📩 APNs/FCM/SMS 供应商通过 Webhook 回调通知送达结果。通知中心接收回执后更新通道任务状态，触发后续流程（已送达 → 停止降级 / 失败 → 触发降级）。所有回调必须验签+防重放+幂等。
POST /internal/v1/callbacks/{providerCode}/delivery 送达回执
供应商确认已送达用户设备。更新 task_status=DELIVERED，记录 delivered_at。
POST /internal/v1/callbacks/{providerCode}/open 打开/阅读回执
邮件追踪像素/短信阅读回执。更新 task_status=READ，记录 read_at。
▼ ▼ ▼
POST /internal/v1/callbacks/{providerCode}/bounce 失败/退信回执
Hard bounce → 标记地址无效 + 加入 suppression。Soft bounce → 可重试。
POST /internal/v1/callbacks/{providerCode}/unsubscribe 用户退订回调
邮件退订链接触发。记录 consent REVOKED + suppression ACTIVE。
🔒 安全要求：所有回调接口必须 1) 验签（供应商签名验证）2) 防重放（timestamp + nonce 校验）3) 幂等（provider_msg_id 去重）
