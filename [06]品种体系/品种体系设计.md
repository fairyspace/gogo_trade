# 品种体系设计 — 终版 v2.0

> **版本**：v2.0 Final（与项目时间配置 v2 / 杠杆配置 v2 / 路由网关 v2 / 行情系统 / 账户体系完成边界对齐）
> **范围**：FX / 贵金属 / 能源 / 股指 / CFD / 加密货币 / 美股 / A 股 / 期货（预留）/ 期权（预留）
> **定位**：品种体系是**品种主数据权威源**——只存品种内在属性（身份/合约规格/交易约束/隔夜利息公式/公司行动），不存已被其他定稿模块负责的杠杆值、交易时段、上游映射。
> **核心**：8 张 `t_` 前缀表，全部对齐已定稿 v2 模块的命名、字段类型、`deleted_at` 软删除、VARCHAR 枚举、CHECK 约束风格。

## 一、设计哲学与全球对标

品种体系是整个交易系统的"主数据底座"——账户、订单、行情、风控、清算都围绕"品种"运转。一切设计取舍围绕**单一权威源、边界不越界、规格可追溯、可演进**四个目标展开。

### 1.1 六条设计原则

1. **单一权威源（Single Source of Truth）**——品种身份、合约规格、交易约束只在品种体系定义一次，其他模块引用 `symbol_id`，不冗余副本。
2. **边界不越界**——已被时间配置 v2（交易时段）、杠杆配置 v2（保证金/杠杆）、路由网关 v2（上游映射）定稿接管的职责，品种体系**不重复定义**，只提供这些模块声明的依赖字段。
3. **规格可追溯（Spec Versioning）**——合约规格（digits/tick_size/contract_size）变更时，历史持仓必须能用开仓时的规格估值。规格快照 + 持仓引用，满足 MiFID II / SEC 17a-4 审计要求。
4. **变更全审计**——所有品种规格变更经应用层 AOP 写入审计表，与时间配置/杠杆/路由三大模块的审计机制对齐。
5. **分类树支撑路径匹配**——分类树形化 + 物化路径，直接支撑时间配置 v2 的 `path_pattern` glob 匹配与账户体系的 `symbol_category + symbol_pattern` 匹配。
6. **主数据与业务规则物理分离**——`t_symbol` 不耦合杠杆字段、不耦合时段字段、不耦合约定的上游代码；杠杆/时段/映射分别在各自模块独立演进。

### 1.2 全球对标与采纳决策

| # | 厂商 | 其核心能力 | 本设计采纳 / 取舍 |
|---|------|-----------|------------------|
| 1 | **MetaQuotes (MT4/MT5)** | Group 承载组级默认参数；Symbol Suffix 机制；合约规格快照；TRADE_CALC_MODE；point/pip 分离 | **主架构基线**：`symbol_key = code + suffix`；组级默认 + 品种覆盖（NULL 继承）；保留 `trade_calc_mode` 类型分类；显式 `point_size/pip_size` |
| 2 | **Spotware (cTrader)** | Symbol Category 树；Asset 多币种；Symbol Id 稳定 | 采纳**分类树形化**（parent_id + 物化路径）；不采纳多币种 Asset 钱包（账户体系已决策单账户单币种） |
| 3 | **Devexperts (DXtrade)** | 多资产；CFI/ISIN/FIGI 完整标识；期货合约生命周期 | 采纳 **FIGI（OpenFIGI）** 标识；采纳 ISIN/CFI；预留期货 `delivery_date/last_trade_date/contract_month` |
| 4 | **Apex Fintech** | 美股清算；公司行动（拆合股/分红/spin-off）；Corporate Action 全生命周期 | 采纳**公司行动类型补全**：SPLIT/REVERSE_SPLIT/CASH_DIVIDEND/STOCK_DIVIDEND/SPIN_OFF/MERGER/RIGHTS_ISSUE/RENAME/DELISTING |
| 5 | **DriveWealth** | API-First；碎股；品种状态机（仅平仓→禁用） | 采纳品种状态 `CLOSE_ONLY`（仅平仓）中间态；碎股由 `volume_step` 覆盖不单列 |
| 6 | **TradingView** | 多交易所；品种树展示；多语言 display_name | 采纳 `symbol_path` 导航路径与 `display_name` 展示分离 |
| 7 | **Trading Technologies (TT)** | 期货 FCM；合约月份；最后交易日；SPAN | 预留期货合约字段；SPAN 计算归杠杆模块，品种只标 `trade_calc_mode=FUTURES` |
| 8 | **Broadridge** | 公司行动事件引擎；Books & Records | 采纳公司行动**公告日/除权日/登记日/支付日**四阶段模型 |
| 9 | **Talos** | 机构加密；多场所；symbol 映射 | 上游映射归路由网关 `t_gateway_symbol_mapping`，品种只保留自身身份 |
| 10 | **恒生电子 (Hundsun)** | 证券主数据；公司行动处理；除权除息 | 采纳公司行动的除权除息价格调整逻辑 |

### 1.3 关键设计决策（显式声明取舍）

- **【决策一】trade_calc_mode 保留，margin 值字段全删。** `trade_calc_mode`（Forex/CFD/Stocks/Futures/Options/Exchange）是品种的**计价类型分类**（对标 MT5 SYMBOL_TRADE_CALC_MODE），决定合约价值如何折算，非"杠杆值"。杠杆模块的 `t_margin_requirement_rule.calc_mode`（FIXED_RATE/SPAN/PORTFOLIO_MARGIN）是**计算策略**。两者不同维度：品种标类型，杠杆模块定策略。`margin_currency` 保留（币种设置非杠杆值）。
- **【决策二】t_symbol_category 从扁平改为树形。** 时间配置 v2 的 `path_pattern` 依赖 `Stocks/US/NYSE/AAPL.US` 树形路径；账户体系用 `symbol_category + symbol_pattern` 匹配。扁平两层（category + group）无法表达。改为 `parent_id` 自引用 + `category_path` 物化路径 + `cfi_prefix`（ISO 10962 第一字符对齐）。
- **【决策三】规格快照按"变更触发"而非"每日全量"。** `t_symbol_spec_snapshot` 在规格关键字段变更时由应用层 AOP 自动写入，持仓记录 `spec_snapshot_id` 引用开仓时规格。避免每日全量快照膨胀；按 `effective_from` 区间查询。
- **【决策四】Swap 归品种，路由 markup_profile 可覆盖。** 杠杆 §2.2 明确"隔夜利息计算公式"归品种模块。`t_symbol_swap_config` 为 BASE 配置；路由网关 `t_markup_profile.swap_long/short` 为商业覆盖层。两级模型：品种定基准，路由按客户分层覆盖。
- **【决策五】交易时段表删除，委托时间配置 v2。** 时间配置已定稿更优的 session 表（UTC 绝对分钟 + DST profile + 版本 + 生效期 + 9 种递推）。品种体系改为向其**提供** `primary_exchange`(MIC)/`primary_timezone`(IANA)/`dst_profile` 三个依赖字段。

## 二、模块边界声明（核心）

品种体系**只做**以下 8 件事，其余职责已在其他定稿模块落地：

| 职责 | 是否品种体系 | 归属模块 | 边界说明 |
|------|:----------:|---------|---------|
| 品种身份（code/suffix/key/标识码） | ✅ | 本模块 | symbol_key 全局唯一，FIGI/ISIN/CFI 标识 |
| 合约规格（contract_size/digits/tick_size） | ✅ | 本模块 | 规格快照保障历史可追溯 |
| 品种分类树（category/group） | ✅ | 本模块 | 树形 path 支撑时间配置 path_pattern |
| 交易约束（手数/订单类型/距离/滑点） | ✅ | 本模块 | 品种级 ALLOWED，账户组可覆盖 |
| 隔夜利息公式（swap_mode/系数） | ✅ | 本模块 | BASE 配置，路由 markup 可覆盖 |
| 公司行动（拆合股/分红/spin-off） | ✅ | 本模块 | 事件表 + 四阶段时间 |
| 点差约束（spread_limit_min/max） | ✅ | 本模块 | 品种级约束上下限 |
| 报价处理参数（gap/filter/depth） | ✅ | 本模块 | 品种内在属性 |
| **交易时段** | ❌ | **时间配置 v2** | 品种提供 primary_exchange/timezone/dst_profile |
| **保证金/杠杆值** | ❌ | **杠杆配置 v2** | 品种只标 trade_calc_mode 类型 |
| **上游 LP 映射** | ❌ | **路由网关 v2** | t_gateway_symbol_mapping N:M |
| **时段点差 schedule** | ❌ | **行情系统** | t_symbol_spread_schedule |
| **商业加点/佣金** | ❌ | **路由网关 v2** | t_markup_profile |

### 2.1 对其他模块的依赖契约

**向时间配置 v2 提供**（时间配置 §3.5 声明）：

| 字段 | 类型 | 含义 |
|------|------|------|
| `primary_exchange` | VARCHAR(16) | 主交易所 MIC 码（ISO 10383），如 XNYS |
| `primary_timezone` | VARCHAR(64) | IANA TZ ID，如 America/New_York |
| `dst_profile` | VARCHAR(16) | DST 归属：NONE/USA/EUROPE/AUSTRALIA/CHILE/NZ |

**向杠杆配置 v2 提供**：`symbol_id`、`group_id`、`trade_calc_mode`、`contract_size`、`digits`、`margin_currency`。

**向路由网关 v2 提供**：`symbol_id`（`t_gateway_symbol_mapping` 关联）。

**向账户体系提供**：`symbol_category`（category_code）+ `symbol_pattern`，供 `t_account_group_symbol_config` 匹配。

## 四、通用约定

与杠杆配置 v2 / 路由网关 v2 / 时间配置 v2 完全一致：

| 项 | 约定 |
|---|------|
| 引擎 | InnoDB |
| 字符集 | utf8mb4 / utf8mb4_unicode_ci |
| 主键 | `id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '自增主键'` |
| 软删除 | `deleted_at BIGINT UNSIGNED NOT NULL DEFAULT 0`（0=正常，非0=删除时 Unix 毫秒时间戳） |
| 状态字段 | `status TINYINT UNSIGNED NOT NULL DEFAULT 1`（0-停用 1-启用） |
| 枚举字段 | `VARCHAR(N) + COMMENT` 描述取值，**不使用 ENUM 类型** |
| 时间戳 | `DATETIME(3)` + `DEFAULT CURRENT_TIMESTAMP(3)` + `ON UPDATE` |
| 唯一键 | `uk_<语义>`，复合 `deleted_at` 以支持软删除后重插同 code |
| 普通索引 | `idx_<字段组合>` |
| CHECK 约束 | `chk_<表缩写>_<语义>` |
| 金额/比率 | DECIMAL(18,4) 金额；DECIMAL(10,6) 比率；DECIMAL(18,8) 价格 |

后续各表 DDL 不再重复以上通用部分。

## 五、完整数据库设计

### 5.1 主数据层

#### 表 1：`t_symbol_category` — 品种分类树（树形化重构）

```sql
-- ----------------------------------------------------------------
-- 品种分类树
--
-- 从原稿扁平结构重构为树形, 支撑:
--   1. 时间配置 v2 的 path_pattern glob 匹配 (如 Stocks/US/** )
--   2. 账户体系的 symbol_category + symbol_pattern 匹配
--   3. 前端品种导航树展示
--
-- 树形设计:
--   parent_id 自引用, NULL=根节点(资产大类: Forex/Metals/Stocks/...)
--   category_path 物化路径, 便于前缀匹配: /Stocks/US/NYSE
--   cfi_prefix 对齐 ISO 10962 第一字符: E=Equity F=Future O=Option
--   is_leaf 标记叶子, 品种只能挂在叶子节点下
--
-- 示例树:
--   /Forex/Major            (leaf)
--   /Forex/Minor            (leaf)
--   /Stocks/US/NYSE         (leaf)
--   /Stocks/US/NASDAQ       (leaf)
--   /Stocks/CN/SH           (leaf)
--   /Crypto/Major           (leaf)
--   /Futures/CME            (leaf)
-- ----------------------------------------------------------------
CREATE TABLE `t_symbol_category` (
    `id`                BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `category_code`     VARCHAR(32)         NOT NULL
        COMMENT '节点编码: FOREX/STOCKS/US/NYSE/CRYPTO_MAJOR, 同级唯一',
    `category_name`     VARCHAR(128)        NOT NULL COMMENT '节点名称',
    `parent_id`         BIGINT UNSIGNED     DEFAULT NULL
        COMMENT '父节点ID, NULL=根节点(资产大类)',
    `category_path`     VARCHAR(512)        NOT NULL
        COMMENT '物化路径: /Forex/Major, /Stocks/US/NYSE; 便于前缀匹配与树展示',
    `cfi_prefix`        VARCHAR(8)          DEFAULT NULL
        COMMENT 'ISO 10962 CFI首字符对齐: E=Equity D=Debt F=Future O=Option R=Rights M=Misc C=Collective',
    `is_leaf`           TINYINT UNSIGNED    NOT NULL DEFAULT 0
        COMMENT '是否叶子节点: 0-否(仅作分类) 1-是(品种可挂此节点下)',
    `level_depth`       TINYINT UNSIGNED    NOT NULL DEFAULT 1
        COMMENT '层级深度: 1=根(资产大类) 2=市场/交易所 3=板块 4=细分',

    `sort_order`        INT                 NOT NULL DEFAULT 0 COMMENT '同级排序',
    `description`       VARCHAR(256)        DEFAULT NULL COMMENT '节点描述',

    `status`            TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `created_at`        DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',
    `updated_at`        DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',
    `deleted_at`        BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳: 0-正常, 非0-删除时Unix毫秒时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_parent_code` (`parent_id`, `category_code`, `deleted_at`),
    UNIQUE KEY `uk_category_path` (`category_path`, `deleted_at`),
    KEY `idx_parent_status` (`parent_id`, `status`),
    KEY `idx_leaf` (`is_leaf`, `status`),
    KEY `idx_cfi_prefix` (`cfi_prefix`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='品种分类树';
```

> **设计要点**：
> - `category_code` 在同级（同 parent_id）唯一，全局路径 `category_path` 唯一。
> - 品种组（表 2）挂在叶子节点上；品种（表 3）通过 `group_id` 间接归属分类树。
> - 删除节点须先校验无子节点、无品种组引用（应用层保证）。

#### 表 2：`t_symbol_group` — 品种组（精简）

```sql
-- ----------------------------------------------------------------
-- 品种组
--
-- 同类品种逻辑分组, 承载组级默认参数, 品种可覆盖(NULL=继承组)
-- 对标 MT5 Group / cTrader Symbol Group
--
-- 相对原稿变更:
--   - 删除 default_margin_mode (保证金归杠杆配置v2)
--   - 保留 default_digits/default_swap_mode/default_execution_mode/default_filling_mode
--   - triple_swap_day 保留(组级默认, 品种swap_config可覆盖)
--   - 新增 category_id 关联分类树叶子节点
-- ----------------------------------------------------------------
CREATE TABLE `t_symbol_group` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `group_code`                VARCHAR(32)         NOT NULL COMMENT '组代码: FOREX_MAJOR/CRYPTO_MAJOR/STOCKS_US',
    `group_name`                VARCHAR(128)        NOT NULL COMMENT '组名称',
    `category_id`               BIGINT UNSIGNED     NOT NULL COMMENT '所属分类树叶子节点ID',

    -- ==================== 组级默认参数(品种可覆盖) ====================
    `default_contract_size`     DECIMAL(18,4)       NOT NULL DEFAULT 100000
        COMMENT '默认合约大小, 外汇通常100000, 股票通常1',
    `default_digits`            TINYINT UNSIGNED    NOT NULL DEFAULT 5
        COMMENT '默认价格小数位数',
    `default_swap_mode`         TINYINT UNSIGNED    NOT NULL DEFAULT 1
        COMMENT '默认隔夜利息模式: 0-禁用 1-按点数 2-按金额 3-按年化利率% 4-按保证金比例% 5-按面值比例% 6-重开',
    `default_execution_mode`    TINYINT UNSIGNED    NOT NULL DEFAULT 1
        COMMENT '默认成交模式: 1-Instant 2-Market 3-Exchange 4-Request',
    `default_filling_mode`      TINYINT UNSIGNED    NOT NULL DEFAULT 1
        COMMENT '默认填充模式: 1-FOK 2-IOC 3-Return',

    `triple_swap_day`           TINYINT UNSIGNED    NOT NULL DEFAULT 3
        COMMENT '三倍隔夜利息的星期几: 1-周一...7-周日, 外汇通常3(周三)',

    `sort_order`                INT                 NOT NULL DEFAULT 0 COMMENT '排序',
    `description`               VARCHAR(256)        DEFAULT NULL COMMENT '描述',

    `status`                    TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '状态: 0-停用 1-启用',
    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',
    `updated_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',
    `deleted_at`                BIGINT UNSIGNED     NOT NULL DEFAULT 0 COMMENT '删除时间戳',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_group_code` (`group_code`, `deleted_at`),
    KEY `idx_category_status` (`category_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='品种组';
```

#### 表 3：`t_symbol` — 品种定义（核心重构）

```sql
-- ----------------------------------------------------------------
-- 品种定义 (核心表)
--
-- 每个可交易品种的身份 + 合约规格 + 报价参数
-- 交易参数/Swap/公司行动 在独立子表
--
-- 相对原稿变更:
--   [删除-归杠杆配置v2] margin_mode(改trade_calc_mode) / margin_initial / margin_maintenance
--                       / margin_hedged / margin_rate_market / margin_rate_limit / margin_rate_stop
--                       / margin_rate_stop_limit / margin_rate_liquidity / margin_rate_currency
--   [删除-归路由网关v2] source_symbol / source_gateway_id / quote_delay_ms
--   [新增-时间配置依赖] primary_exchange / primary_timezone / dst_profile
--   [新增-标识码] figi (OpenFIGI)
--   [新增-语义明确] point_size / pip_size / trade_calc_mode
--   [新增-期货预留] contract_month / delivery_date / last_trade_date
--   [保留] 合约规格/货币/点差约束/报价处理/展示/状态
--
-- 命名约定:
--   symbol_code  = 标准代码 (EURUSD)
--   symbol_suffix= 后缀 (m/e), MT5同基础品种不同后缀表示不同账户组版本
--   symbol_key   = code+suffix 完整唯一标识
--   figi         = OpenFIGI 全球金融工具标识 (机构标配)
--   isin         = 股票国际证券识别码
--   cfi          = ISO 10962 金融工具分类码
-- ----------------------------------------------------------------
CREATE TABLE `t_symbol` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `symbol_code`               VARCHAR(32)         NOT NULL COMMENT '品种代码: EURUSD, XAUUSD, BTCUSD, AAPL',
    `symbol_suffix`             VARCHAR(8)          DEFAULT NULL COMMENT '品种后缀: m(微型)/e(ECN)',
    `symbol_key`                VARCHAR(40)         NOT NULL COMMENT '完整唯一标识: symbol_code+suffix',
    `group_id`                  BIGINT UNSIGNED     NOT NULL COMMENT '所属品种组ID',

    -- ==================== 基本信息 ====================

> **设计要点**：
> - `figi` 加唯一键：OpenFIGI 是机构标配的唯一标识，同品种不同后缀共享同一 FIGI（后缀是商业版本，非不同标的）。实际若后缀品种需独立 FIGI，运营保证唯一即可。
> - `trade_calc_mode` 仅作类型分类，保证金计算策略由杠杆模块 `t_margin_requirement_rule.calc_mode` 决定。
> - `point_size` 与 `tick_size` 通常相等，但部分品种 tick_size 可能是 point_size 的倍数（如某些期货），故分开存储。

CREATE TABLE `t_symbol_spec_snapshot` (
    `id`                        BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `symbol_id`                 BIGINT UNSIGNED     NOT NULL COMMENT '关联品种ID',
    `snapshot_version`          INT UNSIGNED        NOT NULL COMMENT '快照版本号, 同symbol递增',

    -- ==================== 快照的规格字段 (变更触发快照的关键字段) ====================
    `contract_size`             DECIMAL(18,4)       NOT NULL COMMENT '快照时合约大小',
    `digits`                    TINYINT UNSIGNED    NOT NULL COMMENT '快照时价格小数位',
    `point_size`                DECIMAL(18,8)       NOT NULL COMMENT '快照时最小报价单位',
    `tick_size`                 DECIMAL(18,8)       NOT NULL COMMENT '快照时最小价格变动',
    `tick_value`                DECIMAL(18,8)       NOT NULL COMMENT '快照时tick价值',
    `trade_calc_mode`           TINYINT UNSIGNED    NOT NULL COMMENT '快照时计价类型',
    `base_currency`             VARCHAR(8)          NOT NULL COMMENT '快照时基准货币',
    `profit_currency`           VARCHAR(8)          NOT NULL COMMENT '快照时盈亏货币',
    `margin_currency`           VARCHAR(8)          NOT NULL COMMENT '快照时保证金货币',
    `volume_min`                DECIMAL(18,4)       NOT NULL COMMENT '快照时最小手数',
    `volume_max`                DECIMAL(18,4)       NOT NULL COMMENT '快照时最大手数',
    `volume_step`               DECIMAL(18,4)       NOT NULL COMMENT '快照时手数步进',

    -- ==================== 生效区间 ====================
    `effective_from`            DATETIME(3)         NOT NULL COMMENT '本快照生效起始时间',
    `effective_to`              DATETIME(3)         DEFAULT NULL COMMENT '本快照失效时间(NULL=当前最新)',

    -- ==================== 触发信息 ====================
    `trigger_reason`            VARCHAR(256)        DEFAULT NULL COMMENT '触发原因: 公司行动/规格调整/监管要求',
    `trigger_action_id`         BIGINT UNSIGNED     DEFAULT NULL COMMENT '触发的公司行动ID(若由公司行动触发)',

    `created_at`                DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_symbol_version` (`symbol_id`, `snapshot_version`),
    KEY `idx_symbol_effective` (`symbol_id`, `effective_from`, `effective_to`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='合约规格版本快照';
```

> **设计要点**：
> - 快照表不软删除（历史规格不可删除，审计要求）。
> - 新快照写入时，前一条快照的 `effective_to` 设为新快照的 `effective_from`。
> - 持仓表新增 `spec_snapshot_id` 字段（由交易系统/持仓表维护），引用开仓时的规格快照。
> - 历史持仓估值时：`SELECT * FROM t_symbol_spec_snapshot WHERE id = :spec_snapshot_id`，O(1) 主键点查。

#### 表 8：`t_symbol_change_log` — 品种变更审计（新增，按月分区）

```sql

--
-- 所有品种配置变更经应用层AOP统一拦截写入, 满足 MiFID II / SEC 17a-4 审计要求

--   - 不存 full_snapshot 大字段 (避免表膨胀)

    `new_value`                 VARCHAR(512)        DEFAULT NULL COMMENT '新值字符串',
    `change_payload`            JSON                DEFAULT NULL COMMENT '完整变更负载(多字段时): {"old":{...},"new":{...}}',

    `trace_id`                  VARCHAR(64)         DEFAULT NULL COMMENT '链路追踪ID',
    `remark`                    VARCHAR(512)        DEFAULT NULL COMMENT '备注',

    PRIMARY KEY (`id`, `created_at`),
    KEY `idx_change_type_at` (`change_type`, `created_at`),
    KEY `idx_target` (`target_table`, `target_id`),
    KEY `idx_symbol_at` (`symbol_id`, `created_at`),
    KEY `idx_symbol_key_at` (`symbol_key`, `created_at`),

PARTITION BY RANGE (TO_DAYS(`created_at`)) (

);
```

## 六、双人审批清单

以下变更强制要求 `approved_by IS NOT NULL`，由应用层校验：

| 表 | 变更类型 | 理由 |
|----|----------|------|
| `t_symbol` | 修改 digits/tick_size/contract_size/tick_value | 影响所有持仓估值，触发规格快照 |
| `t_symbol` | 修改 trade_calc_mode | 影响保证金计算口径 |
| `t_symbol` | 状态从 1→0 或 1→2(仅平仓) | 影响客户交易能力 |
| `t_symbol_corporate_action` | 创建/执行任意行动 | 直接影响持仓与资金 |
| `t_symbol` | 删除品种（软删除） | 须先确认无持仓无挂单 |

-- 撮合引擎启动时全量加载, 合并为 SymbolConfig{} 内存对象

### 8.2 查询品种的当前交易时段状态（委托时间配置）

```sql
-- 委托时间配置v2: 直接查 t_daily_session 快照
SELECT `trading_date`, `seq_no`, `session_type`, `open_min`, `close_min`,
       `state`, `runtime_state`
FROM `t_daily_session`
WHERE `symbol_id` = :symbol_id
  AND `trading_date` = :today
  AND `state` IN ('SCHEDULED', 'ACTIVE')
ORDER BY `seq_no`;
```

### 8.3 历史持仓估值时取开仓规格

```sql
-- 持仓表记录 spec_snapshot_id, O(1) 主键点查
SELECT `contract_size`, `digits`, `point_size`, `tick_size`, `tick_value`,
       `trade_calc_mode`, `base_currency`, `profit_currency`, `margin_currency`
FROM `t_symbol_spec_snapshot`
WHERE `id` = :spec_snapshot_id;
```

### 8.4 按分类路径树查询品种（支撑时间配置 path_pattern）

```sql
-- 查 Stocks/US 下所有品种 (前缀匹配)
SELECT s.`id`, s.`symbol_key`, s.`display_name`
FROM `t_symbol` s
JOIN `t_symbol_group` sg ON s.`group_id` = sg.`id`
JOIN `t_symbol_category` sc ON sg.`category_id` = sc.`id`
WHERE sc.`category_path` LIKE '/Stocks/US/%'
  AND s.`status` = 1 AND s.`deleted_at` = 0;
```

### 8.5 待执行的公司行动扫描

```sql
-- 定时任务每日扫描: 今日除权除息日的待执行行动
SELECT ca.`id`, ca.`symbol_id`, s.`symbol_key`, ca.`action_type`,
       ca.`ratio_numerator`, ca.`ratio_denominator`,
       ca.`dividend_per_share`, ca.`dividend_currency`,
       ca.`resulting_symbol_id`, ca.`exchange_ratio`
FROM `t_symbol_corporate_action` ca
JOIN `t_symbol` s ON ca.`symbol_id` = s.`id`
WHERE ca.`ex_date` = CURDATE()
  AND ca.`status` = 0
ORDER BY ca.`symbol_id`;
```

### 8.6 品种规格变更追踪（合规审计）

```sql
-- 查某品种近30天所有变更
SELECT `change_type`, `operation`, `target_table`, `field_name`,
       `old_value`, `new_value`, `change_reason`, `change_source`,
       `operator_name`, `approved_by`, `ip_address`, `created_at`
FROM `t_symbol_change_log`
WHERE `symbol_id` = :symbol_id
  AND `created_at` >= DATE_SUB(NOW(), INTERVAL 30 DAY)
ORDER BY `created_at` DESC
LIMIT 200;
```

## 十一、与其他模块的集成契约

### 11.1 向时间配置 v2 提供（时间配置 §3.5）

时间配置预计算任务读取 `t_symbol` 的 `primary_exchange`、`primary_timezone`、`dst_profile`，用于：
1. 运营录入交易所公告时的本地时间 → UTC 转换辅助
2. DST 自动版本生成
3. 前端按客户时区展示

时间配置的 `t_symbol_session`、`t_daily_session`、`t_session_override` 通过 `symbol_id` 关联本模块 `t_symbol`，通过 `path_pattern`（glob）匹配本模块 `t_symbol_category.category_path`。

### 11.2 向杠杆配置 v2 提供

杠杆模块 8 级解析链读取 `t_symbol.group_id`（→ `t_symbol_group` → `t_symbol_category`）定位品种组/分类，读取 `t_symbol.trade_calc_mode`/`contract_size`/`digits`/`margin_currency` 辅助保证金计算。品种**不提供**杠杆值。

### 11.3 向路由网关 v2 提供

路由网关 `t_gateway_symbol_mapping` 通过 `symbol_id` 关联本模块，建立上游 LP 代码与本系统品种的 N:M 映射。品种的 `contract_size`/`digits`/`tick_size` 用于价格/手数转换基准。

### 11.4 向行情系统提供

行情系统读取 `t_symbol.spread_limit_min/max`（点差约束）、`filter_bad_ticks`（异常过滤）、`gap_mode/threshold`（跳空控制）、`allow_negative_price`、`market_depth`、`chart_mode`。行情系统的 `t_symbol_spread_schedule` 在品种点差约束之上叠加时段点差。

### 11.5 向账户体系提供

账户体系 `t_account_group_symbol_config` 用 `symbol_category`（category_code）+ `symbol_pattern` 匹配品种。品种组/分类树的 `category_path` 提供树形匹配能力。

## 结语

本设计在已定稿的五大模块（时间配置 v2 / 杠杆配置 v2 / 路由网关 v2 / 行情系统 / 账户体系）边界约束下，精准切割了品种体系的职责范围——**只做品种主数据权威源**，不越界重复定义时段、保证金、上游映射。通过规格快照与变更审计补齐了全球券商标准的可追溯能力，通过分类树形化统一支撑了各模块的路径匹配需求。

> **下一步可演进方向**：当业务发展到需要"组合保证金跨品种对冲折扣"精细化时，品种体系可在 `t_symbol_group` 新增 `cross_margin_group_code` 字段，配合杠杆模块的 `t_margin_requirement_rule.cross_margin_group_code` 实现组合保证金分组，无需新增表。

### B.4 期货连续合约/展期/主力切换（补缺）

旧稿 `t_symbol` 含 `contract_month`/`delivery_date` 预留字段但无流程。v2.1 补：

- **连续合约**（Continuous Symbol）：如 `CL` 而非 `CLZ6`，映射到当前主力合约。

### B.5 公司行动执行 Saga（补缺）

旧稿仅有文字逻辑。v2.1 升级为 Saga：

```
公司行动 scheduled_at 到达
  → 扫描 t_symbol_corporate_action WHERE status=PENDING AND effective_at <= now
  → Step1: 锁定相关持仓（标记 processing）
  → Step2: 调整持仓/价格（SPLIT/REVERSE_SPLIT）
       或 调资金域 CreditAccount（CASH_DIVIDEND）
       或 强制平仓（DELISTING）
  → Step3: 写新 spec_snapshot_id
  → Step4: 发 symbol.corporate_action_executed.v1（交易/资金/行情消费）
  → Step5: 解锁持仓，标记 status=EXECUTED
  → 失败补偿：任一 Step 失败 → 回滚 + 告警 + 人工
```

- DELISTING 异常：有未平持仓/挂单/期权链 → 强制平仓优先级：期权链 → 挂单 → 持仓；通知运营介入。
- 与行情联动：除权后历史 K 线复权由行情域消费 `symbol.corporate_action_executed.v1` 重算。
