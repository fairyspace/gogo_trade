### 1.1 六大核心职能

```
系统职能地图

├── 职能①：金融新闻聚合与分发
│   ├── 多源采集    → 财联社/新华社/路透社/彭博/SEC/交易所公告
│   ├── 内容加工    → NLP提取、摘要生成、情感分析、关联标的
│   ├── 个性推送    → 基于持仓+关注+偏好的个性化分发
│   └── 实时性      → 重大新闻秒级触达，财报30秒内解读推送
│
├── 职能②：公告与财报系统
│   ├── 财报采集    → 季报/年报/半年报 结构化解析
│   ├── 公告监控    → 停牌/复牌/重组/分红 实时监控
│   ├── 数据对比    → 历史同期对比、行业对比
│   └── AI解读      → 关键指标自动提取与解读
│
├── 职能③：论坛社区
│   ├── 个股论坛    → 每个标的独立讨论区
│   ├── 话题广场    → 热门话题、主题讨论
│   ├── 精华帖      → 优质内容沉淀
│   └── 投资日记    → 个人持仓记录与分享
│
├── 职能④：IM即时通讯
│   ├── 私信        → 用户1对1通讯
│   ├── 群聊        → 话题群/兴趣群/私募群
│   ├── 频道        → KOL一对多广播
│   └── 投顾服务    → 投顾与客户专属沟通
│
├── 职能⑤：内容生态运营
│   ├── KOL体系     → 认证分析师/博主体系
│   ├── 积分激励    → 内容贡献激励
│   ├── 内容审核    → AI+人工合规审核
│   └── 热榜系统    → 实时热度计算
│
└── 职能⑥：信息与交易联动
    ├── 新闻→行情   → 新闻关联实时行情
    ├── 新闻→交易   → 一键从新闻跳转下单
    ├── 讨论→标的   → 帖子关联标的行情
    └── 情绪指数    → 社区情绪与市场联动
```

### 2.2 NLP内容处理流水线

```
内容处理 Pipeline

原始内容输入
     │
     ▼
┌─────────────────────────────────────────────────────────┐
│ Step1: 基础清洗                                          │
│   去HTML标签 / 去广告 / 编码统一 / 语言检测              │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ Step2: NLP理解                                           │
│   ├── 分词 & 词性标注  (jieba / HanLP)                  │
│   ├── 命名实体识别NER  → 公司名/人名/地名/机构           │
│   ├── 关键词提取       → TF-IDF / TextRank              │
│   └── 自动摘要         → extractive / abstractive       │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ Step3: 金融专项理解                                       │
│   ├── 标的关联         → 股票代码/基金代码识别与关联      │
│   ├── 事件分类         → 财报/并购/分红/政策/宏观        │
│   ├── 情感分析         → 正面/负面/中性 + 情感强度       │
│   └── 影响评估         → 短期/长期影响预判               │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ Step4: 内容标签化                                         │
│   ├── 行业标签         → 科技/金融/医疗/能源...          │
│   ├── 主题标签         → 财报季/降息/通胀/贸易...        │
│   ├── 重要程度         → 1-5级(影响面+新鲜度)           │
│   └── 时效性标记       → 实时/短期/长期                  │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ Step5: 合规审核                                           │
│   ├── 违禁词过滤                                         │

└──────────────────────────┬──────────────────────────────┘
                           ▼
                      写入存储 & 建立索引
```

## 三、完整表结构设计

### 3.1 新闻内容域

```sql
-- ① 新闻来源配置表
CREATE TABLE `news_source` (
  `source_id`       BIGINT       NOT NULL AUTO_INCREMENT,
  `source_code`     VARCHAR(64)  NOT NULL COMMENT '来源编码',
  `source_name`     VARCHAR(128) NOT NULL COMMENT '来源名称',
  `source_type`     VARCHAR(32)  COMMENT '类型 rss/api/crawler/exchange/sec',
  `source_url`      VARCHAR(512) COMMENT '来源URL',
  `source_logo`     VARCHAR(512) COMMENT 'Logo URL',
  `credibility`     TINYINT      DEFAULT 3 COMMENT '可信度 1-5',
  `authority_level` TINYINT      COMMENT '权威等级 1-官方 2-主流媒体 3-自媒体',
  `crawl_interval`  INT          DEFAULT 60 COMMENT '抓取间隔(秒)',
  `crawl_config`    JSON         COMMENT '抓取配置(选择器/接口参数)',
  `language`        VARCHAR(8)   DEFAULT 'zh' COMMENT '语言',
  `country`         VARCHAR(16)  COMMENT '国家/地区',
  `is_enabled`      TINYINT      DEFAULT 1,
  `last_crawl_time` DATETIME     COMMENT '最近抓取时间',
  `create_time`     DATETIME     DEFAULT CURRENT_TIMESTAMP,
  `update_time`     DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`source_id`),
  UNIQUE KEY `uk_source_code` (`source_code`)
) ENGINE=InnoDB COMMENT='新闻来源配置表';


-- ② 新闻文章主表 (元数据存MySQL，正文存MongoDB)
CREATE TABLE `news_article` (
  `article_id`       BIGINT       NOT NULL AUTO_INCREMENT,
  `article_uuid`     VARCHAR(64)  NOT NULL COMMENT '全局唯一ID',
  `source_id`        BIGINT       NOT NULL COMMENT '来源ID',
  `source_url`       VARCHAR(1024) COMMENT '原文URL',
  `url_hash`         VARCHAR(64)  NOT NULL COMMENT 'URL Hash(去重)',
  `title`            VARCHAR(512) NOT NULL COMMENT '标题',
  `summary`          VARCHAR(1024) COMMENT 'AI生成摘要',
  `author`           VARCHAR(128) COMMENT '作者',
  `category`         VARCHAR(32)  COMMENT '分类 macro/company/industry/policy/global',
  `article_type`     VARCHAR(32)  COMMENT '类型 news/report/announcement/analysis',
  `importance`       TINYINT      DEFAULT 3 COMMENT '重要程度 1-5',
  `language`         VARCHAR(8)   DEFAULT 'zh',
  `publish_time`     DATETIME     NOT NULL COMMENT '发布时间',
  `crawl_time`       DATETIME     COMMENT '抓取时间',
  `process_time`     DATETIME     COMMENT 'NLP处理时间',
  -- NLP处理结果
  `keywords`         JSON         COMMENT '关键词列表',
  `entities`         JSON         COMMENT '实体识别结果[{type,name,confidence}]',
  `sentiment`        TINYINT      COMMENT '情感倾向 1-负面 2-中性 3-正面',
  `sentiment_score`  DECIMAL(5,4) COMMENT '情感得分 -1到1',
  `event_type`       VARCHAR(64)  COMMENT '事件类型 earnings/merger/dividend/policy',
  `impact_level`     TINYINT      COMMENT '影响级别 1-5',
  `impact_duration`  VARCHAR(16)  COMMENT '影响时效 short/medium/long',
  -- 内容状态
  `cover_image`      VARCHAR(512) COMMENT '封面图URL',
  `content_hash`     VARCHAR(64)  COMMENT '内容Hash(去重)',
  `word_count`       INT          COMMENT '字数',
  `read_time`        INT          COMMENT '预估阅读时长(秒)',
  `process_status`   TINYINT      DEFAULT 0 COMMENT '处理状态 0-待处理 1-处理中 2-完成 3-失败',
  `audit_status`     TINYINT      DEFAULT 1 COMMENT '审核状态 0-待审 1-通过 2-拒绝',
  `is_top`           TINYINT      DEFAULT 0 COMMENT '是否置顶',
  `is_breaking`      TINYINT      DEFAULT 0 COMMENT '是否突发新闻',
  -- 统计数据
  `view_count`       BIGINT       DEFAULT 0 COMMENT '阅读数',
  `like_count`       INT          DEFAULT 0 COMMENT '点赞数',
  `comment_count`    INT          DEFAULT 0 COMMENT '评论数',
  `share_count`      INT          DEFAULT 0 COMMENT '分享数',
  `collect_count`    INT          DEFAULT 0 COMMENT '收藏数',
  `hot_score`        DECIMAL(12,4) DEFAULT 0 COMMENT '热度分',
  `create_time`      DATETIME     DEFAULT CURRENT_TIMESTAMP,
  `update_time`      DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`article_id`),
  UNIQUE KEY `uk_url_hash` (`url_hash`),
  UNIQUE KEY `uk_article_uuid` (`article_uuid`),
  KEY `idx_publish_time` (`publish_time`),
  KEY `idx_source_id` (`source_id`),
  KEY `idx_category` (`category`, `publish_time`),
  KEY `idx_importance` (`importance`, `publish_time`),
  KEY `idx_hot_score` (`hot_score`)
) ENGINE=InnoDB COMMENT='新闻文章主表';


-- ③ 新闻-标的关联表
CREATE TABLE `news_symbol_relation` (
  `id`              BIGINT       NOT NULL AUTO_INCREMENT,
  `article_id`      BIGINT       NOT NULL COMMENT '文章ID',
  `symbol_code`     VARCHAR(32)  NOT NULL COMMENT '标的代码(股票/基金/期货)',
  `symbol_name`     VARCHAR(128) COMMENT '标的名称',
  `symbol_type`     VARCHAR(16)  COMMENT '标的类型 stock/fund/index/futures',
  `market`          VARCHAR(16)  COMMENT '市场 SH/SZ/HK/US',
  `relevance`       DECIMAL(5,4) COMMENT '相关度 0-1',
  `mention_count`   INT          DEFAULT 1 COMMENT '文章中提及次数',
  `is_primary`      TINYINT      DEFAULT 0 COMMENT '是否主要关联标的',
  `sentiment`       TINYINT      COMMENT '对该标的情感 1-负 2-中 3-正',
  `create_time`     DATETIME     DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_article_id` (`article_id`),
  KEY `idx_symbol_code` (`symbol_code`, `create_time`),
  KEY `idx_symbol_time` (`symbol_code`, `is_primary`)
) ENGINE=InnoDB COMMENT='新闻-标的关联表';


-- ④ 财报/公告表
CREATE TABLE `news_announcement` (
  `announce_id`     BIGINT       NOT NULL AUTO_INCREMENT,
  `announce_uuid`   VARCHAR(64)  NOT NULL,
  `symbol_code`     VARCHAR(32)  NOT NULL COMMENT '标的代码',
  `symbol_name`     VARCHAR(128) COMMENT '标的名称',
  `market`          VARCHAR(16)  COMMENT '市场',
  `announce_type`   VARCHAR(32)  COMMENT '公告类型 annual_report/quarterly/semi_annual/
                                          dividend/suspend/resume/merger/rights_issue',
  `title`           VARCHAR(512) NOT NULL COMMENT '公告标题',
  `announce_date`   DATE         COMMENT '公告日期',
  `report_period`   VARCHAR(16)  COMMENT '报告期 2024Q1/2024H1/2023FY',
  `source_url`      VARCHAR(1024) COMMENT '原文URL',
  `file_url`        VARCHAR(1024) COMMENT 'PDF文件URL',
  -- 财报核心数据(结构化)
  `revenue`         DECIMAL(20,4) COMMENT '营收(元)',
  `revenue_yoy`     DECIMAL(10,4) COMMENT '营收同比增长率',
  `net_profit`      DECIMAL(20,4) COMMENT '净利润',
  `net_profit_yoy`  DECIMAL(10,4) COMMENT '净利润同比',
  `gross_margin`    DECIMAL(8,4)  COMMENT '毛利率',
  `eps`             DECIMAL(10,4) COMMENT '每股收益',
  `roe`             DECIMAL(8,4)  COMMENT '净资产收益率',
  `debt_ratio`      DECIMAL(8,4)  COMMENT '资产负债率',
  `operating_cf`    DECIMAL(20,4) COMMENT '经营现金流',
  -- AI解读
  `ai_summary`      TEXT          COMMENT 'AI生成财报摘要',
  `ai_highlights`   JSON          COMMENT 'AI提取的亮点列表',
  `ai_risks`        JSON          COMMENT 'AI提取的风险点',
  `ai_rating`       VARCHAR(16)   COMMENT 'AI评级 beat/meet/miss',
  `sentiment`       TINYINT       COMMENT '整体情感',
  `importance`      TINYINT       COMMENT '重要程度 1-5',
  `audit_status`    TINYINT       DEFAULT 1,
  `view_count`      BIGINT        DEFAULT 0,
  `create_time`     DATETIME      DEFAULT CURRENT_TIMESTAMP,
  `update_time`     DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`announce_id`),
  UNIQUE KEY `uk_announce_uuid` (`announce_uuid`),
  KEY `idx_symbol_code` (`symbol_code`, `announce_date`),
  KEY `idx_announce_type` (`announce_type`, `announce_date`),
  KEY `idx_announce_date` (`announce_date`)
) ENGINE=InnoDB COMMENT='财报公告表';


-- ⑤ 标的情绪指数表 (按天计算)
CREATE TABLE `news_symbol_sentiment_daily` (

  -- 新闻情绪
  `news_count`          INT          DEFAULT 0 COMMENT '当日相关新闻数',

  `news_neutral`        INT          DEFAULT 0 COMMENT '中性新闻数',
  `news_sentiment_score` DECIMAL(8,4) COMMENT '新闻情感得分 -1到1',
  -- 社区情绪
  `post_count`          INT          DEFAULT 0 COMMENT '社区帖子数',
  `post_positive`       INT          DEFAULT 0 COMMENT '看涨帖子数',
  `post_negative`       INT          DEFAULT 0 COMMENT '看跌帖子数',
  `community_sentiment` DECIMAL(8,4) COMMENT '社区情感得分',
  `bull_bear_ratio`     DECIMAL(8,4) COMMENT '多空比',
  -- 综合情绪
  `composite_sentiment` DECIMAL(8,4) COMMENT '综合情绪得分',
  `sentiment_level`     TINYINT      COMMENT '情绪等级 1-极度悲观 2-悲观 3-中性 4-乐观 5-极度乐观',
  `attention_score`     DECIMAL(12,4) COMMENT '关注热度得分',
  `hot_rank`            INT          COMMENT '热度排名',
  -- 行情数据(冗余方便联合分析)
  `close_price`         DECIMAL(12,4) COMMENT '收盘价',
  `price_change_rate`   DECIMAL(8,4)  COMMENT '涨跌幅',
  `turnover_rate`       DECIMAL(8,4)  COMMENT '换手率',
  `create_time`         DATETIME      DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_symbol_date` (`symbol_code`, `stat_date`),
  KEY `idx_stat_date` (`stat_date`),
  KEY `idx_hot_rank` (`stat_date`, `hot_rank`)
```

## 四、核心算法设计

### 4.1 热度分计算模型

```
热度分计算公式 (Reddit + 时间衰减 改良版)

                 互动分 × 权重因子
热度分 = ──────────────────────────────────── × 内容质量系数
          (发布时间距今小时数 + 2) ^ 衰减系数


互动分计算：
┌────────────────────────────────────────────────────┐
│  action        weight   说明                        │
│  ──────────────────────────────────────────────── │
│  view          0.1      浏览                        │
│  like          3        点赞                        │
│  comment       5        评论                        │
│  share         8        分享(高意图)                │
│  collect       4        收藏                        │
│  follow_from   10       因此内容涨粉                │
│  report        -10      被举报                      │
└────────────────────────────────────────────────────┘

权重因子 (根据来源调整)：
  官方媒体   → 1.5
  认证分析师 → 1.3
  普通用户   → 1.0
  历史低质用户 → 0.5

衰减系数：
  新闻类    → 1.8  (快速衰减)
  财报类    → 1.2  (慢速衰减)
  社区帖子  → 1.5  (中速衰减)

内容质量系数：
  优质标签  → 1.2
  精华帖    → 1.5
  违规历史  → 0.3
```
