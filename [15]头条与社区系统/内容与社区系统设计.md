# 内容与社区系统设计（权威版）

> **版本**：v1.0 Final  
> **定位**：经纪平台的**资讯/公告/投教内容权威** + **社群 IM（大厅/广场/群聊）权威**。  
> **对标**：App 资讯流、投教中心、交易社群聊天（能力级）。  
> **运行时分层**：Content CMS · IM Service（会话/消息/审核）；推送复用通知域。**方向设计不锁定实现语言。**  
> **域地图**：见 [`[总览]经纪平台域地图.md`](../[总览]经纪平台域地图.md)。

---

## 〇、为何独立成域

| 本域 | 不放这里 |
|------|----------|
| 新闻头条、公告、投教 CMS | CRM 任务 / 赠金活动规则（营销与推荐） |
| 群聊大厅、广场频道、私聊 | 跟单订阅 / 交易排行榜（社交交易） |
| 消息审核、晒单授权卡片 | 通知通道实现（通知域只送达） |

「社区」在本域指 **聊天与内容社区**，不是跟单「社交交易」。

---

## 一、系统定位

1. **内容 CMS**：`NEWS` / `ANNOUNCEMENT` / `EDUCATION` 等，版本化与审批发布  
2. **社群 IM**：大厅/频道/群/私聊、成员角色、禁言、举报审核、系统卡片  
3. **外链 IM**（可选）：Telegram 等绑定与合规披露  

**不属于本系统**：

| 概念 | 归属 |
|------|------|
| 营销活动规则、发奖、CRM Desk | 营销与推荐 [14] |
| 跟单、排行榜、信号放大 | 社交交易 [13] |
| 推荐「推哪篇文章」的策略 | 营销与推荐（可引用本域 content_id） |
| 多通道 Push/短信送达 | 通知 [3] |
| 成交写权威 | 交易引擎 |

---

## 二、十二条不可破坏决策

1. **`entity_id` 贯穿**内容可见性与群；禁止跨实体拉群。  
2. **内容版本化 + 审批**后方可 `PUBLISHED`。  
3. **IM 禁止自动下单**：聊天不得变为 `SubmitOrder`。  
4. **晒单须授权**：交易只读脱敏 API + 用户授权 → `SHARE_CARD`。  
5. **跟单仅深链**：跳转社交交易落地页；订阅不在本域。  
6. **诱导交易/喊单**：审核队列 + 关键词/举报。  
7. **推送不拥有正文**：`content.published` 触发通知；正文权威在本域。  
8. **事件经 Outbox**。  
9. **PII/留痕**：消息可审计保留（按时限）。  
10. **外链模式**：可不自研消息栈，绑定配置仍属本域。  
11. **与营销解耦**：活动文案可链到文章，活动资格不在本域。  
12. **不做**：CRM、赠金引擎、跟单放大、排行榜。

## 三、跨域契约

| 系统 | 协作 |
|------|------|
| 通知 | 消费 `content.published` 等做推送 |
| 营销与推荐 | 读 content 元数据做推荐位；本域不算推荐分 |
| 社交交易 | IM 深链 Provider；可消费 `copy.*` 发系统卡片 |
| 交易 | 晒单授权只读 |
| 用户 / 权限 | 身份与审核员 RBAC |

**本域事件**：`content.published/unpublished.v1`、`im.message.flagged.v1`、`im.member.banned.v1`、`im.share_card.requested.v1`。

## 四、内容 CMS

| `content_type` | 用途 |
|----------------|------|
| `NEWS` | 金融信息头条 |
| `ANNOUNCEMENT` | 系统/产品公告 |
| `EDUCATION` | 投教 |
| `LEGAL_NOTICE` | 运营向提示（强制协议仍在准入） |

发布流：`DRAFT → IN_REVIEW → PUBLISHED ⇄ UNPUBLISHED`。  
字段：多语言、实体/辖区、生效/过期、风险脚注。

## 五、社群 IM（大厅 / 广场）

### 5.1 能力

- 广场/大厅频道、主题群、私聊  
- 角色：OWNER / ADMIN / MEMBER  
- 禁言、踢人、举报、审核台  
- 系统消息与晒单卡片  

### 5.2 模型纲要

```text
ImConversation（DIRECT / GROUP / PLAZA_CHANNEL）
ImMember
ImMessage（text/image/system/share_card）
ImModerationCase
ImExternalBinding
ImShareAuthorization
```

### 5.3 与社交交易

| 动作 | 归属 |
|------|------|
| 「跟单该信号源」按钮 | 跳转 [13] |
| 订阅成功系统卡片 | 本域发 IM；事实来自 [13] 事件 |
| 信号源主页评论 | [13] 轻量；大厅群聊在本域 |

## 六、表与 API 纲要

```sql
t_content_article, t_content_publish_log
t_im_conversation, t_im_member, t_im_message
t_im_moderation_case, t_im_external_binding, t_im_share_authorization
```

API：App 头条/详情、App IM、Admin 内容审批、Admin 社群审核。
