| Outbox | 6.4「发 MQ」与 `t_engine_outbox` 并存 | 统一 Outbox：`t_engine_outbox` 含 `retry_count`、`next_retry_at`、死信 topic（见补充卷 §4.2） |

| 错误码 | 自定义字符串 | 统一引用补充卷 §三 错误码注册表（`TRD_*`、`PLATFORM_*`） |

## 一、撮合引擎在整个系统中的位置

```
Trading Engine 不是一个独立"业务模块"
它是交易域的常驻内存运行时进程
是订单/成交/持仓/保证金在运行时的"中枢调度者"

启动时从数据库加载数据到内存
运行时在内存中完成所有实时计算
只在状态变更时经 WAL/异步 Writer 写回数据库（投影）

┌─────────────────────────────────────────────────────────────────────┐
│                     撮合引擎进程 (Trading Engine)                    │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                     内存数据结构                              │   │
│  │  SymbolBook{}        每个品种的报价+挂单簿                    │   │

│  │  SymbolConfig{}      品种参数缓存                            │   │

│  │ QuoteEng  │  │ OrderEng  │  │ RiskEng   │  │ SettleEng    │   │
│  └─────┬─────┘  └─────┬─────┘  └─────┬─────┘  └──────┬────────┘   │
│  ┌─────┴───────────────────────────────────────────┴──────────┐  │
│  │                    事件总线 (EventBus，进程内)                  │  │
```

### 1.1 跨域依赖（RPC + 共享内存）

| 依赖 | 方式 | 提供方 | 缓存策略 |
|------|------|--------|----------|
| Core SymbolBook | 共享内存（SHM SPSC） | 行情体系 | 实时订阅，seq 单调 |
| 交易时段 | gRPC `SessionQueryService` | 时间配置域 | 本地缓存 + 订阅 override Pub/Sub 失效 |
| 杠杆解析 | gRPC `LeverageResolver` | 杠杆配置域 | Caffeine + Redis，按 `config_version` 失效 |
| 路由解析 | gRPC `RouteResolver` | 路由网关 | 本地缓存 + 配置广播失效 |
| 资金写 | gRPC `FundService` | 资金域 | 不缓存，事务内调用 |
| 风控规则 | 引擎内嵌内存 | 风控域 | 配置热更 shadow→swap |

## 二、引擎启动：从数据库到内存

```
引擎启动流程（必须按顺序，有依赖关系）
═══════════════════════════════════════════════════════════

Step 1: 加载基础配置
    ├── t_system_config → SystemConfig{}
    │     服务器时区、全局开关、默认参数
    └── 订阅 SessionQueryService（时间配置域）
          初始化 SessionCache{}：拉取当日 + 次日 t_daily_session 快照
          订阅 t_session_override Pub/Sub，实时失效本地缓存
          【删除旧稿的 t_system_trading_session / t_trading_calendar / t_symbol_session 直算】

Step 2: 加载品种体系
    ├── t_symbol_category → Map<categoryId, Category>
    ├── t_symbol_group → Map<groupId, SymbolGroup>
    ├── t_symbol → Map<symbolKey, SymbolConfig>
    │     合约规格、计价模式、报价过滤参数
    │     品种级 NULL → 从 SymbolGroup 继承
    ├── t_symbol_trading_config → 合并到 SymbolConfig
    ├── t_symbol_swap_config → 合并到 SymbolConfig（Swap BASE，加点在路由）
    └── t_symbol_spread_schedule → 合并到 SymbolConfig
          【删除旧稿的 t_symbol_session，时段由 SessionQueryService 提供】

Step 3: 加载账户体系
    ├── t_account_group → Map<groupId, AccountGroupConfig>
    │     强平阈值、追保阈值、负余额保护、持仓限制
    │     【杠杆值不再在此加载，由 LeverageResolver 实时解析】
    ├── t_trading_account → Map<accountId, AccountState>
    │     余额、信用、冻结、净值、保证金占用（引擎快照）
    └── t_user_account_relation → 关系索引

Step 4: 加载持仓与挂单
    ├── t_position_active → Map<accountId, List<Position>>
    │     含 spec_snapshot_id（品种规格快照引用）
    ├── t_order WHERE status IN (WORKING, PLACED, PENDING_EXEC, DEALER_HOLD, PARTIALLY_FILLED)
    │   → 重建 SymbolBook 挂单簿（按品种分组，价格排序）
    └── t_execution_link WHERE status = PENDING → 重建 A-Book 在途

Step 5: 加载风控规则
    ├── t_risk_rule → RiskRuleSet{}
    │     按 priority 降序，scope 多级
    ├── t_risk_config_version → 版本戳
    └── 订阅风控配置热更广播（shadow → 原子 swap）

Step 6: 加载路由规则（缓存层）
    ├── 调用 RouteResolver.PullSnapshot() → RouteCache{}
    │     含 match_conditions 求值器（共享库）
    └── 订阅路由配置广播，config_version 变更时失效

Step 7: 加载杠杆解析（缓存层）
    ├── 调用 LeverageResolver.PullSnapshot() → LeverageCache{}
    └── 订阅杠杆配置广播，config_version 变更时失效

Step 8: 连接行情与网关
    ├── 订阅 Core SymbolBook SHM 流（行情体系）
    │     初始化 SymbolBook{}：bid/ask/seq/stale/gap_level
    ├── 连接路由网关 Gateway Client（A-Book 执行）
    └── 连接资金域 FundService Client（ledger 写）

Step 9: 启动 WAL 恢复
    ├── 读取本地 WAL 日志，重放未 Commit 的事务
    ├── 与 MySQL 投影对账，修复不一致
    └── 进入 ACTIVE 状态，开始接受订单
```

### 2.1 启动幂等与重入

- 引擎重启用 `instance_id + boot_seq` 标识，WAL 重放按 `seq` 单调去重。
- 启动加载失败任一 Step 即拒绝进入 ACTIVE，告警人工介入。
- 多实例分片启动时，按 `account_id shard` 亲和，避免跨分片持仓。

## 三、Tick 链路（行情驱动）

### 3.1 Core Book Delta 入口

```
行情体系经 SHM 推送 Core Book Delta：
  applyCoreBookDelta(symbol_id, seq, bid, ask, stale, gap_level)

伪代码：
function onCoreBookDelta(delta):
    book = SymbolBook[delta.symbol_id]
    if delta.seq <= book.seq:
        return  # 乱序丢弃，行情侧保证 seq 单调
    book.bid = delta.bid
    book.ask = delta.ask
    book.seq = delta.seq
    book.stale = delta.stale
    book.gap_level = delta.gap_level
    EventBus.emit(CORE_BOOK_DELTA, book)

    # 触发下游
    if not book.stale:
        checkPendingOrders(book)      # 挂单触发
        checkSLTP(book)               # 止损止盈
        checkPriceAlert(book)          # 价格预警
    checkMarginCallAndStopOut(book)   # 风控（含 stale 处理）
```

### 3.2 挂单触发（只扫内存）

```
function checkPendingOrders(book):
    pendings = book.pendingOrdersByPrice()  # 内存有序集合
    for order in pendings:
        if order.type == LIMIT and priceCrossed(order, book):
            fillOrder(order, book)          # 转成交
        elif order.type == STOP and priceCrossed(order, book):
            promoteToMarket(order, book)
    # 禁止热路径扫 t_order，只扫 SymbolBook 内存索引
```

### 3.3 穿越判定（防漏触发）

- LIMIT BUY：`book.ask <= order.price`
- LIMIT SELL：`book.bid >= order.price`
- STOP BUY：`book.ask >= order.price`
- STOP SELL：`book.bid <= order.price`
- 跳空（`gap_level > 0`）：按穿越后第一档价成交，记录滑点到 deal。

### 3.4 STALE 处理

- `book.stale = true` 时：**禁开新仓**（返回 `TRD_DEGRADED_STALE_QUOTE`），允许平仓与挂单触发（用最后已知价）。
- 强平在 STALE 期间**跳过**，避免用坏价打客户；恢复后立即补扫。

## 四、下单链路

### 4.1 PreTrade 检查链（10 步，不可重排）

```
function submitOrder(req):
    # 幂等：先查 (account_id, client_order_id)
    existing = idempotentCache.get(req.account_id, req.client_order_id)
    if existing: return existing

    acct = AccountState[req.account_id]
    sym = SymbolConfig[req.symbol_id]

    # Step 0: 自成交防护（新增，对标 TT）
    if isSelfTrade(acct, req):
        return reject(TRD_VALIDATION_SELF_TRADE)

    # Step 1: 账户状态
    if acct.status not in [ACTIVE]: return reject(ACC_STATE_ACCOUNT_FROZEN)
    if not acct.can_trade: return reject(ACC_STATE_ACCOUNT_FROZEN)

    # Step 2: Kill Switch（风控域广播）
    if riskKillSwitchActive(acct.entity_id, sym.symbol_id):
        return reject(RISK_STATE_KILL_SWITCH_ON)

    # Step 3: 品种可访问性 + 时段
    sess = SessionQueryService.IsTradable(sym.symbol_id, now, acct.entity_id)
    if not sess.tradable:
        if sess.runtime_state == CLOSE_ONLY and req.is_open:
            return reject(SESS_STATE_CLOSE_ONLY)
        return reject(SESS_STATE_MARKET_CLOSED)

    # Step 4: 订单参数
    if not validateOrderParams(req, sym): return reject(TRD_VALIDATION_VOLUME_INVALID)

    # Step 5: 持仓限额
    if exceedsPositionLimit(acct, sym, req): return reject(TRD_LIMIT_POSITION_EXCEEDED)

    # Step 6: 保证金（调用 LeverageResolver）
    lev = LeverageResolver.Resolve(acct.id, sym.id, notional, equity, config_version)
    margin = calcMargin(req, lev, sym)
    if acct.free_margin < margin: return reject(TRD_LIMIT_MARGIN_INSUFFICIENT)

    # Step 7: 日损
    if exceedsDailyLoss(acct): return reject(TRD_LIMIT_DAILY_LOSS_EXCEEDED)

    # Step 8: 自定义规则（风控域 RiskRuleSet）
    if not evalCustomRules(acct, sym, req): return reject(RISK_LIMIT_*)

    # Step 9: Core Book 健康
    if SymbolBook[sym.id].stale and req.is_open:
        return reject(TRD_DEGRADED_STALE_QUOTE)

    # Step 10: 公司敞口预检
    if not houseExposurePrecheck(acct, sym, req): return reject(RISK_LIMIT_HOUSE_EXPOSURE_EXCEEDED)

    # 通过 → 路由
    route = RouteResolver.ResolveRoute(sym.id, acct.id, sess.session, req.volume, req.side, config_version)
    if route.reject_reason: return reject(route.reject_reason)

    # 执行
    return executeByRoute(req, route, lev)
```

## 五、执行路径（A-Book / B-Book / Mixed）

### 5.1 B-Book（内部簿记）

```
function executeBBook(req, route, lev):
    book = SymbolBook[req.symbol_id]
    if book.stale: return reject(TRD_DEGRADED_STALE_QUOTE)

    price = (req.side == BUY) ? book.ask : book.bid
    # 滑点保护
    if req.slippage_points and priceDeviation(price, req) > req.slippage_points:
        return reject(TRD_LIMIT_SLIPPAGE_EXCEEDED)

    deal = createDeal(req, price, lev)
    # 原子提交：WAL + 内存 + Outbox（见 §六）
    commitTransaction(deal)
    # 异步：FundService.CreditAccount/DebitAccount 写 ledger
    # 异步：发 deal.created.v1（Outbox）
    return SubmitOrderResponse(order_id, FILLED, deal_id=deal.id)
```

### 5.2A A-Book（LP 执行）

```
function executeABook(req, route, lev):
    order.status = PENDING_EXEC
    writeWAL(order)
    # 调用网关 Gateway.SubmitToLP(route.gateway_id, req)
    link = createExecutionLink(req, route)
    # 等待 LP 回报（异步，超时见 §5.2B）
    return SubmitOrderResponse(order_id, PENDING_EXEC)
    # LP 回报经 onLpReport 回调处理（见 §5.2B）
```

### 5.2B A-Book 超时与重定价

```
function onLpReport(link, report):
    if report.timeout:
        # 不用 sleep(executionDelayMs)（旧稿作废）
        # 改为延迟队列 + 下一 Tick 重定价
        scheduleReprice(link, nextTickOrDelay(route.execution_delay_ms))
        if link.retry_count >= max_retry:
            cancelOrder(link.order_id, reason=TRD_EXTERNAL_LP_TIMEOUT)
        return
    if report.rejected:
        order.status = REJECTED
        order.reject_reason = TRD_EXTERNAL_LP_REJECTED
        commitTransaction(order)
        return
    # 成交
    deal = createDealFromReport(link, report)
    commitTransaction(deal)
```

### 5.2C Mixed（按 allocation_ratio 拆单）

```
function executeMixed(req, route, lev):
    # route.allocations = [{gateway_id, ratio, execution_profile_id}, ...]
    subDeals = []
    for alloc in route.allocations:
        sub = splitOrder(req, alloc.ratio)
        if alloc.gateway_id == INTERNAL:
            subDeal = executeBBook(sub, route, lev)
        else:
            subDeal = executeABook(sub, route, lev)
        subDeals.append(subDeal)

    # 客户侧单 Deal：加权均价（按各子单实际成交量）
    clientDeal = mergeDeals(subDeals)
    clientDeal.price = weightedAvg(subDeals, byVolume)
    clientDeal.commission_amount = sum(sub.commission for sub in subDeals)
    # 客户只看到一笔聚合 Deal，子单在 t_execution_link 关联
    commitTransaction(clientDeal)
    return SubmitOrderResponse(order_id, FILLED, deal_id=clientDeal.id)
```

```
function dealerHold(req):
```

## 六、WAL + Commit（持久化与恢复）

### 6.1 WAL 设计原则

- **WAL 前置 ACK**：订单/成交/持仓变更先写本地 WAL，再返回客户端；异步 Writer 批量刷 MySQL。
- **崩溃恢复**：重启时重放 WAL，与 MySQL 投影对账，修复不一致。
- **RPO = 0**：WAL 落盘前不 ACK；同步复制到备机（同城双活）。

### 6.2 WAL Entry Schema

```
WAL Entry（二进制，按 seq 单调）:
  - seq: int64          # 引擎内单调递增
  - account_id: int64
  - payload_type: enum # ORDER/DEAL/POSITION/MARGIN
  - payload_crc: uint32
  - payload: bytes     # 序列化的变更
  - outbox_intents: []  # 关联的 Outbox 事件
  - timestamp: int64
```

### 6.3 Commit 流程

```
function commitTransaction(change):
    # 1. 写 WAL（同步落盘 + 复制备机）
    walSeq = wal.append(change)
    # 2. 应用到内存
    applyToMemory(change)
    # 3. 写 Outbox（同事务）
    for event in change.outboxEvents:
        outbox.append(event, walSeq)
    # 4. ACK 客户端
    ack(change)
    # 5. 异步：MySQL Writer 批量刷 + Kafka 发布 Outbox
```

### 6.4 Outbox 死信与重试

- `t_engine_outbox` 含 `retry_count`、`next_retry_at`、`status`。
- 重试 N 次（默认 5）入死信 topic `dlq.trading.deal`，告警 + 人工。
- Kafka 发布成功后标记 `status=PUBLISHED`，定期清理。

## 七、强平链路

### 7.1 Margin Call 与 Stop Out 触发

```
function checkMarginCallAndStopOut(book):
    for acct in AccountState.values():
        ml = marginLevel(acct, book)
        if ml <= acct.stop_out_level:
            stopOut(acct, book)
        elif ml <= acct.margin_call_level:
            if not acct.in_margin_call:
                acct.in_margin_call = true
                EventBus.emit(MARGIN_CALL, acct)
                # 发 margin.call.v1（Outbox）
```

### 7.2 Stop Out 三模式

```
function stopOut(acct, book):
    mode = acct.stop_out_mode
    positions = sortedPositions(acct, by=mode)
    while marginLevel(acct, book) <= acct.stop_out_level and positions:
        pos = positions.pop(0)
        deal = closePosition(pos, book, reason=STOP_OUT)
        commitTransaction(deal)
        # 释放保证金后重新计算 ml
    # 负余额保护
    if acct.balance < 0 and acct.negative_balance_protection:
        FundService.CreditAccount(acct, -acct.balance, ledger_type=NEGATIVE_BALANCE_CORRECTION)
```

| 模式 | 排序 |
|------|------|
| 1 亏损最大优先 | 按 floating_pnl 升序 |
| 2 保证金占用最大 | 按 margin_used 降序 |
| 3 Smart 部分平仓 | 按释放量/损失比降序，部分平仓至 ml > stop_out |

### 7.3 强平失败升级

- 无报价/STALE：跳过，恢复后补扫。
- 连续失败 N 次：Dealer 介入 + 品种 HALT + 通知运营（`risk.kill_switch.toggled.v1`）。
- 最大滑点/重试次数由账户组配置。

## 十一、容灾与降级

| 场景 | 降级 |
|------|------|
| 主引擎故障 | WAL 重放，备机接管 < 30s；切换期间**只平不开** |
| Core Book STALE | 禁开仓，允许平仓 |
| SessionQueryService 不可用 | 本地缓存兜底（最近快照），告警 |
| LeverageResolver 不可用 | Caffeine + Redis 兜底，超时用最近解析值 |
| RouteResolver 不可用 | 本地 RouteCache 兜底，信用检查降级为拒绝 |
| FundService 不可用 | 拒绝新成交（避免账实不一致），允许平仓挂账待恢复 |
| Kafka 不可用 | Outbox 积压本地，恢复后补发 |
