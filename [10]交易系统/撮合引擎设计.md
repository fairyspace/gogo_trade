# 撮合引擎设计（Trading Engine 运行时）

> **定位**：交易域的 **Trading Engine 运行时实现说明书（HOW）**。  
> **上级权威**：[《交易系统设计》](./交易系统设计.md) v1.0 — 域决策、状态机、DDL、RPC/事件契约以彼为准；**本文不得另立冲突决策**。  
> **术语**：本平台「撮合」= 零售 CFD/FX 的 **簿记执行引擎**（STP / B-Book），**不是**交易所多边中央对手方 Match Engine。  
> **职责边界**：本文保留启动加载、内存结构、Tick/下单/强平/日切算法伪代码；**不重复**完整 DDL 与跨域事件 schema（见交易系统设计）；报价接入细节见《行情体系设计》，引擎**只消费 Core SymbolBook**。伪代码与进程边界描述算法与职责，**不锁定实现语言**。

---

## 〇、相对《交易系统设计》的对齐声明

| 主题 | 本文实现约束 |
|------|-------------|
| 订单持久态 | 预交易校验不落库；接受后写 `WORKING`（市价）或 `PLACED`（挂单）；禁止把 `SUBMITTED/VALIDATED` 当持久态 |
| 定价 | 挂单触发 / B-Book 成交 / 估值 / 强平一律用 **Core Book**；Client Quote 仅展示 |
| 挂单触发 | **只扫内存** `SymbolBook` 有序集合；禁止热路径扫 `t_order` |
| 事件外发 | 状态提交经 `t_engine_outbox` → Kafka；事件名见交易系统（如 `deal.created.v1`）；不再使用「尽力发 MQ」 |
| ledger | 结构权威在资金域；引擎只写 `TRADE_PROFIT/COMMISSION/SWAP/DIVIDEND/FEE` 等交易类型 |
| 幂等 | `client_order_id`（账户内）+ `request_id`（传输层）必填；重复返回首次结果 |
| A-Book | 发往 LP 后状态 `PENDING_EXEC`；关联写入 `t_execution_link` |

## 一、撮合引擎在整个系统中的位置

```
Trading Engine 不是一个独立"业务模块"
它是交易域的常驻内存运行时进程
是订单/成交/持仓/保证金在运行时的"中枢调度者"

启动时从数据库加载数据到内存
运行时在内存中完成所有实时计算
只在状态变更时经 WAL/异步 Writer 写回数据库（投影）
```

## 二、引擎启动：从数据库到内存

```
引擎启动流程（必须按顺序，有依赖关系）
═══════════════════════════════════════════════════════════

Step 1: 加载基础配置
    ├── t_system_config → SystemConfig{}
    │     服务器时区、全局开关、默认参数
    │
    ├── t_system_trading_session → SessionManager{}
    │     系统级交易时段（平台什么时候开/关）
    │
    └── t_trading_calendar → SessionManager{}
          今日及未来7天的节假日/特殊时段


Step 2: 加载品种体系
    ├── t_symbol_category → Map<categoryId, Category>
    │
    ├── t_symbol_group → Map<groupId, SymbolGroup>
    │     含组级默认参数
    │
    ├── t_symbol → Map<symbolKey, SymbolConfig>
    │     合约规格、保证金模式、报价参数
    │     如果品种级参数为NULL → 从SymbolGroup继承
    │
    ├── t_symbol_trading_config → 合并到 SymbolConfig
    │     手数限制、订单类型、距离限制
    │
    ├── t_symbol_swap_config → 合并到 SymbolConfig
    │     Swap计算参数、逐日系数
    │
    ├── t_symbol_session → 合并到 SymbolConfig
    │     每个品种的交易/报价时段
    │
    └── t_symbol_spread_schedule → 合并到 SymbolConfig
          点差时段附加配置


Step 3: 加载账户体系
    ├── t_account_group → Map<groupId, AccountGroupConfig>
    │     杠杆、风控参数、持仓限制
    │
    ├── t_account_group_commission → 合并到 AccountGroupConfig
    │     各品种类别的佣金规则
    │
    ├── t_account_group_symbol_config → 合并到 AccountGroupConfig
    │     各品种的准入和参数覆盖
    │
    ├── t_leverage_template + t_leverage_template_tier
    │     → Map<templateId, LeverageTiers[]>
    │     浮动杠杆阶梯
    │
    └── t_trading_account (status IN (0,1,2)) → Map<accountId, AccountState>
          每个活跃账户的初始状态
          AccountState {
              accountId, accountNo, groupId
              currency, leverage, swapFree
              balance, credit, equity
              marginUsed, marginFree, floatingPnl
              tradeEnabled, expertEnabled
              positions: Map<positionTicket, PositionState>
              pendingOrders: Map<orderTicket, OrderState>
              frozenAmount (从t_account_fund_freeze汇总)
          }


Step 4: 加载活跃持仓和挂单
    ├── t_position (status=1) → 加载到对应 AccountState.positions
    │     PositionState {
    │         positionTicket, symbolKey, side
    │         volume, priceOpen, priceSl, priceTp
    │         swap, margin, profit
    │         openedAt
    │     }
    │
    └── t_order (status IN (0,1,2,3,4)) → 加载到对应 AccountState.pendingOrders
          OrderState {
              orderTicket, symbolKey, orderType, side
              priceRequested, priceTrigger, priceSl, priceTp
              volumeRequested, volumeFilled, volumeRemaining
              expirationType, expireAt, fillingMode
          }


Step 5: 构建品种订单簿
    对每个品种:
    SymbolBook {
        symbolKey
        currentBid, currentAsk, currentLast
        spreadPoints
        tickTime
        
        // 该品种的所有挂单（从所有AccountState中收集）
        buyLimitOrders:  SortedSet<OrderState>  按价格降序
        sellLimitOrders: SortedSet<OrderState>  按价格升序
        buyStopOrders:   SortedSet<OrderState>  按触发价升序
        sellStopOrders:  SortedSet<OrderState>  按触发价降序
        
        // 该品种的所有活跃持仓引用（用于SL/TP检查）
        activePositions: List<PositionState>
        
        // 该品种的价格预警
        priceAlerts: List<PriceAlert>
    }


Step 6: 加载风控与路由
    ├── t_risk_rule (status=1) → RiskRuleSet
    ├── t_route_rule (status=1) → RouteRuleSet
    ├── t_fund_risk_rule (status=1) → FundRiskRuleSet
    └── t_risk_margin_call (status=1) → 标记对应AccountState


Step 7: 连接网关
    ├── t_gateway (status=1) → 逐个建立连接
    └── t_gateway_symbol_mapping → 建立品种映射表
        GatewayConnection {
            gatewayId, protocol, host, port
            symbolMappings: Map<localSymbolKey, remoteSymbol>
            status: CONNECTING → CONNECTED
        }


Step 8: 开始接收报价 → 引擎进入运行态
```

## 三、核心链路1：Tick到达 → 全链路处理

```
这是引擎最高频的操作
每秒可能处理数千~数万个Tick
必须全部在内存中完成，不能有数据库IO

※ 报价权威边界（与《行情体系设计》对齐）:
   LP 原始接入 / 池聚合 / Core Book 产出 / Redis·CH·WS 分发 → 行情体系
   本引擎 Tick 入口 = 已归一化的 Core SymbolBook 更新（含 seq / STALE）
   下列 PHASE 1 若仍描述 mapping/加点，视为历史算法参考；落地以行情体系为准
   本引擎热路径只保证: 更新本地 book → 挂单/SLTP/浮盈/强平

行情体系推送 Core Book Delta（或同进程共享内存读）
    │
    ▼
═══════════════════════════════════════════════════════════
 PHASE 1: 本地 Core Book 应用 (原 QuoteEngine 消费侧)
═══════════════════════════════════════════════════════════
    │
    ├── 1.0 接受 Core Bid/Ask/seq/stale（权威价，非 Client Quote）
    │
    ├── 1.1 品种映射（仅当引擎旁路接入调试时；生产由行情体系完成）
    │     remoteSymbol "EUR/USD" → localSymbolKey "EURUSD"
    │     来源: GatewayConnection.symbolMappings
    │     (对应表: t_gateway_symbol_mapping)
    │
    ├── 1.2 价格转换
    │     IF mapping.price_scale_factor != NULL:
    │         bid = remoteBid × price_scale_factor
    │         ask = remoteAsk × price_scale_factor
    │     IF mapping.price_digits_remote != symbol.digits:
    │         四舍五入到 symbol.digits 位小数
    │
    ├── 1.3 加点(Markup)
    │     // 网关级加点
    │     bid = bid - (mapping.markup_points × symbol.tick_size)
    │     ask = ask + (mapping.markup_points × symbol.tick_size)
    │     
    │     // 时段附加点差
    │     查 SymbolConfig.spreadSchedules → 当前时段
    │     IF 命中时段配置:
    │         IF spread_override != NULL:
    │             强制点差 = spread_override
    │         ELSE:
    │             bid = bid - (extra_spread_points/2 × tick_size)
    │             ask = ask + (extra_spread_points/2 × tick_size)
    │     (对应表: t_symbol_spread_schedule)
    │
    ├── 1.4 点差约束
    │     spread = (ask - bid) / tick_size
    │     IF spread < symbol.spread_limit_min:
    │         调整ask使点差 = spread_limit_min
    │     IF symbol.spread_limit_max != NULL AND spread > spread_limit_max:
    │         调整使点差 = spread_limit_max
    │     (对应表: t_symbol.spread_limit_min/max)
    │
    ├── 1.5 异常报价过滤
    │     IF symbol.filter_bad_ticks > 0:
    │         deviation = |newBid - oldBid| / tick_size
    │         IF deviation > filter_bad_ticks:
    │             标记为可疑Tick
    │             IF 连续N个方向一致 → 接受(趋势行情)
    │             ELSE → 丢弃，记录日志
    │
    │     IF symbol.allow_negative_price == 0 AND bid <= 0:
    │         丢弃
    │
    ├── 1.6 跳空处理
    │     IF symbol.gap_mode != 0:
    │         gap = |newBid - oldBid| / tick_size
    │         IF gap > gap_threshold_l3: gap_state = L3
    │         ELIF gap > gap_threshold_l2: gap_state = L2
    │         ELIF gap > gap_threshold_l1: gap_state = L1
    │         ELSE: gap_state = NORMAL
    │     (对应表: t_symbol.gap_mode/gap_threshold_*)
    │
    ├── 1.7 更新SymbolBook
    │     book = symbolBooks[symbolKey]
    │     book.previousBid = book.currentBid
    │     book.currentBid = bid
    │     book.currentAsk = ask
    │     book.currentLast = last  // 如果有
    │     book.spreadPoints = spread
    │     book.tickTime = tickTime
    │     book.tickDirection = bid > previousBid ? 1 : bid < previousBid ? -1 : 0
    │
    └── 1.8 行情落库/推送（非本引擎职责）
          Redis / ClickHouse / WS / K 线 → 《行情体系设计》异步完成
          本引擎不在热路径重复写行情管道

    │
    ▼
═══════════════════════════════════════════════════════════
 PHASE 2: 挂单触发检查 (OrderEngine)
═══════════════════════════════════════════════════════════
    │
    │  ※ 只扫内存 SymbolBook 挂单索引，禁止热路径查 t_order
    │  ※ 触发价比较使用 Core Book 的 Bid/Ask（非 Client Quote）
    │  ※ Core Book STALE 时跳过触发与新开仓成交
    │
    │  遍历该品种的挂单簿（内存有序集合）
    │  注意: 只有价格穿越时才触发，不是等于就触发
    │  用 previousBid/Ask 和 currentBid/Ask 判断穿越
    │
    ├── 2.1 Buy Limit 检查
    │     book.buyLimitOrders 中:
    │     IF previousAsk > order.priceRequested
    │        AND currentAsk <= order.priceRequested:
    │         → 触发！生成市价买单
    │         fillPrice = MIN(currentAsk, order.priceRequested)
    │                     // Limit单价格改善：成交价不劣于请求价
    │
    ├── 2.2 Sell Limit 检查
    │     book.sellLimitOrders 中:
    │     IF previousBid < order.priceRequested
    │        AND currentBid >= order.priceRequested:
    │         → 触发！
    │         fillPrice = MAX(currentBid, order.priceRequested)
    │
    ├── 2.3 Buy Stop 检查
    │     book.buyStopOrders 中:
    │     IF previousAsk < order.priceTrigger
    │        AND currentAsk >= order.priceTrigger:
    │         → 触发！
    │         IF order.orderType == BUY_STOP:
    │             fillPrice = currentAsk  // Stop单按当前价成交，可能滑点
    │         IF order.orderType == BUY_STOP_LIMIT:
    │             转为 BUY_LIMIT 挂单，挂单价 = order.priceRequested
    │
    ├── 2.4 Sell Stop 检查
    │     book.sellStopOrders 中:
    │     IF previousBid > order.priceTrigger
    │        AND currentBid <= order.priceTrigger:
    │         → 触发！
    │         类似 Buy Stop 逻辑
    │
    └── 2.5 对每个触发的挂单
          从挂单簿移除
          → 进入 PHASE 5 (订单执行流程)
          source = 'SYSTEM', close_reason 根据类型标记

    │
    ▼
═══════════════════════════════════════════════════════════
 PHASE 3: 止损/止盈检查 (OrderEngine)
═══════════════════════════════════════════════════════════
    │
    │  遍历该品种的所有活跃持仓 book.activePositions
    │
    ├── 3.1 多头持仓(side=BUY)
    │     // 多头止损: bid跌破止损价
    │     IF position.priceSl != NULL
    │        AND previousBid > position.priceSl  // 之前在止损价上方
    │        AND currentBid <= position.priceSl:  // 现在跌破
    │         → 生成平仓卖单
    │         close_reason = 'SL'
    │
    │     // 多头止盈: bid涨到止盈价
    │     IF position.priceTp != NULL
    │        AND previousBid < position.priceTp
    │        AND currentBid >= position.priceTp:
    │         → 生成平仓卖单
    │         close_reason = 'TP'
    │
    └── 3.2 空头持仓(side=SELL)
          // 空头止损: ask涨破止损价
          IF position.priceSl != NULL
             AND previousAsk < position.priceSl
             AND currentAsk >= position.priceSl:
              → 生成平仓买单
              close_reason = 'SL'

          // 空头止盈: ask跌到止盈价
          IF position.priceTp != NULL
             AND previousAsk > position.priceTp
             AND currentAsk <= position.priceTp:
              → 生成平仓买单
              close_reason = 'TP'

    │
    ▼
═══════════════════════════════════════════════════════════
 PHASE 4: 保证金与强平检查 (RiskEngine)
═══════════════════════════════════════════════════════════
    │
    │  只需要检查持有该品种的账户（不是所有账户）
    │  book.activePositions 按 accountId 分组
    │
    ├── 4.1 更新每个受影响持仓的浮动盈亏
    │     FOR each position in book.activePositions:
    │         IF position.side == BUY:
    │             position.profit = (currentBid - position.priceOpen)
    │                             × position.volume
    │                             × symbolConfig.contractSize
    │                             × exchangeRate(profitCurrency → accountCurrency)
    │             position.priceCurrentSnapshot = currentBid
    │         ELSE:
    │             position.profit = (position.priceOpen - currentAsk)
    │                             × position.volume
    │                             × symbolConfig.contractSize
    │                             × exchangeRate(profitCurrency → accountCurrency)
    │             position.priceCurrentSnapshot = currentAsk
    │
    ├── 4.2 更新每个受影响账户的净值
    │     FOR each accountId in affectedAccounts:
    │         state = accountStates[accountId]
    │         state.floatingPnl = SUM(pos.profit for pos in state.positions.values())
    │         state.equity = state.balance + state.credit + state.floatingPnl
    │         state.marginFree = state.equity - state.marginUsed
    │         IF state.marginUsed > 0:
    │             state.marginLevel = (state.equity / state.marginUsed) × 100
    │         ELSE:
    │             state.marginLevel = NULL  // 无持仓
    │
    ├── 4.3 Margin Call 检查
    │     groupConfig = accountGroupConfigs[state.groupId]
    │     
    │     IF state.marginLevel != NULL:
    │         IF state.marginLevel <= groupConfig.marginCallLevel:
    │             IF NOT state.marginCallActive:
    │                 // 触发 Margin Call
    │                 state.marginCallActive = true
    │                 state.tradeEnabled = false  // 禁止新开仓
    │                 
    │                 → 写 t_risk_margin_call (异步)
    │                 → 写 t_risk_event_log (异步)
    │                 → 发 MQ: MARGIN_CALL 事件 → 通知系统
    │                 (对应表: t_risk_margin_call, t_risk_event_log)
    │
    │         ELIF state.marginCallActive
    │              AND state.marginLevel > groupConfig.marginCallLevel:
    │             // 解除 Margin Call
    │             state.marginCallActive = false
    │             state.tradeEnabled = true  // 恢复开仓权限
    │             
    │             → 更新 t_risk_margin_call status=RESOLVED (异步)
    │             → 发 MQ: MARGIN_CALL_RESOLVED
    │
    └── 4.4 Stop Out 检查
          IF state.marginLevel != NULL
             AND state.marginLevel <= groupConfig.stopOutLevel:
              
              → 进入强平流程 (见 PHASE 7)

    │
    ▼
═══════════════════════════════════════════════════════════
 PHASE 5: 价格预警检查 (QuoteEngine)
═══════════════════════════════════════════════════════════
    │
    │  遍历 book.priceAlerts
    │
    ├── alert.alertType == 1 (Bid上穿):
    │     IF previousBid < alert.targetPrice
    │        AND currentBid >= alert.targetPrice → 触发
    │
    ├── alert.alertType == 2 (Bid下穿):
    │     IF previousBid > alert.targetPrice
    │        AND currentBid <= alert.targetPrice → 触发
    │
    └── 触发后:
          alert.status = TRIGGERED
          alert.triggeredAt = now
          alert.triggeredPrice = currentBid/Ask
          → 更新 t_price_alert (异步)
          → 发 MQ: PRICE_ALERT_TRIGGERED → 通知系统
          IF alert.repeatMode == 1: 从内存移除
          (对应表: t_price_alert)

═══════════════════════════════════════════════════════════
 单个Tick处理完毕
 总耗时目标: < 1ms (500品种平均)
═══════════════════════════════════════════════════════════
```

## 四、核心链路2：客户下单 → 成交 → 持仓

```
这是第二高频的操作
从客户提交到成交完成，目标 < 50ms

 PHASE 5: 订单提交与验证
═══════════════════════════════════════════════════════════

客户通过WebSocket/API提交下单请求

5.1 参数解析

5.2 快速拒绝检查(纯内存，< 0.1ms)
    │
    ├── 全局熔断
    │     Redis: risk:circuit_breaker → global_trading
    │     IF "OFF" → REJECT "Trading is temporarily suspended"
    │
    ├── 品种停牌
    │     Redis: risk:symbol_halt:{symbolKey}
    │     IF halted → REJECT "Symbol is halted"
    │
    ├── 品种交易时段

    │     还要检查 SessionManager → 系统级时段和节假日

    │     (对应表: t_symbol_session, t_system_trading_session, t_trading_calendar)

    ├── 账户状态
    │     state = accountStates[accountId]
    │     IF state == NULL → REJECT "Account not found"
    │     IF state.status != ACTIVE → REJECT "Account is not active"
    │     IF state.tradeEnabled == false → REJECT "Trading is disabled"

    │

    │     IF state.marginCallActive AND 本单是开仓:
    │         → REJECT "Margin Call active, only closing allowed"
    │
    └── 品种准入
          groupConfig = accountGroupConfigs[state.groupId]
          symbolCategoryConfig = groupConfig.symbolConfigs[symbol.category]
          IF symbolCategoryConfig == NULL OR enabled == false:
              → REJECT "Symbol not available for this account group"
          IF symbolCategoryConfig.tradeMode == CLOSE_ONLY AND 本单是开仓:
              → REJECT "Only closing positions allowed"
          (对应表: t_account_group_symbol_config)
    │
    ▼
5.3 订单参数校验(纯内存，< 0.1ms)
    │
    ├── 手数检查
    │     tradingConfig = symbolConfig.tradingConfig
    │     IF volume < tradingConfig.minVolume → REJECT "Volume too small"
    │     IF volume > tradingConfig.maxVolume → REJECT "Volume too large"
    │     IF (volume - minVolume) % volumeStep != 0 → REJECT "Invalid volume step"
    │     // 品种组覆盖检查
    │     IF symbolCategoryConfig.minLot != NULL:
    │         IF volume < symbolCategoryConfig.minLot → REJECT
    │     (对应表: t_symbol_trading_config, t_account_group_symbol_config)
    │
    ├── 订单类型检查
    │     IF orderType == LIMIT AND tradingConfig.allowLimitOrder == false → REJECT
    │     IF orderType == STOP AND tradingConfig.allowStopOrder == false → REJECT
    │     (对应表: t_symbol_trading_config.allow_*_order)
    │
    ├── 挂单价格距离检查
    │     IF 挂单类型:
    │         distance = |requestedPrice - currentPrice| / tickSize
    │         IF distance < tradingConfig.stopLevel:
    │             → REJECT "Price too close to market"
    │     (对应表: t_symbol_trading_config.stop_level)
    │
    └── 止损止盈距离检查
          IF priceSl != NULL:
              slDistance = |priceSl - expectedFillPrice| / tickSize
              IF slDistance < tradingConfig.stopLevel:
                  → REJECT "SL too close"
          (同上)
    │
    ▼
5.4 持仓限制检查(纯内存，< 0.1ms)
    │
    ├── 持仓数量
    │     IF state.positions.size() >= groupConfig.maxPositions:
    │         → REJECT "Maximum positions reached"
    │     (对应表: t_account_group.max_positions)
    │
    ├── 挂单数量(如果是挂单)
    │     IF state.pendingOrders.size() >= groupConfig.maxPendingOrders:
    │         → REJECT "Maximum pending orders reached"
    │
    ├── 单品种持仓手数
    │     IF tradingConfig.maxPositionVolume != NULL:
    │         sameSymbolSameDirection = SUM(pos.volume
    │             for pos in state.positions.values()
    │             where pos.symbolKey == symbolKey AND pos.side == side)
    │         IF sameSymbolSameDirection + volume > maxPositionVolume:
    │             → REJECT "Maximum position volume reached"
    │     (对应表: t_symbol_trading_config.max_position_volume)
    │
    └── 总手数
          IF groupConfig.maxLots != NULL:
              totalVolume = SUM(pos.volume for all positions)
              IF totalVolume + volume > maxLots:
                  → REJECT
          (对应表: t_account_group.max_lots)
    │
    ▼
5.5 保证金计算与验证(纯内存，< 0.5ms)
    │
    ├── 计算本单所需保证金
    │     marginRequired = calcMargin(
    │         symbolConfig,
    │         volume,
    │         expectedFillPrice,  // 市价=currentAsk/Bid, 限价=requestedPrice
    │         state.leverage,
    │         groupConfig.leverageTemplateId,  // 如果有浮动杠杆
    │         leverageTiers,
    │         symbolCategoryConfig.maxLeverageOverride
    │     )
    │     (对应表: t_symbol, t_trading_account, t_account_group,
    │      t_leverage_template, t_leverage_template_tier,
    │      t_account_group_symbol_config)
    │
    │     // 完整的 calcMargin 逻辑见风控系统设计
    │
    ├── 对冲保证金折扣(如果有反向持仓)
    │     IF groupConfig.allowHedging == 1:
    │         oppositeVolume = SUM(pos.volume
    │             for pos in state.positions.values()
    │             where pos.symbolKey == symbolKey AND pos.side == oppositeSide)
    │         IF oppositeVolume > 0:
    │             hedgedVolume = MIN(volume, oppositeVolume)
    │             unhedgedVolume = volume - hedgedVolume
    │             marginRequired = calcMargin(unhedgedVolume, ...)
    │                            + calcMargin(hedgedVolume, ...) × marginHedgedRatio
    │     (对应表: t_account_group.allow_hedging, t_symbol.margin_hedged)
    │
    └── 余额充足性验证
          marginFreeAfter = state.equity - state.marginUsed - marginRequired
          IF marginFreeAfter < 0:
              → REJECT "Not enough margin"

          marginLevelAfter = state.equity / (state.marginUsed + marginRequired) × 100
          IF marginLevelAfter < groupConfig.marginCallLevel:
              → REJECT "Order would trigger margin call"
          (对应表: t_account_group.margin_call_level)
    │
    ▼
5.6 自定义风控规则检查(纯内存，< 0.5ms)
    │
    FOR each rule in riskRuleSet (按priority DESC):
    │
    │   IF rule匹配当前订单(scope/symbol/side/orderType):
    │
    │       SWITCH rule.ruleType:
    │
    │           MAX_SINGLE_ORDER_VALUE:
    │               notionalValue = volume × contractSize × price × rate(→USD)
    │               IF notionalValue > rule.thresholdValue → 触发
    │
    │           ORDER_FREQUENCY_LIMIT:
    │               recentCount = Redis ZCOUNT risk:order_rate:{accountId}
    │                             (now - rule.thresholdSeconds*1000, now)
    │               IF recentCount >= rule.thresholdCount → 触发
    │
    │           TIME_WINDOW_RESTRICTION:
    │               IF 当前时间在 rule.timeWindowStart~End
    │                  AND 当天是 rule.timeWindowWeekdays 中的某天 → 触发
    │
    │           NEW_ACCOUNT_RESTRICTION:
    │               accountAgeDays = (now - state.openedAt).days
    │               IF accountAgeDays < rule.thresholdDays → 触发
    │
    │           CORRELATED_EXPOSURE:
    │               查 t_risk_symbol_correlation
    │               合并同相关性组的敞口
    │               IF 合并敞口 > thresholdValue → 触发
    │
    │       触发后按 rule.actionType:
    │           1-REJECT → REJECT
    │           2-DEALER_REVIEW → 订单挂起等Dealer审批
    │           3-ALERT → 不阻断，发告警
    │
    │   → 写 t_risk_event_log (异步)
    │   (对应表: t_risk_rule, t_risk_event_log, t_risk_symbol_correlation)
    │
    ▼
5.7 生成订单记录（预交易已通过；SUBMITTED/VALIDATED 不落库）
    │
    ticket_id = snowflake.nextId()
    
    // 写入内存（运行时权威）
    orderState = OrderState{ clientOrderId, requestId, ... }
    IF 挂单类型:
        state.pendingOrders[ticket_id] = orderState
        book.addPendingOrder(orderState)   // 仅内存索引供 Tick 触发
    
    // WAL / 异步投影（持久态见《交易系统设计》）
    IF 市价单:
        → 投影 t_order status=WORKING
        → 直接进入 PHASE 6 (执行)
    IF 挂单:
        → 投影 t_order status=PLACED，记录 margin_reserved
        → 等待价格触发（PHASE 2，只扫 SymbolBook）
    
    → Outbox: order.accepted.v1
    → 写 t_risk_event_log (PRE_TRADE_PASS) 异步
    → Redis: ZADD risk:order_rate:{accountId} {now} {ticketId}

 PHASE 6: 订单执行（成交）
═══════════════════════════════════════════════════════════
    │
    ▼
6.1 路由决策
    │
    │  遍历 routeRuleSet (按priority DESC):
    │  找到第一条匹配的规则
    │
    │  匹配条件(AND关系):
    │    rule.matchSymbolCategory == symbol.category (或NULL)
    │    rule.matchSymbolGroupId == symbol.groupId (或NULL)
    │    rule.matchSymbolId == symbol.id (或NULL)
    │    rule.matchAccountGroupId == state.groupId (或NULL)
    │    rule.matchVolumeMin <= volume (或NULL)
    │    rule.matchVolumeMax >= volume (或NULL)
    │  (对应表: t_route_rule)
    │
    │  执行模式决定:
    │    executionMode = rule.executionMode
    │    1-A_BOOK: 发到LP
    │    2-B_BOOK: 内部消化
    │    3-MIXED: 按abook_ratio分配
    │
    ▼
6.2A A-Book执行（STP到LP）
    │
    ├── 订单投影 status=PENDING_EXEC
    │
    ├── 选择网关
    │     gatewayId = rule.targetGatewayId
    │     IF gateway不可用: gatewayId = rule.backupGatewayId
    │     IF 都不可用: → REJECT "No liquidity available"
    │
    ├── 转换为上游订单
    │     mapping = gateway.symbolMappings[symbolKey]
    │     remoteSymbol = mapping.remoteSymbol
    │     remoteVolume = volume × mapping.volumeScaleFactor
    │     // 发送FIX/REST订单到LP
    │     → 写 t_execution_link(order_ticket, gateway_id, external_order_id, ...)
    │
    ├── 等待LP回复（异步投递回 EventLoop，超时=requestTimeoutMs）
    │     LP回复: fillPrice, fillVolume
    │     → 更新 t_execution_link.external_status / filled_volume
    │     
    │     IF LP拒绝:
    │         尝试备用网关
    │         IF 都失败: → REJECT "Execution failed"（无成交时）
    │
    ├── 滑点检查
    │     slippage = |fillPrice - requestedPrice| / tickSize
    │     IF slippage > rule.maxSlippagePoints (或 tradingConfig.maxNegativeSlippage):
    │         IF slippage对客户不利:
    │             → REJECT "Excessive slippage"
    │         // 对客户有利的滑点(价格改善) → 允许
    │
    └── fillPrice 确定 → 进入6.3
    │
    ▼
6.2B B-Book执行（内部消化）
    │
    ├── 价格确定（必须用 Core Book，禁止 Client Quote）
    │     IF book.stale → REJECT "Quote stale"
    │     IF 买单: fillPrice = book.currentAsk   // Core Ask
    │     IF 卖单: fillPrice = book.currentBid   // Core Bid
    │
    ├── 滑点模拟（如果配置了executionDelayMs）
    │     IF rule.executionDelayMs > 0:
    │         sleep(executionDelayMs)
    │         // 延迟后重新取价
    │         fillPrice = book.currentAsk/Bid (延迟后的最新价)
    │
    ├── 滑点检查
    │     IF order.maxDeviation != NULL:
    │         deviation = |fillPrice - requestedPrice| / tickSize
    │         IF deviation > maxDeviation:
    │             → REJECT "Price deviation exceeded"
    │
    └── fillPrice 确定 → 进入6.3
    │
    ▼
6.2C Mixed执行（混合模式）
    │
    │  abookVolume = volume × (rule.abookRatio / 100)
    │  bbookVolume = volume - abookVolume
    │  
    │  // A-Book部分走6.2A
    │  // B-Book部分走6.2B
    │  // 对客户而言只看到一个统一的成交
    │  fillPrice = 取A-Book的成交价（对客户透明）
    │
    ▼
6.3 成交处理（最关键的原子操作）
    │
    │  ※ 以下所有步骤在同一个逻辑事务中完成
    │  ※ 先在内存中更新，再异步写数据库
    │
    ├── 6.3.1 生成Deal
    │     dealTicket = snowflake.nextId()
    │     
    │     IF 开仓:
    │         deal = Deal{
    │             dealTicket, orderTicket, accountId, symbolKey
    │             dealType: BUY 或 SELL
    │             entry: "IN"
    │             price: fillPrice
    │             volume: fillVolume
    │             profit: 0  // 开仓无盈亏
    │             commission: calcCommission(...)  // 见6.3.5
    │         }
    │     IF 平仓:
    │         deal = Deal{
    │             dealType: BUY_CLOSE 或 SELL_CLOSE
    │             entry: "OUT"
    │             price: fillPrice
    │             volume: fillVolume
    │             profit: calcProfit(...)  // 见6.3.4
    │             commission: calcCommission(...)
    │         }
    │
    ├── 6.3.2 更新/创建持仓（内存）
    │     
    │     IF 开仓(entry=IN):
    │         IF Hedging模式 OR 无同品种同方向持仓:
    │             // 新建持仓
    │             posTicket = snowflake.nextId()
    │             position = PositionState{
    │                 positionTicket: posTicket
    │                 symbolKey, side: order.side
    │                 volume: fillVolume
    │                 volumeInitial: fillVolume
    │                 priceOpen: fillPrice
    │                 priceSl: order.priceSl
    │                 priceTp: order.priceTp
    │                 swap: 0
    │                 margin: marginRequired  // 之前算好的
    │                 openedAt: now
    │             }
    │             state.positions[posTicket] = position
    │             book.activePositions.add(position)
    │             deal.positionId = posTicket
    │
    │         IF Netting模式 AND 有同品种同方向持仓:
    │             // 合并到已有持仓
    │             existing = 找到已有持仓
    │             newAvgPrice = (existing.priceOpen × existing.volume
    │                         + fillPrice × fillVolume)
    │                         / (existing.volume + fillVolume)
    │             existing.priceOpen = newAvgPrice
    │             existing.volume += fillVolume
    │             existing.margin = recalcMargin(existing)
    │             deal.positionId = existing.positionTicket
    │
    │     IF 平仓(entry=OUT):
    │         targetPosition = state.positions[order.closePositionId]
    │         targetPosition.volume -= fillVolume
    │         
    │         IF targetPosition.volume <= 0:
    │             // 全部平仓 → 移除
    │             state.positions.remove(targetPosition.positionTicket)
    │             book.activePositions.remove(targetPosition)
    │             // 标记需要移至 t_position_history
    │         ELSE:
    │             // 部分平仓 → 更新保证金
    │             targetPosition.margin = recalcMargin(targetPosition)
    │         
    │         deal.positionId = targetPosition.positionTicket
    │
    ├── 6.3.3 更新账户状态（内存）
    │     
    │     // 保证金更新
    │     state.marginUsed = SUM(pos.margin for all positions)
    │     
    │     // 余额更新(平仓盈亏+佣金)
    │     IF 平仓:
    │         state.balance += deal.profit
    │         state.balance += deal.commission  // 负值=扣费
    │     IF 开仓 AND 佣金在开仓时收:
    │         state.balance += deal.commission
    │     
    │     // 重算净值
    │     state.floatingPnl = SUM(pos.profit for all positions)
    │     state.equity = state.balance + state.credit + state.floatingPnl
    │     state.marginFree = state.equity - state.marginUsed
    │
    ├── 6.3.4 盈亏计算(平仓时)
    │     
    │     calcProfit(position, fillPrice, fillVolume):
    │         IF position.side == BUY:
    │             priceDiff = fillPrice - position.priceOpen
    │         ELSE:
    │             priceDiff = position.priceOpen - fillPrice
    │         
    │         profit = priceDiff × fillVolume
    │                × symbolConfig.contractSize
    │                × exchangeRate(profitCurrency → accountCurrency)
    │         
    │         // 汇率来源:
    │         // 如EURUSD盈亏货币是USD=账户币种，汇率=1
    │         // 如EURJPY盈亏货币是JPY，需要×USDJPY汇率(取当前报价)
    │         
    │         RETURN profit
    │
    ├── 6.3.5 佣金计算
    │     
    │     calcCommission(accountGroupConfig, symbolConfig, deal):
    │         commRule = accountGroupConfig.commissions
    │                    .find(match symbol.category)
    │         (对应表: t_account_group_commission)
    │         
    │         SWITCH commRule.chargeType:
    │             1-PER_LOT:
    │                 commission = deal.volume × commRule.chargeValue
    │             2-PER_DEAL:
    │                 commission = commRule.chargeValue
    │             3-BY_VALUE:
    │                 dealValue = deal.volume × contractSize × fillPrice
    │                 commission = dealValue × commRule.chargeValue
    │             4-BY_POINTS:
    │                 // 按点差收
    │         
    │         // 收取时机
    │         IF commRule.chargeMoment == 1:  // 开仓全额
    │             IF entry == IN: 收全额
    │             ELSE: 不收
    │         IF commRule.chargeMoment == 2:  // 各半
    │             commission = commission / 2
    │         IF commRule.chargeMoment == 3:  // 平仓全额
    │             IF entry == OUT: 收全额
    │             ELSE: 不收
    │         
    │         // 上下限
    │         IF commRule.minCharge != NULL:
    │             commission = MAX(commission, minCharge)
    │         IF commRule.maxCharge != NULL:
    │             commission = MIN(commission, maxCharge)
    │         
    │         RETURN -commission  // 负值=从账户扣除
    │
    └── 6.3.6 IB佣金计提（异步）
          
          // 查询客户的IB归属链
          ibRelation = ibClientRelations[state.accountId]
          (对应表: t_ib_client_relation)
          
          IF ibRelation != NULL:
              ibChain = ibRelation.ibChain  // [directIbId, parentIbId, rootIbId]
              
              FOR each ibId in ibChain:
                  contract = ibCommissionContracts[ibId]
                      .find(match symbol.category, effective today)
                  (对应表: t_ib_commission_contract)
                  
                  ibCommission = calcIbCommission(contract, deal)
                  
                  // 上级实得 = 自己的费率计算结果 - 下级的费率计算结果
                  IF ibId不是直属IB:
                      ibCommission = ibCommission - childIbCommission
                  
                  → 异步写 t_ib_commission_accrual
    │
    ▼
6.4 持久化（异步批量写入，不阻塞主线程）
    │
    │  ※ 这些操作通过事件队列异步执行
    │  ※ 引擎内存是"真相"，数据库是"持久化副本"
    │  ※ 如果引擎崩溃，从数据库恢复到最后持久化的状态
    │
    ├── 写 t_deal (INSERT，不可变；含 book_seq/core_bid/core_ask 审计字段)
    │
    ├── 更新 t_order
    │     市价单全成: status=FILLED → 移至 t_order_history
    │     部分成交: volumeFilled更新, status=PARTIALLY_FILLED（仍在活跃表）
    │
    ├── 写/更新 t_position
    │     开仓: INSERT
    │     部分平仓: UPDATE volume/margin
    │     全部平仓: DELETE + INSERT t_position_history
    │
    ├── 写 t_account_ledger（结构见《资金系统设计》；引擎写交易类型）
    │     盈亏: ledger_type=3 (TRADE_PROFIT)
    │     佣金: ledger_type=4 (COMMISSION)
    │     idempotency_key = request_id 派生键；禁止编排层直写
    │
    ├── 更新 t_trading_account 资金快照
    │     balance, equity, margin_used, margin_free, floating_pnl
    │     last_trade_at
    │
    ├── 同提交写 t_engine_outbox
    │     deal.created.v1 / order.filled.v1 或 order.partially_filled.v1
    │     position.opened.v1 或 position.updated.v1 / position.closed.v1
    │     （Partner 佣金由代理体系消费 deal.*，引擎不直写 IB 账本）
    │
    ├── 更新 Redis 投影
    │     fund:available:{accountId}
    │     risk:margin:{accountId}
    │
    ├── 写 t_risk_event_log (执行结果) 异步
    │
    └── 发 MQ 事件
          → deal.created (IB佣金服务消费)
          → order.filled (通知系统消费 → 推送给用户)
          → position.opened / position.closed (其他消费者)
```

## 五、核心链路3：强平流程

```
═══════════════════════════════════════════════════════════
 PHASE 7: 强制平仓 (Stop Out)
═══════════════════════════════════════════════════════════

触发条件: PHASE 4 检测到 marginLevel ≤ stopOutLevel
    │
    ▼
7.1 创建强平执行记录
    executionNo = snowflake.nextId()
    
    → 异步写 t_risk_stop_out_execution (status=执行中)
    → 异步写 t_risk_event_log (STOP_OUT_TRIGGERED)
    │
    ▼
7.2 确定强平策略
    stopOutMode = groupConfig.stopOutMode
    
    MODE 1 - 亏损最大优先:
        candidates = state.positions.values()
            .filter(pos.profit < 0)        // 只平亏损的
            .sortBy(pos.profit ASC)         // 亏损最大的排前面
        IF candidates.isEmpty():
            // 所有持仓都盈利但margin_level仍不够
            // 按保证金占用最大排序
            candidates = state.positions.values()
                .sortBy(pos.margin DESC)
    
    MODE 2 - 保证金最大优先:
        candidates = state.positions.values()
            .sortBy(pos.margin DESC)
    
    MODE 3 - Smart Stop Out (部分平仓):
        // 计算需要释放多少保证金
        equityNeeded = state.marginUsed × (stopOutLevel / 100) + buffer
        marginToRelease = state.marginUsed - (state.equity / stopOutLevel × 100)
        
        candidates = state.positions.values()
            .sortBy(pos.margin DESC)  // 从最大持仓开始
    │
    ▼
7.3 逐个平仓循环
    │
    sequence = 0
    
    WHILE state.marginLevel <= groupConfig.stopOutLevel
          AND candidates.hasNext():
        
        pos = candidates.next()
        sequence++
        
        // 确定平仓手数
        IF stopOutMode == 3 (Smart):
            // 计算部分平仓手数
            marginPerLot = pos.margin / pos.volume
            volumeToClose = CEIL(marginToRelease / marginPerLot
                                 / symbolConfig.volumeStep) × volumeStep
            volumeToClose = MIN(volumeToClose, pos.volume)
            volumeToClose = MAX(volumeToClose, symbolConfig.minVolume)
        ELSE:
            volumeToClose = pos.volume  // 全平
        
        // 确定平仓价格
        IF pos.side == BUY:
            closePrice = symbolBooks[pos.symbolKey].currentBid
        ELSE:
            closePrice = symbolBooks[pos.symbolKey].currentAsk
        
        IF closePrice == NULL OR closePrice <= 0:
            // 该品种当前无有效报价（休市/异常）
            → 记录失败, 跳过此持仓, 尝试下一个
            → 写 t_risk_stop_out_detail (status=失败)
            CONTINUE
        
        // 生成系统强平订单
        systemOrder = OrderRequest{
            accountId: state.accountId
            symbolKey: pos.symbolKey
            orderType: SELL (如果pos是BUY) 或 BUY (如果pos是SELL)
            volume: volumeToClose
            closePositionId: pos.positionTicket
            source: 'SYSTEM'
            comment: "Stop Out: ML={marginLevel}%"
        }
        
        // 强平订单跳过 Pre-Trade 风控检查（PHASE 5）
        // 直接进入 PHASE 6 执行
        result = executeOrder(systemOrder, skipRiskCheck=true)
        
        IF result.success:
            // 更新强平明细
            → 写 t_risk_stop_out_detail{
                executionId, sequence
                positionTicket, symbolKey, side
                volumeBefore: pos.volume + volumeToClose
                volumeClosed: volumeToClose
                volumeRemaining: pos.volume  // 内存中已更新
                priceOpen: pos.priceOpen
                priceClose: closePrice
                profit: result.deal.profit
                marginReleased: result.marginReleased
                marginLevelAfter: state.marginLevel
                status: 成功
            }
            → 写 t_risk_event_log (STOP_OUT_POSITION_CLOSED)
            
            // 重新计算marginLevel（PHASE 6.3.3已更新）
            // 检查是否恢复到安全水平
        ELSE:
            → 写 t_risk_stop_out_detail (status=失败, failReason)
    │
    ▼
7.4 强平结束
    │
    ├── 正常结束: marginLevel > stopOutLevel
    │     → 更新 t_risk_stop_out_execution{
    │         status: 已完成
    │         positionsClosed, volumeClosed, realizedPnl
    │         finalMarginLevel, finalEquity, finalBalance
    │         completedAt, durationMs
    │     }
    │     → 写 t_risk_event_log (STOP_OUT_COMPLETED)
    │
    ├── 部分完成: 还有持仓无法平(无报价)
    │     → status: 部分完成
    │     → 告警管理员
    │
    └── 负余额处理
          IF state.balance < 0
             AND groupConfig.negativeBalanceProtection == 1:
              
              resetAmount = ABS(state.balance)
              state.balance = 0
              
              // 写 Ledger: 补偿到零
              → t_account_ledger{
                  ledgerType: 8 (ADJUSTMENT)
                  direction: 1
                  amount: resetAmount
                  remark: "Negative balance protection"
              }
              
              → 更新 t_risk_stop_out_execution{
                  negativeBalanceOccurred: 1
                  negativeBalanceAmount: -resetAmount
                  negativeBalanceReset: 1
                  resetLedgerId: ledger.id
              }
              → 写 t_risk_event_log (NEGATIVE_BALANCE_RESET)
              → 发 MQ: NEGATIVE_BALANCE_RESET → 通知系统
    │
    ▼
    → 发 MQ: STOP_OUT_EXECUTED → 通知系统
       用户收到通知: "您的账户已触发强制平仓，
                     平仓{N}笔，已实现盈亏{profit}"
```

## 六、核心链路4：每日结算

```
═══════════════════════════════════════════════════════════
 PHASE 8: 日终结算 (End-of-Day Settlement)
═══════════════════════════════════════════════════════════

触发: 每日服务器时间00:00:00（或配置的结算时间）
      由定时任务触发，但在引擎进程内执行

    │
    ▼
8.1 Swap(隔夜利息)结算
    │
    │  今天星期几 → dayOfWeek
    │
    │  FOR each accountState in accountStates.values():
    │      FOR each position in accountState.positions.values():
    │          
    │          // 检查是否免Swap
    │          IF position.swapFree == true: CONTINUE
    │          
    │          // 获取Swap配置
    │          swapConfig = symbolConfigs[position.symbolKey].swapConfig
    │          (对应表: t_symbol_swap_config)
    │          
    │          IF swapConfig.swapMode == 0: CONTINUE  // 该品种禁用Swap
    │          
    │          // 获取今日系数
    │          coeff = swapConfig.swapCoeffs[dayOfWeek]
    │          // 如周三=3.0(外汇T+2结算含周末)
    │          IF coeff == 0: CONTINUE
    │          
    │          // 获取多空Swap值
    │          IF position.side == BUY:
    │              swapValue = swapConfig.swapLong
    │          ELSE:
    │              swapValue = swapConfig.swapShort
    │          
    │          // 按模式计算
    │          SWITCH swapConfig.swapMode:
    │              1-按点数:
    │                  swapAmount = swapValue × coeff
    │                            × symbolConfig.tickValue
    │                            × position.volume
    │                            × exchangeRate(profitCurrency → accountCurrency)
    │              2-按金额:
    │                  swapAmount = swapValue × coeff × position.volume
    │              3-按年化利率:
    │                  notional = position.volume × contractSize × position.priceOpen
    │                  swapAmount = notional × (swapValue/100) / 365 × coeff
    │                            × exchangeRate(marginCurrency → accountCurrency)
    │              4-按保证金比例:
    │                  swapAmount = position.margin × (swapValue/100) × coeff
    │          
    │          // 更新内存
    │          position.swap += swapAmount
    │          accountState.balance += swapAmount  // 正=收入，负=支出
    │          
    │          // 异步写 Ledger
    │          → t_account_ledger{
    │              ledgerType: 5 (SWAP)
    │              direction: swapAmount > 0 ? 1 : -1
    │              amount: ABS(swapAmount)
    │              positionTicket: position.positionTicket
    │          }
    │          
    │          // 异步更新持仓
    │          → UPDATE t_position SET swap = position.swap
    │
    ▼
8.2 挂单过期清理
    │
    │  FOR each accountState:
    │      FOR each order in accountState.pendingOrders.values():
    │          IF order.expirationType == DAY:
    │              // 当日有效，日终过期
    │              → 移除挂单
    │              → t_order → t_order_history (status=EXPIRED)
    │          IF order.expirationType == SPECIFIED
    │             AND order.expireAt <= now:
    │              → 同上
    │
    ▼
8.3 模拟账户过期处理
    │
    │  FOR each accountState:
    │      IF accountState.environment == DEMO
    │         AND accountState.demoExpireAt != NULL
    │         AND accountState.demoExpireAt <= now:
    │          → 平掉所有持仓
    │          → 取消所有挂单
    │          → accountState.status = ARCHIVED
    │          → UPDATE t_trading_account status=4
    │
    ▼
8.4 账户快照（风控日报）
    │
    │  FOR each accountState:
    │      → INSERT t_risk_account_snapshot{
    │          snapshotDate: today
    │          所有资金/保证金/持仓/交易指标
    │          risk_level: 根据marginLevel评级
    │          top1/2/3 持仓品种和占比
    │      }
    │  (对应表: t_risk_account_snapshot)
    │
    ▼
8.5 快照同步到数据库
    │
    │  批量更新 t_trading_account:
    │      balance, equity, margin_used, margin_free, floating_pnl
    │  批量更新 t_position:
    │      price_current, profit, swap, margin
    │
    ▼
8.6 大额交易扫描
    │
    │  查当日 t_deal:
    │      按账户汇总成交金额
    │      IF 某账户当日总成交 > CTR阈值($10,000):
    │          → INSERT t_risk_large_exposure_report
    │  (对应表: t_risk_large_exposure_report)
    │
    ▼
8.7 IB佣金汇总结算
    │
    │  查 t_ib_commission_accrual (status=ACCRUED, accrual_date=yesterday)
    │  按 ib_id + currency 分组汇总
    │  → INSERT t_ib_settlement
    │  → 审核通过后 → 更新 t_ib.wallet_balance + 写 t_ib_wallet_ledger
    │  (对应表: t_ib_commission_accrual, t_ib_settlement, t_ib_wallet_ledger)
    │
    ▼
8.8 Ledger完整性校验
    │
    │  执行三个校验SQL(上一次设计的):
    │  ① 余额一致性
    │  ② 链式连续性
    │  ③ 累加验证
    │  异常 → 告警
    │
    ▼
日终结算完毕
```

## 七、引擎内存数据结构（伪代码）

```java
// ===================================================================
// 核心数据结构（全部常驻内存，运行时不查数据库）
// ===================================================================

class TradingEngine {
    
    // ---------- 品种 ----------
    Map<String, SymbolBook> symbolBooks;        // key=symbolKey
    Map<String, SymbolConfig> symbolConfigs;     // 品种配置
    
    // ---------- 账户 ----------
    Map<Long, AccountState> accountStates;       // key=accountId
    Map<Long, AccountGroupConfig> groupConfigs;  // key=groupId
    Map<Long, LeverageTier[]> leverageTiers;     // key=templateId
    
    // ---------- 风控 ----------
    List<RiskRule> riskRules;                    // 按priority排序
    List<RouteRule> routeRules;                  // 按priority排序
    
    // ---------- IB ----------
    Map<Long, IbClientRelation> ibRelations;     // key=accountId
    Map<Long, List<IbCommContract>> ibContracts; // key=ibId
    
    // ---------- 网关 ----------
    Map<Long, GatewayConnection> gateways;       // key=gatewayId
    
    // ---------- 时段 ----------
    SessionManager sessionManager;               // 交易时段管理
}

class SymbolBook {
    String symbolKey;
    
    // 当前报价
    Decimal currentBid, previousBid;
    Decimal currentAsk, previousAsk;
    Decimal currentLast;
    int spreadPoints;
    long tickTimeMs;
    int tickDirection;   // 1=up, -1=down, 0=flat
    
    // 挂单簿（该品种所有账户的挂单）
    TreeSet<OrderState> buyLimitOrders;   // 按price DESC
    TreeSet<OrderState> sellLimitOrders;  // 按price ASC
    TreeSet<OrderState> buyStopOrders;    // 按trigger ASC
    TreeSet<OrderState> sellStopOrders;   // 按trigger DESC
    
    // 活跃持仓引用（该品种所有账户的持仓）
    List<PositionState> activePositions;
    
    // 价格预警
    List<PriceAlert> priceAlerts;
}

class AccountState {
    long accountId;
    String accountNo;
    long groupId;
    String currency;
    int leverage;
    boolean swapFree;
    int environment;     // 1=Live, 2=Demo
    
    // 资金（实时精确值）
    Decimal balance;     // 余额
    Decimal credit;      // 信用额
    Decimal equity;      // 净值 = balance + credit + floatingPnl
    Decimal floatingPnl; // 浮动盈亏
    Decimal marginUsed;  // 已用保证金
    Decimal marginFree;  // 可用保证金 = equity - marginUsed
    Decimal marginLevel; // 保证金水平(%) = equity/marginUsed×100
    Decimal frozenAmount;// 冻结金额（出金冻结）
    
    // 状态
    boolean tradeEnabled;
    boolean expertEnabled;
    boolean marginCallActive;
    
    // 持仓与挂单（该账户的）
    Map<Long, PositionState> positions;      // key=positionTicket
    Map<Long, OrderState> pendingOrders;     // key=orderTicket
}

class PositionState {
    long positionTicket;
    long accountId;       // 反向引用
    String symbolKey;
    int side;             // 1=BUY, 2=SELL
    Decimal volume;
    Decimal volumeInitial;
    Decimal priceOpen;
    Decimal priceSl, priceTp;
    Decimal priceCurrentSnapshot;
    Decimal profit;       // 浮动盈亏（每Tick更新）
    Decimal swap;         // 累计Swap
    Decimal margin;       // 占用保证金
    boolean swapFree;
    long openedAtMs;
}

class OrderState {
    long orderTicket;
    long accountId;
    String symbolKey;
    int orderType;        // 1-8
    int side;             // 1=BUY, 2=SELL
    Decimal priceRequested;
    Decimal priceTrigger; // Stop类型
    Decimal priceSl, priceTp;
    Decimal volumeRequested;
    Decimal volumeFilled;
    Decimal volumeRemaining;
    int expirationType;
    long expireAtMs;
    int fillingMode;
    int status;
}
```
