⬇️ 三级降级策略
🟢 正常模式 (PDP 在线)
L2 Redis 读取权限快照，命中率 > 95%
实时 RBAC + ABAC + SoD 全套决策
P99 < 20ms
⬇️ PDP 错误率 > 5% 或 P99 > 100ms 持续 30s
🟡 L1 本地缓存模式
使用 Caffeine 本地缓存（TTL 30s）
仅校验 RBAC，跳过 ABAC/风险评分
标记 DEGRADED_ALLOW · T+1 合规补偿审查
⬇️ L1 也失效或连续 1 分钟未刷新
🔴 保守白名单模式
仅放行"必须可用操作"（登录、查看订单/持仓）
所有写操作、大额、敏感数据 → 全部拒绝
触发 P0 告警 · SRE 5 分钟内响应
🔒 核心原则：高风险操作（出金、大额交易、敏感数据导出、权限变更）降级期间必须全部拒绝，宁可业务中断不可权限失控
⏱️ 鉴权请求分级 SLA
优先级 典型场景 延迟 SLA 降级策略 缓存策略
P0 关键 交易下单前鉴权 · 出金审批 P99 < 20ms 降级必拒绝 L1 本地缓存必命中
P1 重要 业务 API 鉴权 · 客户数据访问 P99 < 30ms 降级允许 RBAC L1 + L2 双级缓存
P2 普通 菜单/按钮可见性 · 报表查询 P99 < 50ms 降级允许 L2 Redis 缓存
P3 低优 管理后台操作 · 配置变更 P99 < 200ms 降级直通 直读 DB
📌 性能约束：单用户最多 20 个角色 · 继承深度上限 3 层 · 单角色最多 500 权限点 · 快照大小上限 64KB
