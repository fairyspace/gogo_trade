
# 交易系统用户体系设计稿

## 一、架构设计理念

用户体系作为交易系统的**身份承载、认证授权与权限映射**核心基座，不承担 KYC 审核等重业务。核心原则：

1. **多监管实体隔离**：用户归属某 `entity_id`，决定其杠杆上限、负余额保护等规则；同一自然人可在不同实体（牌照品牌）下各注册独立账号。
2. **适当性快照**：KYC 在 Onboarding 完成，交易系统单向缓存适当性结果，避免高频交易时的跨服务 RPC。
3. **IB 溯源解耦**：主表固化 IB ID 作为分润基准，代理树与分润计算交由独立代理系统。
4. **多态凭证与 OAuth 分离**：密码 / 2FA / API Key 抽象为统一凭证表；OAuth 独立建表，遵循“授权”语义。
5. **软硬风控分离**：硬风控交由独立风控引擎，用户表仅保留“单日止损”等软风控偏好。

## 二、表清单（9 张）

```text
身份与业务基座 (3)
├── t_user                    用户主表 (身份、实体、IB溯源、全局状态、安全控制)
├── t_user_profile_snapshot   适当性快照 (从Onboarding单向同步)
└── t_user_preference         交易偏好 (UI偏好、软风控止损)

认证与授权 (3)
├── t_user_credential         多态凭证 (密码/TOTP/API Key/Passkey)
├── t_user_oauth              第三方授权绑定 (Google/Apple/微信)
└── t_user_device             受信设备 (指纹与信任管理)

运营与分群 (1)
└── t_user_tag_relation       用户标签关系 (VIP/高频/高风险)

审计日志 (2，按季度分区)
├── t_user_login_log          登录与安全事件日志
└── t_user_operation_log      业务操作审计日志
```

## 三、核心 SQL 设计

### 1. 用户主表 `t_user`

> 纯身份主体。`email`/`phone` 按 `entity_id` 隔离唯一；`security_stamp` 用于令牌失效；`status` 为权威生命周期，`is_deleted` 仅作技术软删标记。注销时 `email`/`phone` 改写为匿名占位值以释放复用。

```sql
CREATE TABLE `t_user` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_uid`              VARCHAR(32)         NOT NULL COMMENT '对外业务唯一ID(雪花算法)',
    `entity_id`             BIGINT UNSIGNED     NOT NULL DEFAULT 1
        COMMENT '所属监管实体ID(关联Onboarding,决定监管规则与账户组权限)',
    `introducing_broker_id` BIGINT UNSIGNED     DEFAULT NULL
        COMMENT '代理IB ID(冗余快照,注册时写入;代理树与分润由独立代理系统负责)',

    -- ==================== 登录标识 ====================
    `email`                 VARCHAR(128)        DEFAULT NULL COMMENT '邮箱(注销时改写为匿名值以释放复用)',
    `phone_country_code`    VARCHAR(8)          DEFAULT NULL COMMENT '手机国际区号',
    `phone_number`          VARCHAR(32)         DEFAULT NULL COMMENT '手机号码(注销时改写为匿名值以释放复用)',

    -- ==================== 身份分类 ====================
    `user_type`             TINYINT UNSIGNED    NOT NULL DEFAULT 1
        COMMENT '法律主体类型: 1-个人 2-企业/机构',

    -- ==================== 全局状态 ====================
    `status`                TINYINT UNSIGNED    NOT NULL DEFAULT 0
        COMMENT '全局状态(权威生命周期): 0-未激活 1-正常 2-全局冻结 3-注销冷静期 4-已注销',
    `kyc_level`             TINYINT UNSIGNED    NOT NULL DEFAULT 0
        COMMENT 'KYC等级(Onboarding同步): 0-未认证 1-L1 2-L2 3-L3',

    -- ==================== 安全控制 ====================
    `mfa_enabled`           TINYINT UNSIGNED    NOT NULL DEFAULT 0
        COMMENT '是否开启2FA(总开关,供网关快速判断): 0-否 1-是',
    `security_stamp`        VARCHAR(64)         DEFAULT NULL
        COMMENT '安全戳:改密码/冻结/踢设备时刷新,使历史Token立即失效(嵌入JWT校验)',

    -- ==================== 注册溯源 ====================
    `register_channel`      VARCHAR(16)         DEFAULT NULL COMMENT '注册渠道: WEB/IOS/ANDROID/API/ADMIN/OAUTH',
    `register_ip`           VARCHAR(45)         DEFAULT NULL COMMENT '注册IP(兼容IPv6)',

    `last_login_at`         DATETIME(3)         DEFAULT NULL COMMENT '最近登录时间(异步更新)',
    `is_deleted`            TINYINT UNSIGNED    NOT NULL DEFAULT 0 COMMENT '技术软删标记(与status=4联动)',
    `version`               INT UNSIGNED        NOT NULL DEFAULT 0 COMMENT '乐观锁版本号',
    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '注册时间',
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_uid` (`user_uid`),
    UNIQUE KEY `uk_entity_email` (`entity_id`, `email`),
    UNIQUE KEY `uk_entity_phone` (`entity_id`, `phone_country_code`, `phone_number`),
    KEY `idx_entity_status` (`entity_id`, `status`),
    KEY `idx_ib_id` (`introducing_broker_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户主表(身份与实体隔离)';
```

### 2. 用户适当性快照表 `t_user_profile_snapshot`

> Onboarding 审核通过后经 MQ 单向同步至此，供硬风控直接查询。`source_version` 防 MQ 乱序/重复回写；`kyc_expired_at` 供本地判断是否需重新认证。

```sql
CREATE TABLE `t_user_profile_snapshot` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联用户ID',
    `source_application_id` BIGINT UNSIGNED     DEFAULT NULL COMMENT '同步来源:Onboarding开户申请ID(溯源)',
    `source_version`        BIGINT UNSIGNED     NOT NULL DEFAULT 0
        COMMENT '同步事件版本号,仅更大时才覆盖,防MQ乱序/重复',

    -- ==================== 身份快照 ====================
    `legal_first_name`      VARCHAR(128)        DEFAULT NULL COMMENT '法定名',
    `legal_last_name`       VARCHAR(128)        DEFAULT NULL COMMENT '法定姓',
    `nationality`           VARCHAR(4)          DEFAULT NULL COMMENT '国籍 ISO 3166-1',
    `country_of_residence`  VARCHAR(4)          DEFAULT NULL COMMENT '居住国',
    `tax_residency_country` VARCHAR(4)          DEFAULT NULL COMMENT '税务居住国(CRS/FATCA)',

    -- ==================== 适当性与风控评估快照 ====================
    `is_pro_client`         TINYINT UNSIGNED    NOT NULL DEFAULT 0
        COMMENT '是否专业客户: 0-零售(受限高杠杆) 1-专业(可申请高杠杆)',
    `risk_tolerance`        TINYINT UNSIGNED    DEFAULT NULL
        COMMENT '风险承受能力: 1-保守 2-稳健 3-平衡 4-积极 5-激进',
    `risk_score`            DECIMAL(5,2)        DEFAULT NULL COMMENT 'AML/适当性综合风险评分0~100',
    `trading_exp_years`     TINYINT UNSIGNED    DEFAULT NULL COMMENT '交易经验年数',
    `kyc_expired_at`        DATETIME            DEFAULT NULL COMMENT 'KYC有效期,过期限制开新仓,NULL=长期有效',

    `synced_at`             DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '最后同步时间',
    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_id` (`user_id`),
    KEY `idx_kyc_expired` (`kyc_expired_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户适当性快照(从Onboarding单向同步)';
```

### 3. 用户交易偏好表 `t_user_preference`

> 仅保留影响前端 UI 交互和本地软拦截的偏好，硬风控交由独立风控引擎。

```sql
CREATE TABLE `t_user_preference` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联用户ID',

    -- ==================== 交易交互偏好 ====================
    `default_leverage`      INT UNSIGNED        DEFAULT NULL COMMENT '默认杠杆(开仓界面默认选中)',
    `default_volume`        DECIMAL(12,4)       DEFAULT NULL COMMENT '默认下单手数',
    `confirm_order`         TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '下单前二次确认: 0-否 1-是',
    `one_click_trading`     TINYINT UNSIGNED    NOT NULL DEFAULT 0 COMMENT '一键交易: 0-否 1-是',

    -- ==================== 软风控偏好(风控引擎读取) ====================
    `daily_loss_limit`      DECIMAL(18,2)       DEFAULT NULL COMMENT '单日最大亏损限额(结算币种),触发禁开新仓并通知',

    -- ==================== 显示偏好 ====================
    `locale`                VARCHAR(8)          NOT NULL DEFAULT 'en' COMMENT '语言: en/zh/ar等',
    `timezone`              VARCHAR(32)         NOT NULL DEFAULT 'UTC' COMMENT '时区,如Asia/Shanghai',
    `chart_theme`           VARCHAR(16)         DEFAULT 'dark' COMMENT '图表主题: dark/light',

    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户交易偏好与软风控';
```

### 4. 原生认证凭证表 `t_user_credential`

> 密码、2FA、API Key 统一抽象，便于统一做防爆破与过期管理。`cred_identifier` 用与用户绑定的全局唯一值（PASSWORD=`user_uid`、API_KEY=AccessKey、TOTP/PASSKEY=`user_uid` 前缀标识）；登录先经 `t_user` 解析 `user_id` 再按 `user_id` 加载。

```sql
CREATE TABLE `t_user_credential` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联用户ID',

    `cred_type`             VARCHAR(16)         NOT NULL
        COMMENT '凭证类型: PASSWORD/TOTP/API_KEY/PASSKEY',
    `cred_identifier`       VARCHAR(256)        NOT NULL
        COMMENT '凭证标识(全局唯一,与用户绑定): PASSWORD=user_uid; API_KEY=AccessKey; TOTP/PASSKEY=user_uid前缀或credential_id',
    `cred_secret`           VARCHAR(512)        NOT NULL
        COMMENT '凭证密文: PASSWORD=bcrypt; TOTP=种子(AES); API_KEY=SecretKey(AES)',

    -- ==================== API Key 专属 ====================
    `api_label`             VARCHAR(64)         DEFAULT NULL COMMENT 'API Key备注名',
    `api_permissions`       VARCHAR(128)        DEFAULT NULL COMMENT 'API权限: READ,TRADE,WITHDRAW',
    `api_ip_whitelist`      VARCHAR(512)        DEFAULT NULL COMMENT 'IP白名单,逗号分隔',

    -- ==================== 安全与生命周期 ====================
    `is_active`             TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '是否启用: 0-禁用 1-启用',
    `fail_count`            INT UNSIGNED        NOT NULL DEFAULT 0 COMMENT '连续失败次数(防爆破)',
    `locked_until`          DATETIME            DEFAULT NULL COMMENT '锁定截止时间',
    `last_used_at`          DATETIME(3)         DEFAULT NULL COMMENT '最后使用时间',
    `password_updated_at`   DATETIME(3)         DEFAULT NULL COMMENT '密码最后修改时间(仅PASSWORD)',
    `expired_at`            DATETIME            DEFAULT NULL COMMENT '过期时间(NULL=永不过期)',

    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_type_identifier` (`cred_type`, `cred_identifier`),
    KEY `idx_user_id` (`user_id`, `cred_type`, `is_active`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='统一认证凭证表(多态设计)';
```

### 5. 第三方授权绑定表 `t_user_oauth`

> OAuth 本质是“授权”，仅存 OpenID。按 `entity_id` 隔离唯一；**解绑物理删除**释放唯一槽位（解绑动作写 `t_user_operation_log` 留痕）。纯登录场景不持久化第三方 Token。

```sql
CREATE TABLE `t_user_oauth` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联本系统用户ID',
    `entity_id`             BIGINT UNSIGNED     NOT NULL DEFAULT 1 COMMENT '所属监管实体ID(多实体隔离)',

    `provider`              VARCHAR(16)         NOT NULL COMMENT '第三方平台: GOOGLE/APPLE/FACEBOOK/WECHAT/LINE',
    `open_id`               VARCHAR(128)        NOT NULL COMMENT '第三方唯一用户ID(OpenID/UnionID)',
    `union_id`              VARCHAR(128)        DEFAULT NULL COMMENT '联合ID(微信等跨应用识别)',

    `nickname`              VARCHAR(128)        DEFAULT NULL COMMENT '第三方昵称(首次授权拉取)',
    `avatar_url`            VARCHAR(512)        DEFAULT NULL COMMENT '第三方头像URL',

    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '首次绑定时间',
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_entity_provider_openid` (`entity_id`, `provider`, `open_id`),
    KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='第三方授权登录绑定表(解绑物理删除)';
```

### 6. 用户受信设备表 `t_user_device`

> 新设备登录需二次验证；受信设备有有效期，防止僵尸设备长期留存。

```sql
CREATE TABLE `t_user_device` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联用户ID',

    `device_fingerprint`    VARCHAR(128)        NOT NULL COMMENT '设备唯一指纹(客户端SDK生成)',
    `device_name`           VARCHAR(128)        DEFAULT NULL COMMENT '设备名称,如iPhone 15 Pro',
    `device_type`           VARCHAR(16)         DEFAULT NULL COMMENT '设备类型: IOS/ANDROID/WEB/DESKTOP',
    `client_ip`             VARCHAR(45)         DEFAULT NULL COMMENT '最近登录IP',

    `is_trusted`            TINYINT UNSIGNED    NOT NULL DEFAULT 0 COMMENT '是否受信: 0-否(需二次验证) 1-是',
    `trusted_at`            DATETIME            DEFAULT NULL COMMENT '受信标记时间',
    `last_active_at`        DATETIME(3)         DEFAULT NULL COMMENT '最后活跃时间',
    `expire_at`             DATETIME            DEFAULT NULL COMMENT '受信过期时间(如90天)',
    `status`                TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '设备状态: 0-已移除 1-活跃',

    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '创建时间',
    `updated_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3) COMMENT '更新时间',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_device` (`user_id`, `device_fingerprint`),
    KEY `idx_user_status` (`user_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户受信设备与风控';
```

### 7. 用户标签关系表 `t_user_tag_relation`

> 运营与风控分群。打标表而非字段，便于灵活扩展与批量反向查询。

```sql
CREATE TABLE `t_user_tag_relation` (

    `expire_at`             DATETIME            DEFAULT NULL COMMENT '标签过期时间(NULL=永久)',
    `assigned_by`           VARCHAR(64)         DEFAULT 'SYSTEM' COMMENT '打标来源: SYSTEM/操作员ID',
    `remark`                VARCHAR(128)        DEFAULT NULL COMMENT '打标备注',
    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '打标时间',

    UNIQUE KEY `uk_user_tag` (`user_id`, `tag_code`),

```

### 8. 登录与安全事件日志 `t_user_login_log`

> 登录与安全敏感操作留痕，不可修改，按季度分区。分区由定时任务每季度滚动新增并 DROP 超出保留期的旧分区。

```sql
CREATE TABLE `t_user_login_log` (
    `id`                    BIGINT UNSIGNED     NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`               BIGINT UNSIGNED     NOT NULL COMMENT '关联用户ID',

    `event_type`            VARCHAR(24)         NOT NULL
        COMMENT '事件类型: LOGIN/LOGOUT/PASSWORD_CHANGED/TOTP_ENABLED/API_KEY_CREATED/DEVICE_TRUSTED',
    `event_result`          TINYINT UNSIGNED    NOT NULL DEFAULT 1 COMMENT '结果: 1-成功 0-失败',
    `fail_reason`           VARCHAR(64)         DEFAULT NULL
        COMMENT '失败原因: WRONG_PASSWORD/ACCOUNT_FROZEN/2FA_FAILED/IP_BLOCKED/DEVICE_UNTRUSTED',

    `client_ip`             VARCHAR(45)         NOT NULL COMMENT '客户端IP(兼容IPv6)',
    `ip_country`            VARCHAR(4)          DEFAULT NULL COMMENT 'IP所属国家,ISO 3166-1',
    `device_fingerprint`    VARCHAR(128)        DEFAULT NULL COMMENT '设备指纹',
    `user_agent`            VARCHAR(512)        DEFAULT NULL COMMENT 'User-Agent',
    `client_type`           VARCHAR(12)         DEFAULT NULL COMMENT '客户端: WEB/IOS/ANDROID/DESKTOP/API',

    `created_at`            DATETIME(3)         NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '事件时间',

    PRIMARY KEY (`id`),
    KEY `idx_user_time` (`user_id`, `created_at`),
    KEY `idx_event_type` (`event_type`, `created_at`),
    KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='登录与安全事件日志'
PARTITION BY RANGE (TO_DAYS(`created_at`)) (
    PARTITION p2025q4 VALUES LESS THAN (TO_DAYS('2026-01-01')),
    PARTITION p2026q1 VALUES LESS THAN (TO_DAYS('2026-04-01')),
    PARTITION p2026q2 VALUES LESS THAN (TO_DAYS('2026-07-01')),
    PARTITION p2026q3 VALUES LESS THAN (TO_DAYS('2026-10-01')),
    PARTITION p2026q4 VALUES LESS THAN (TO_DAYS('2027-01-01')),
    PARTITION p_max   VALUES LESS THAN MAXVALUE
);
```

### 9. 用户业务操作审计日志 `t_user_operation_log`

> 非交易类业务操作（改资料、改偏好、绑卡、解绑等）留痕，供客诉追溯与合规检查，按季度分区。

```sql
CREATE TABLE `t_user_operation_log` (
    `id`              BIGINT UNSIGNED   NOT NULL AUTO_INCREMENT COMMENT '自增主键',
    `user_id`         BIGINT UNSIGNED   NOT NULL COMMENT '用户ID',
    `module`          VARCHAR(32)       NOT NULL COMMENT '模块: PROFILE/PREFERENCE/PAYMENT_METHOD/API_KEY/DEVICE/OAUTH',
    `action`          VARCHAR(32)       NOT NULL COMMENT '操作: CREATE/UPDATE/DELETE/BIND/UNBIND',
    `target_type`     VARCHAR(32)       DEFAULT NULL COMMENT '对象类型: BANK_CARD/ALERT/PREFERENCE/API_KEY/OAUTH_BINDING',
    `target_id`       BIGINT UNSIGNED   DEFAULT NULL COMMENT '对象ID',
    `change_summary`  TEXT              DEFAULT NULL COMMENT '变更摘要JSON: {"field":"phone","old":"138***","new":"139***"}',
    `client_ip`       VARCHAR(45)       NOT NULL COMMENT '客户端IP',
    `client_type`     VARCHAR(12)       DEFAULT NULL COMMENT 'WEB/IOS/ANDROID/API',
    `user_agent`      VARCHAR(512)      DEFAULT NULL COMMENT 'User-Agent',
    `created_at`      DATETIME(3)       NOT NULL DEFAULT CURRENT_TIMESTAMP(3) COMMENT '操作时间',

    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`, `created_at`),
    KEY `idx_module` (`module`, `action`, `created_at`),
    KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户操作审计日志'
PARTITION BY RANGE (TO_DAYS(`created_at`)) (
    PARTITION p2025q4 VALUES LESS THAN (TO_DAYS('2026-01-01')),
    PARTITION p2026q1 VALUES LESS THAN (TO_DAYS('2026-04-01')),
    PARTITION p2026q2 VALUES LESS THAN (TO_DAYS('2026-07-01')),
    PARTITION p2026q3 VALUES LESS THAN (TO_DAYS('2026-10-01')),
    PARTITION p2026q4 VALUES LESS THAN (TO_DAYS('2027-01-01')),
    PARTITION p_max   VALUES LESS THAN MAXVALUE
);
```

## 四、系统边界交互

| 系统 | 交互方式 |
|------|----------|
| **Onboarding** | KYC 通过后经 MQ 发 `UserApprovedEvent`，交易系统创建/更新 `t_user` 并 Upsert `t_user_profile_snapshot`；**绝不**反向调用查询，保证低延迟。 |
| **Account 账号体系** | 开户时账号体系按 `t_user.entity_id` 过滤可用 `t_account_group`，确保牌照隔离。 |
| **IB 代理系统** | 仅注册时落库 `introducing_broker_id`；代理系统按 IB 拉取用户列表 + 账单流水自行结算分润，用户体系不参与。 |
| **Risk Control 风控** | 硬风控（制裁名单/刷单）直接拦截交易请求；软风控（单日止损）将 `daily_loss_limit` 同步至风控 Redis，网关本地校验。 |
| **Notification 通知** | 用户体系不维护通知偏好；仅投递 `{user_id, event_type}` 到 MQ，通知系统自查偏好决定渠道。 |

## 五、登录鉴权链路

### 场景 A：账号密码 / 手机验证码登录
1. 用户输入 Email + Password；网关按品牌/域名/App 标识确定 `entity_id`。
2. 用 `(entity_id, email)` 查 `t_user` 解析 `user_id`（多实体隔离）。
3. 按 `user_id` + `cred_type='PASSWORD'` 查 `t_user_credential`，校验 bcrypt，检查 `fail_count`/`locked_until`。
4. 查 `t_user_device` 校验设备指纹；`is_trusted=0` 或 `mfa_enabled=1` 时触发二次验证。
5. 签发的 Token 内嵌当前 `security_stamp`；异步更新 `last_login_at`、`last_active_at`，写 `t_user_login_log`。

### 场景 B：Google / Apple 第三方登录
1. 客户端拉起第三方 SDK，得到 `provider`、`open_id`；网关确定 `entity_id`。
2. 查 `t_user_oauth (entity_id, provider, open_id)`。
3. **查到**：取 `user_id`，发 Token 登录（能查到即有效绑定）。
4. **未查到**：创建 `t_user`（`status=0`，`register_channel=OAUTH`）→ 插入绑定记录 → 引导补充手机/邮箱（合规要求）→ `status=1`。

> **解绑**：`DELETE FROM t_user_oauth WHERE id=?`，并写 `t_user_operation_log(module=OAUTH, action=UNBIND)` 留痕。解绑前需校验用户至少保留一种可登录凭证。

### 场景 C：API 程序化交易登录
1. 量化客户用 API Key/Secret 连接撮合网关。
2. 查 `t_user_credential (cred_type='API_KEY')`，校验 Secret 与 `api_ip_whitelist`。
3. 校验 `api_permissions` 含 `TRADE` → 放行。

### 场景 D：令牌校验与全局下线
1. 每次请求验签后，比对 Token 内 `security_stamp` 与 `t_user.security_stamp`（Redis 缓存），不一致即失效。
2. 改密码 / 重置 2FA / 全局冻结 / “退出所有设备”时刷新 `security_stamp`，历史 Token 立即全部失效。
3. `2-全局冻结` / `4-已注销` 用户在验签前直接拦截。
