Event-driven Integration — "Asynchronous decoupling."
📨
剧情：用户体系通过 MQ 与 Onboarding / 账户 / 风控 / 通知 / IB 系统解耦。只发不收(除消费 Onboarding 的 UserApprovedEvent)。 消费侧必须幂等(event_id 去重),必须支持乱序(source_version)。
PUBLISH topic: user.events 本服务发布的事件
事件类型触发时机下游消费者
UserCreatedEvent注册成功CRM / 营销
UserActivatedEvent激活(status: 0→1)账户 / IB
UserFrozenEvent冻结(status: 1→2)账户 / 风控 / 通知
UserDeactivateRequestedEvent申请注销账户 / 通知 / 营销
UserDeactivatedEvent冷静期结束(status: 3→4)账户 / IB / CRM
UserLoggedInEvent登录成功风控 / 行为分析
UserPasswordChangedEvent改密风控 / 通知
UserApiKeyCreatedEventAPI Key 创建/重置风控 / 通知
UserTagAddedEvent打标CRM / 营销
SUBSCRIBE topic: onboarding.events 本服务消费的事件
