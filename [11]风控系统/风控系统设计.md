# 风控系统设计（权威版）

> **版本**：v1.0 Final  
> **定位**：零售 / 专业客户多资产 CFD·FX 经纪平台的**风险规则权威源**与**双层风控（客户 + 公司）设计权威**。  
> **运行时分层**：规则/限额配置在风控域（MySQL + 版本号）；**热路径执行嵌入 Trading Engine 内存**；Risk Ops 负责运营配置、异步审计投影、内部报告；Kafka Outbox 跨域事件。**方向设计不锁定实现语言。**  
> **对标**：MetaQuotes / Spotware(cTrader) / Devexperts / Apex / DriveWealth / TradingView / TT / Broadridge / Talos / 恒生电子。  
> **与原稿关系**：原稿中 PreTrade 检查链、Margin Call / Stop Out、规则表、事件审计、相关性敞口、`calcMargin` 予以吸收；**运行时权威（内存执行）、公司敞口、Kill Switch 内存模型、与交易/资金/行情契约、监管仅运营级、分阶段里程碑以本文为准**。

## 一、系统定位与对标决策

### 1.1 系统定位

Risk Domain 是经纪平台的：

1. **交易风控规则权威**（`t_risk_rule`、相关性组、日损协同策略）  
2. **熔断 / 品种 Halt 策略权威**（四级 Kill Switch）  
3. **公司敞口限额权威**（`t_risk_house_limit`、House Exposure 聚合语义）  
4. **风控审计与运营报告权威**（事件日志、日快照、内部大额/可疑标记）  
5. **跨域风控事件语义定义方**（`risk.*` / 协同 `margin.*` / `order.rejected` 原因码）

以下概念**不属于**本系统（仅契约对接）：

| 概念 | 归属 |
|------|------|
| Order / Deal / Position 生命周期、成交原子提交 | 交易系统（引擎执行 PreTrade / 强平） |
| Core SymbolBook、STALE | 行情体系（引擎只读 Core） |
| 出入金限额、冷静期、原路退回 | 资金体系 `t_fund_risk_rule` |
| 账户组硬阈值字段权威（`margin_call_level` 等） | 账户体系（风控消费并在引擎加载） |
| KYC / AML 准入、账户封禁原因上游 | 准入系统 |
| 渠道刷量、对倒、CPA 回拨 | 代理体系 |
| 美股 DTC 清算、期货 SPAN 全栈、完整监管报送闭环 | **明确不做**（扩展点预留） |

### 1.2 行业参考的正确使用方式

| # | 厂商 | 可借鉴 | 本设计采纳 / 取舍 |
|---|------|--------|------------------|
| 1 | **MetaQuotes** | Group 阈值、StopOut、负余额保护、Dealer | **客户风控语义基线**；不照搬 Manager 单体包办 |
| 2 | **Spotware (cTrader)** | Smart Stop Out、账户风险设置 | 采纳 Mode 3 部分平仓；不做多 Asset 钱包 |
| 3 | **Devexperts** | 风险维度嵌入交易路径 | PreTrade / House 检查嵌入引擎；**不做 SPAN** |
| 4 | **Apex Fintech** | 审计可追溯、状态可查询 | 决策快照 + 事件日志；异步不阻塞热路径 |
| 5 | **DriveWealth** | 指令与执行分离意识 | 规则配置 vs 引擎执行分离；不做美股托管 |
| 6 | **TradingView** | 外部入口不影响内部风控 | Broker API 下单仍走同一 PreTrade |

### 1.3 产品定位（拍板）

- **P0（M1）**：客户 PreTrade、Margin Call、Stop Out（Mode 1/2）、全局/品种熔断内存化、事件审计、账户日快照、公司敞口聚合与基础限额告警。  
- **P1（M2）**：Smart Stop Out、相关性敞口、日损限额、Dealer HOLD、公司限额阻断开仓、可疑行为标记、内部大额报告。  
- **P2（M3）**：公司 CLOSE_ONLY 自动策略完善、监管报送适配器扩展点、可选压力情景；**不做**完整 CTR/STR/Basel 报送闭环。

## 二、十四条不可破坏决策

1. **规则配置权威在风控域，执行权威在引擎内存**：热路径禁止读 MySQL / Redis 做放行决策；Redis 仅为运营投影。  
2. **客户风险与公司风险同一事件时钟、同一 Core Book**：估值、保证金、强平、公司敞口名义一律用 Core SymbolBook。  
3. **PreTrade 失败不产生 Deal、不改余额、不改公司敞口库存**（预检用试算，成交原子提交时才落库存）。  
4. **强平是系统订单**：`source=SYSTEM`，`close_reason=STOP_OUT`；可跳过部分客户限制（如保证金不足、Margin Call 禁开仓），**不可跳过成交原子性与账本完整性**。  
5. **配置热更新原子切换**：规则 / 熔断 / 公司限额经 `config_version` 切换；进行中订单使用「接受时快照」或「下一 Tick 生效」，禁止半新半旧混算。  
6. **审计异步、决策同步**：热路径只记内存决策结果意图；`t_risk_event_log` 异步落库，不得阻塞成交。  
7. **`entity_id` 隔离**：客户账户、公司敞口、限额、报告均带实体；禁止跨实体合并敞口或串账。  
8. **不做 SPAN / 完整组合保证金**：相关性用可配置因子组简化合并；账户组若预留 Portfolio 枚举，本引擎不实现。  
9. **出入金风控不进交易热路径**：引擎不加载 `t_fund_risk_rule`；出金可提以资金域 `PreviewWithdraw` 为准。  
10. **Kill Switch 四级**：GLOBAL → ENTITY → SYMBOL → ACCOUNT_GROUP；命中即按策略拒开仓或全停。  
11. **STALE 禁新风险**：Core Book STALE / 断流时拒绝新开仓与依赖报价的触发；强平若无有效价则告警并重试，不静默用过期价成交。  
12. **公司减仓不替代客户强平**：公司限额动作默认 ALERT / REJECT_NEW_OPEN / CLOSE_ONLY；不自动替客户乱平仓；对冲台减仓走独立运营流程。  
13. **冻结余额 ≠ 保证金预占**：`frozen_balance` 属资金域；交易保证金预占在引擎 `margin_used` / 挂单预占语义。  
14. **监管报送可插拔**：P0 只做内部大额与可疑标记；外部 `reported_to` 闭环为扩展，不阻塞交易上线。

## 三、边界与跨系统契约

### 3.1 领域边界

| 系统 | 权威数据 | 风控系统的使用方式 |
|------|---------|-------------------|
| 交易引擎 | Order/Deal/Position、AccountState、成交原子提交 | **内嵌执行** PreTrade / MC / StopOut / House 聚合 |
| 行情 | Core SymbolBook、`seq`、STALE | 只读；禁止用 Client Quote |
| 品种 / 时间 | 规格、`t_daily_session`、日历 | 时段与 CLOSE_ONLY 判断 |
| 账户 | 账户、账户组硬阈值、杠杆 | 加载阈值；`blocked_reason` 可由风控回写 |
| 资金 | 冻结、出入金风控、ledger 结构 | 负余额归零 ledger 类型；`PreviewWithdraw` 含 margin 模拟 |
| 路由 | A/B/Mixed 实际成交路由 | House 敞口按路由结果记账 |
| 准入 | KYC 状态、用户封禁 | PreTrade 读账户/用户状态字段 |
| 代理 | 渠道风控 | 弱耦合；可消费 `deal.*` / 拒单事件 |
| 通知 | 模板送达 | 消费 `margin.*` / `risk.*` |
| 权限 | RBAC / SoD | Dealer 审批、人工强平、Kill Switch 操作四眼 |

### 3.2 运行时职责切分

```
┌─────────────────────────────────────────────────────────────────┐
│ Risk Ops（配置权威 + 异步投影）                                     │
│  规则/限额/熔断 CRUD → t_risk_* + config_version                   │
│  消费 Outbox → t_risk_event_log / 快照 / 内部报告 / 可疑标记         │
│  管理端 Kill Switch / 人工干预 API                                  │
└────────────────────────────┬────────────────────────────────────┘
                             │ 配置版本推送 / 启动加载
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│ Trading Engine（热路径执行权威）                                    │
│  RiskRuleSet / KillSwitchState / HouseExposureBook / AccountRisk │
│  SubmitOrder → PreTrade(Client+House) → Route → Commit           │
│  Tick → PnL → SLTP/挂单 → MarginCall → StopOut                   │
│  Commit 时同步更新 HouseExposureBook                              │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 引擎加载范围（拍板）

**加载：** `t_risk_rule`、`t_risk_symbol_correlation`、`t_risk_house_limit`、Kill Switch / Symbol Halt 配置、账户组风控字段、品种交易参数、`t_daily_session`。  

**不加载：** `t_fund_risk_rule`、代理渠道规则、AML 名单明细（仅消费账户已投影的 `status` / `trade_enabled` / `blocked_reason`）。

### 3.4 与交易 RPC / 状态协同

| 交易能力 | 风控协同 |
|----------|---------|
| `SubmitOrder` / `ModifyOrder` | 完整 PreTrade（改单重算预占与限额） |
| `CancelOrder` | 一般不检保证金；Halt=FULL 时仍允许撤单 |
| `ClosePosition` | Margin Call / CLOSE_ONLY / 公司 CLOSE_ONLY 下**允许** |
| `DealerApprove` / `DealerReject` | 规则 `DEALER_REVIEW` → 订单 `DEALER_HOLD` |
| 系统强平 | 跳过客户「保证金不足 / 禁开仓」类检查；仍校验持仓归属与价格有效性 |

## 四、客户风控（Client Risk）

### 4.1 PreTrade 检查顺序（不可重排）

每一笔客户/API/EA 下单（及挂单触发转执行前的保证金复核）必须按序通过。任一步失败 → 拒绝，不进入 Route/Exec。

```
SubmitOrder / 挂单触发转执行
    │
    ▼
 1. ACCOUNT_STATE
    - user.status=正常；account.status=活跃
    - trade_enabled=1；EA 单要求 expert_enabled=1
    - blocked_reason 非交易禁止类
    失败 → ACCOUNT_DISABLED / ACCOUNT_BLOCKED

 2. KILL_SWITCH
    - GLOBAL / ENTITY / ACCOUNT_GROUP / SYMBOL 四级
    - FULL → 拒绝一切新单（撤单除外）
    - CLOSE_ONLY → 仅允许平仓/减仓
    失败 → TRADING_HALTED / SYMBOL_HALTED

 3. SYMBOL_ACCESS
    - symbol.status 启用；CLOSE_ONLY 品种仅平仓
    - 账户组品种准入 enabled + trade_mode
    - 当前时间 ∈ t_daily_session TRADE；日历无特殊休市
    失败 → SYMBOL_NOT_AVAILABLE / MARKET_CLOSED

 4. ORDER_PARAMS
    - volume ∈ [min,max] 且为 volume_step 整数倍
    - 订单类型 / 过期类型在品种允许集合内
    - 挂单价距 ≥ stop_level；freeze_level 内禁改挂单
    - PRICE_DEVIATION：委托价相对 Core 偏离不超过规则/品种上限
    失败 → INVALID_VOLUME / INVALID_PRICE / PRICE_DEVIATION

 5. POSITION_LIMITS（账户组 + 规则叠加，取更严）
    - 持仓数 / 挂单数 / 单品种同向手数 / 总手数
    失败 → MAX_POSITIONS / MAX_PENDING / MAX_VOLUME

 6. MARGIN_SUFFICIENCY
    - margin_required = calcMargin(...)（见第八章）
    - 含对冲折扣、挂单 margin_rate_*、阶梯杠杆
    - margin_free_after >= 0
    - 开仓后预计 margin_level >= margin_call_level（护栏）
    - 已 ACTIVE Margin Call → 禁止新开仓（仅平仓）
    失败 → NOT_ENOUGH_MARGIN / MARGIN_LEVEL_TOO_LOW / MARGIN_CALL_ACTIVE

 7. DAILY_LOSS（若账户组/账户配置）
    - 当日已实现亏损 + 本单最坏情景（可选）超限 → 禁开仓
    失败 → DAILY_LOSS_LIMIT

 8. CUSTOM_RULES（t_risk_rule，按 priority DESC，全部匹配执行）
    - 任一 REJECT → 拒单
    - 任一 DEALER_REVIEW → DEALER_HOLD（不进自动执行）
    - ALERT 不阻断，异步告警
    失败 → RULE_REJECTED / 进入 DEALER_HOLD

 9. CORE_BOOK_HEALTH
    - Core 非 STALE；seq 新鲜度在阈值内
    失败 → QUOTE_STALE

10. HOUSE_EXPOSURE_PRECHECK（见第五章）
    - 试算本单成交后公司净敞口是否突破限额动作
    - 动作为 REJECT_NEW_OPEN / FORCE_CLOSE_ONLY 且本单为开仓 → 拒绝
    失败 → HOUSE_LIMIT_BREACH

    │ PASS
    ▼
  订单进入 Route → Exec → Atomic Commit
```

**挂单预占（与交易权威对齐）：** 默认 `PLACED` 时预占保证金；修改挂单重算差额；取消/过期释放。触发转执行时再跑 PreTrade（可用触发价重算）。

### 4.2 拒绝原因码（面向 API / 审计）

| reason | 含义 |
|--------|------|
| `ACCOUNT_DISABLED` | 账户/用户不可交易 |
| `ACCOUNT_BLOCKED` | 风控/合规封禁 |
| `TRADING_HALTED` | 全局或实体熔断 |
| `SYMBOL_HALTED` | 品种 Halt |
| `SYMBOL_NOT_AVAILABLE` | 品种未启用或组未准入 |
| `MARKET_CLOSED` | 非交易时段 |
| `INVALID_VOLUME` | 手数非法 |
| `INVALID_PRICE` | 价格/止损距离非法 |
| `PRICE_DEVIATION` | 错价/偏离保护 |
| `MAX_POSITIONS` / `MAX_PENDING` / `MAX_VOLUME` | 持仓类限制 |
| `NOT_ENOUGH_MARGIN` | 保证金不足 |
| `MARGIN_LEVEL_TOO_LOW` | 开仓后将低于追保线 |
| `MARGIN_CALL_ACTIVE` | 追保中禁开仓 |
| `DAILY_LOSS_LIMIT` | 日损触达 |
| `RULE_REJECTED` | 自定义规则拒绝 |
| `QUOTE_STALE` | 报价不可用 |
| `HOUSE_LIMIT_BREACH` | 公司敞口限额 |
| `DEALER_PENDING` | 等待人工（非终态拒绝，订单 HOLD） |

### 4.3 盘中监控（Intra-Day）

行情 Tick（Core）到达后，引擎内存路径（**禁止扫库**）：

```
Tick(symbol_key, bid, ask, seq)
  → 更新该品种持仓浮动盈亏
  → equity = balance + credit + floating_pnl
  → margin_level = equity / margin_used × 100%（无持仓则 NULL）
  → SL/TP 触发 → 系统平仓单
  → 挂单触发 → 转执行（再 PreTrade）
  → Margin Call / Stop Out 状态机
  → 投影意图：risk:margin:{account_id}（异步/旁路写 Redis，不参与决策）
```

浮盈公式（账户币种）：

```
BUY:  (bid - price_open) × volume × contract_size × fx
SELL: (price_open - ask) × volume × contract_size × fx
```

### 4.4 Margin Call 状态机

| 条件 | 状态 | 动作 |
|------|------|------|
| `margin_level > margin_call_level` | 正常 | 若曾 ACTIVE → RESOLVED，恢复开仓 |
| `stop_out_level < margin_level ≤ margin_call_level` | ACTIVE | 禁新开仓；通知；写 `t_risk_margin_call`；事件 `margin.call.v1` |
| `margin_level ≤ stop_out_level` | ESCALATED | 进入 Stop Out |

解除原因：`DEPOSIT` / `CLOSE_POSITION` / `PRICE_RECOVERY` / `STOP_OUT`。  
持续 ACTIVE 时按配置间隔重复提醒（通知域消费事件，风控不直发邮件）。

### 4.5 Stop Out（强制平仓）

**触发：** `margin_level ≤ stop_out_level` 且存在可平持仓。

**模式（`t_account_group.stop_out_mode`）：**

| Mode | 名称 | 行为 |
|------|------|------|
| 1 | 亏损最大优先 | MT5 默认；按浮动亏损从大到小整笔平 |
| 2 | 保证金占用最大优先 | 按占用保证金从大到小整笔平 |
| 3 | Smart Stop Out | 计算恢复安全线所需释放保证金，从最大仓起**部分平仓**，步进 `volume_step` |

```
WHILE margin_level ≤ stop_out_level AND 仍有可平持仓:
  1. 按 mode 选仓（Mode 3 算 volume_to_close）
  2. 生成系统平仓单 source=SYSTEM, close_reason=STOP_OUT
  3. 跳过客户「保证金不足/禁开仓」；不跳过价格有效性与原子提交
  4. 成功 → 重算 margin_level；写 stop_out detail
  5. 失败（无价/流动性）→ 记失败，试下一仓；全部失败 → 告警运营

结束后:
  - t_risk_stop_out_execution 汇总
  - margin.stopout.v1
  - 若 balance < 0 且 negative_balance_protection=1
    → 引擎写资金域约定 ledger（如 NEGATIVE_BALANCE_CORRECTION）归零
```

Mode 3 释放量示意：

```
target_margin = equity / (stop_out_level / 100)   -- 恢复到强平线以上的目标占用
release_needed = max(0, margin_used - target_margin)
volume_to_close = ceil_to_step(release_needed / margin_per_lot, volume_step)
```

### 4.6 日损限额（Daily Loss）

- 阈值来源：账户组 `daily_loss_limit`，账户级可覆盖（更严）。  
- 统计口径（拍板）：**自然日（实体时区）已实现亏损**（含强平、佣金；不含浮动；Swap 计入已实现）。  
- 触达 → 当日禁开仓（可平仓/可出金按资金规则）；事件 `risk.rule.triggered`（类型 DAILY_LOSS）。  
- 与杠杆 PDT/风控降杠杆协同：**先应用杠杆 override，再算保证金与日损**。

### 4.7 日终风控（End-of-Day）

| 步骤 | 说明 |
|------|------|
| 1 | Swap 由交易/结算路径入账（风控不重复计息） |
| 2 | 引擎保证金字段投影到账户/持仓快照 |
| 3 | 写入 `t_risk_account_snapshot`（健康度、集中度 Top3） |
| 4 | 扫描生成**内部**大额敞口报告（非外部报送） |
| 5 | 公司敞口小时/日快照 `t_risk_house_exposure_snapshot` |
| 6 | Ledger 完整性校验告警（校验逻辑归资金域，风控订阅结果） |

## 五、公司风控（House Risk）

### 5.1 定位

对标券商 Risk Desk / TT 盘前限额 / Talos 敞口可见：在**实体维度**控制公司因客户成交产生的市场风险，与客户保证金风控正交。

### 5.2 敞口记账模型（拍板）

以**公司视角**净名义敞口（折算限额币种，默认 USD）：

| 路由 | 客户成交 | 公司敞口变化 |
|------|----------|--------------|
| **B-Book** | 客户 BUY | 公司 SHORT +名义 |
| **B-Book** | 客户 SELL | 公司 LONG +名义 |
| **A-Book** | 客户成交、LP 未确认 | 记入 **TEMP** 临时敞口（与客户同向） |
| **A-Book** | LP 对冲成交确认 | TEMP ↓，对冲后净敞口回落（理想为 0 ± 滑点残差） |
| **Mixed** | 按实际 B/A 分量分别累计 | 同左 |

```
notional_usd = volume × contract_size × mid(Core) × fx_to_usd
house_signed_qty:
  B-Book: house_side = opposite(client_side)
  A-Book TEMP: house_side = client_side（未对冲风险）
```

**残差：** A-Book 对冲失败/超时导致的残留计入 `UNHEDGED`，与 TEMP 一并受限额约束。

### 5.3 运行时：`HouseExposureBook`

引擎内存结构（按 `entity_id`）：

```
HouseExposureBook
  ├── by_symbol[symbol_id]: { long_usd, short_usd, net_usd, temp_usd }
  ├── by_corr_group[group]: { net_usd }   // 相关性合并
  ├── by_currency[ccy]: { net_usd }
  └── global: { net_usd, gross_usd, day_pnl_est }
```

- **更新时机**：客户 Deal 原子提交成功时同步更新；A-Book LP 回报确认时调整 TEMP→对冲。  
- **PreTrade**：用试算副本 `what_if`，不污染正式账簿。  
- **投影**：异步写快照表 / Redis 看板。

### 5.4 限额维度（`t_risk_house_limit`）

| scope_type | 含义 |
|------------|------|
| `GLOBAL` | 实体总净名义 / 总毛名义 |
| `SYMBOL` | 单品种 |
| `CORRELATION_GROUP` | 相关性组合并净敞口 |
| `CURRENCY` | 计价/基础货币净敞口 |
| `SYMBOL_GROUP` | 品种组（若配置） |

阈值字段：`max_net_usd`、`max_gross_usd`、`max_temp_usd`（A-Book 临时）、`max_day_loss_usd`（可选告警）。

### 5.5 突破动作

| action | 行为 |
|--------|------|
| `ALERT` | 不阻断；事件 + 运营看板 |
| `REJECT_NEW_OPEN` | 拒绝该实体下增加公司风险的开仓；平仓/减仓允许 |
| `FORCE_CLOSE_ONLY` | 实体或品种进入仅平仓（客户侧体现为 CLOSE_ONLY） |
| `DEALER_REVIEW` | 大单开仓进 Dealer |

**禁止：** 因公司限额自动批量平客户仓（避免与客户 Stop Out 语义混淆）。公司需要减库存时走对冲台/运营工单（`t_risk_house_action_log`）。

### 5.6 公司风控与客户风控的交互

```
客户 PreTrade 通过 1~9
  → House what_if
       若突破 REJECT_NEW_OPEN/FORCE_CLOSE_ONLY → 拒开仓
  → 成交 Commit
       → 更新客户 AccountState
       → 更新 HouseExposureBook
       → 若跨越告警线 → risk.house.limit.breached.v1
```

## 八、保证金计算（calcMargin）

与交易系统共用实现；风控定义校验语义，引擎执行。

```
calcMargin(symbol, volume, price, account, order_type):

1. mode = symbol.margin_mode
2. leverage:
   - 若 group.leverage_template_id → 按名义阶梯
   - 否则 account.leverage
   - 再受 group_symbol.max_leverage_override、风控/PDT override 约束（取更严）
3. 原始保证金:
   Forex:   (volume × contract_size) / leverage
   CFD:     (volume × contract_size × price) / leverage
   Fixed:   volume × margin_initial
   Exchange: volume × contract_size × price × margin_initial_rate
4. × symbol.margin_rate_{order_type}
5. 阶梯杠杆分段累加（若模板）
6. 对冲折扣（allow_hedging=1）:
   hedged = min(buy_vol, sell_vol)
   margin = unhedged_margin + hedged_margin × margin_hedged_ratio
7. 币种转换 → account.currency
```

**明确不做：** Portfolio Margin / SPAN / 跨品种官方保证金抵扣（相关性仅用于**敞口限额**合并，不自动减收保证金）。

价格输入：开仓用 Core Ask/Bid；挂单预占用挂单价；禁止 Client Quote。

## 十、Redis 投影设计（非决策权威）

> **铁律：** PreTrade / 强平 / House 限额**不得**以 Redis 值为放行依据。下列 Key 供管理端看板、客户 App 展示、降载查询。

```
risk:margin:{account_id}          Hash   净值/保证金/margin_level 投影
risk:house:{entity_id}            Hash   公司净/毛/临时敞口汇总投影
risk:margin_call:{account_id}     String 展示用标记(可选)
risk:kill_switch:view             Hash   当前熔断状态只读投影
risk:order_rate:{account_id}      ZSet   可选: Ops 分析; 引擎内另有内存滑动窗口做频率规则
```

引擎内频率限制使用**内存环形/滑动窗口**；若需多实例共享，以引擎分片账户粘性为准，不引入热路径跨机 Redis round-trip。

## 十一、验收 SLO 与对账

| 指标 | 目标 |
|------|------|
| PreTrade 纯内存（不含网络）P99 | &lt; 1ms |
| Tick→单账户 margin_level 更新 P99 | &lt; 1ms（单品种受影响账户集） |
| Stop Out 首笔系统平仓发出 P99 | &lt; 5ms（有有效价时） |
| 风控配置热更生效 | 版本切换后新单即用新配置 |
| 事件审计 | 每笔拒单/MC/StopOut/House 突破可检索 |
| 公司敞口 | 日终 House 快照与 Deal 重放差异 = 0（抽样对账） |

## 十二、落地分期

| 阶段 | 范围 | 完成标准 |
|------|------|---------|
| **M1** | 客户 PreTrade（1–9 步核心）、MC、Stop Out Mode1/2、负余额保护、事件日志、账户日快照、GLOBAL/SYMBOL 熔断内存化、House 聚合 + ALERT 限额 | 模拟盘拒单原因正确；强平可回放；熔断不读 Redis |
| **M2** | Mode3 Smart、相关性敞口、日损、Dealer HOLD、House REJECT/CLOSE_ONLY、可疑标记、内部大额报告、配置版本热更 | 公司突破可阻断开仓；可疑单可运营闭环 |
| **M3** | 公司干预工单完善、监管报送适配器扩展点、压力情景（可选） | 扩展字段可接外部报送且不影响交易路径 |
