# 营销与推荐系统设计（权威版）

> **版本**：v1.1 Final  
> **定位**：经纪平台的**营销活动、客户分群推荐与 Desk 经营权威**——CRM 分群/归属、B2C 活动发奖、个性化推荐、投诉工单。  
> **命名说明**：业界常称 CRM；本域强调「营销与推荐」，**不含**资讯 CMS 与 IM 广场（见《内容与社区系统设计》）。  
> **对标**：券商中台 CRM、零售 Growth/Loyalty、推荐投放（能力级）。  
> **运行时分层**：CRM Desk · Segment/Recommend · Campaign Orchestrator · Ticket；触达走通知域；发奖走资金引擎契约。**方向设计不锁定实现语言。**  
> **域地图**：见 [`[总览]经纪平台域地图.md`](../[总览]经纪平台域地图.md)。

## 一、系统定位与对标决策

### 1.1 系统定位

1. **CRM Desk**：分群、客户经理归属、任务跟进、生命周期阶段、经营标签  
2. **Growth**：入金/交易赠金、优惠券、竞赛、邀请好友、Promo Code  
3. **推荐**：基于分群/行为的活动与内容卡片推荐意图（内容实体仍在内容域）  
4. **工单/投诉**：受理、路由、结案  

以下概念**不属于**本系统：

| 概念 | 归属 |
|------|------|
| 资讯 CMS、IM 广场/群聊 | 内容与社区 [15] |
| 跟单、排行榜 | 社交交易 [13] |
| Partner 归因与返佣 | 代理 [7] |
| 通知送达 | 通知 [3] |
| 余额直写 | 资金 / 引擎 |
| KYC 审核结论 | 准入 |

### 1.2 与代理的关系

| 事项 | 代理 [7] | 本域 [14] |
|------|----------|-----------|
| 客户归因 | **唯一权威** | 只读渠道来源投影 |
| IB 佣金 | **唯一计提** | 不参与 |
| B2C 赠金活动 | 不拥有 | **编排权威** |
| 内部客户经理 KPI | 不属于代理 | 本域内部激励（若做） |

禁止：改写 Attribution；Partner 代发 B2C Credit。

## 二、十二条不可破坏决策

1. **`entity_id` 贯穿**；禁止跨实体活动/分群串数据。  
2. **不改交易真相**：禁止写 Order/Deal/Position/ledger。  
3. **发奖只经资金契约**：`AdjustCredit` 等 + 幂等键。  
4. **不改代理归因**。  
5. **营销触达遵守 opt-in**（通知域 MARKETING）。  
6. **推荐不替代内容权威**：推荐只产出「推哪篇/哪个活动」；正文在内容域。  
7. **PII 最小必要**；Desk 导出审计。  
8. **四眼/SoD**：大额发奖、归属批量转移。  
9. **事件经 Outbox**。  
10. **活动预算独立**：≠ Partner CPA 营销池。  
11. **工单不改余额**：资金争议协同资金域冻结。  
12. **不做**：IM、CMS、跟单引擎、第二套通知通道。

## 三、跨域契约

| 系统 | 使用方式 |
|------|----------|
| 用户 / 准入 / 账户 | 读身份、KYC、账户；经营标签本域治理 |
| 资金 | 发奖；听 `deposit.*` 做资格 |
| 交易 | 只读成交做竞赛/推荐特征 |
| 代理 | 只读归因投影 |
| 内容与社区 | 推荐可引用 `content_id`；不写文章 |
| 社交交易 | 可推荐「去跟单」深链；不拥有订阅 |

**本域事件**：`campaign.*`、`crm.task.*`、`crm.assignment.*`、`ticket.*`、`recommend.impression/click.v1`（可选）。

**发奖流**：

```text
Campaign → 资格校验 → reward_id 幂等 → 引擎/资金 AdjustCredit
  → campaign.reward.granted.v1
```

### 9.4 推荐系统详设（补缺）

#### 槽位模型

| 槽位 | 位置 | 候选类型 |
|------|------|----------|
| HOME_BANNER | 首页 banner | 活动、内容、Provider |
| DEPOSIT_PROMO | 入金页 | 充值活动 |
| SYMBOL_HOT | 热门品种 | 品种卡片 |
| PROVIDER_REC | 推荐信号源 | Provider 深链 |
| CONTENT_REC | 推荐资讯 | 内容卡片 |

#### 召回→排序→重排

```
召回（多路）：
  - 活动召回（符合资格的活动）
  - 内容召回（基于兴趣标签）
  - Provider 召回（基于跟单偏好）
  - 冷启动召回（新用户默认）
→ 合并去重
→ 排序（规则/模型）：
  - 规则：优先级 + 权重
  - 模型：CTR 预估（P2）
→ 重排：
  - 频控（每用户每槽位每日 ≤ N）
  - 去重（24h 内不重复曝光）
  - 多样性（同类型 ≤ 2）
  - A/B 分流
```

#### 冷启动

- 新用户：默认槽位（平台精选活动 + 热门品种 + 教育内容）。
- 探索/利用：ε-greedy（ε=0.1 探索新内容）。
- 兴趣标签：基于浏览/点击埋点构建。

#### 频控与去重

- `t_recommend_exposure`（user × slot × day × content_id）。
- 频控：营销 PUSH 每日 ≤ 1，banner 每日 ≤ 5。
- 去重：24h 内同 content 不重复曝光。

#### A/B 实验

- `t_ab_experiment`（experiment_id、variant、user_bucket）。
- 埋点 `recommend.impression.v1` / `recommend.click.v1` 含 experiment_id。
- 效果归因：CTR、转化率、ROI。

### 9.5 Growth 活动资格引擎（补缺）

#### 资格规则表达式

```json
{
  "campaign_id": "int64",
  "campaign_type": "DEPOSIT_BONUS|TRADE_REBATE|COUPON|CONTEST|REFERRAL|PROMO_CODE",
  "eligibility_rule": {
    "kyc_level": ">= 2",
    "client_category": "IN [RETAIL, PROFESSIONAL]",
    "min_deposit": 1000,
    "min_trade_lots": 0,
    "symbol_scope": "ALL|LIST",
    "time_window": "2026-07-01 to 2026-08-01",
    "entity_id": "string"
  },
  "reward_rule": {
    "type": "BONUS|REBATE|COUPON|CASH",
    "amount": 100,
    "currency": "USD",
    "cap_per_user": 500,
    "cap_total": 100000
  }

```

#### 各 campaign_type 详设

| 类型 | 资格 | 发奖时机 | 上限 | 叠加 |
|------|------|----------|------|------|
| DEPOSIT_BONUS | 入金 ≥ 门槛 | 入金到账后 | 按比例封顶 | 不叠加同类型 |
| TRADE_REBATE | 交易手数达标 | 日/周结算 | 按手数封顶 | 可叠加 |
| COUPON | 领取 | 凭码使用 | 一人一码 | — |
| CONTEST | 报名 | 排行结算 | 按名次 | 与社交排行隔离 |
| REFERRAL | 邀请有效 | 被邀请入金 | 按邀请数封顶 | — |
| PROMO_CODE | 输入码 | 首次入金 | 一人一次 | — |

### 9.6 发奖 AdjustCredit 全链路（补缺）

```
活动触发（入金/交易/领码）
  → 资格校验（eligibility_rule）
  → 幂等检查（reward_id = {campaign_id}:{user_uid}:{event_id}）
  → 预算扣减（原子 UPDATE t_campaign_budget，乐观锁）
  → 调资金域 FundService.AdjustCredit(reason=CAMPAIGN_BONUS, reward_id, amount)
  → 成功：发 campaign.reward.granted.v1 → 通知域
  → 失败：预算回滚 + 死信 + 告警
  → 日终对账：Σ reward vs Σ AdjustCredit vs ledger
```

- `reward_id` 幂等：资金域以此去重。
- 失败重试：N 次（指数退避）入死信，人工补发。
- 预算熔断：预算消耗 > 90% 告警，> 100% 自动拒绝新发奖。

### 9.7 预算熔断与对账（补缺）

- `t_campaign_budget`：`total_budget`、`used_budget`、`reserved_budget`、`status`。
- 熔断：`used + reserved > total × 0.95` → 告警；`> total` → 拒绝（`MKT_LIMIT_BUDGET_EXCEEDED`）。
- 日终对账：`Σ reward_amount` vs `Σ ledger(CAMPAIGN_BONUS)` vs `budget.used`，不一致告警。

### 9.8 Promo/Referral 反作弊（补缺）

| 规则 | 说明 |
|------|------|
| 一人一次 | 同 user_uid 一码一次 |
| 设备指纹 | 同设备多账号 → 标记 |
| IP 频控 | 同 IP 短时间多领 → 限流 |
| 关联账户 | 同 IB 归因的互邀 → 拒绝 |
| 机器人检测 | 行为模式分析 → 标记 |
| 人工审核 | 高额奖励触发人工 |

### 9.9 工单 SLA 与资金争议协同（补缺）

| 优先级 | SLA | 类型 |
|--------|-----|------|
| P0 | 4h | 资金争议、发奖失败 |
| P1 | 24h | 投诉、活动咨询 |
| P2 | 72h | 一般咨询 |

- 路由：按类型 + 辖区 + 语言分配。
- 与资金争议协同：工单触发资金冻结（`can_withdraw=0`），结案后解冻。
- 升级：超 SLA 自动升级 + 通知主管。

### 9.10 完整 DDL 补充（补缺）

```sql
CREATE TABLE t_recommend_slot (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  slot_code VARCHAR(64) NOT NULL,
  entity_id VARCHAR(32) NOT NULL,
  slot_config JSON NOT NULL COMMENT '槽位配置',
  status TINYINT UNSIGNED DEFAULT 1,
  created_at BIGINT UNSIGNED NOT NULL,
  UNIQUE KEY uk_slot (entity_id, slot_code)
) COMMENT='推荐槽位';

CREATE TABLE t_recommend_exposure (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  user_uid VARCHAR(64) NOT NULL,
  slot_code VARCHAR(64) NOT NULL,
  content_id BIGINT UNSIGNED NOT NULL,
  content_type VARCHAR(32) NOT NULL,
  experiment_id BIGINT UNSIGNED NULL,
  exposed_at BIGINT UNSIGNED NOT NULL,
  clicked_at BIGINT UNSIGNED NULL,
  INDEX idx_user_slot_day (user_uid, slot_code, exposed_at)
) COMMENT='推荐曝光埋点';

CREATE TABLE t_campaign_budget (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  campaign_id BIGINT UNSIGNED NOT NULL,
  total_budget DECIMAL(18,2) NOT NULL,
  used_budget DECIMAL(18,2) DEFAULT 0,
  reserved_budget DECIMAL(18,2) DEFAULT 0,
  currency VARCHAR(8) NOT NULL,
  version INT UNSIGNED DEFAULT 0 COMMENT '乐观锁',
  status TINYINT UNSIGNED DEFAULT 1,
  UNIQUE KEY uk_campaign (campaign_id)
) COMMENT='活动预算池';

CREATE TABLE t_campaign_eligibility_eval (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  campaign_id BIGINT UNSIGNED NOT NULL,
  user_uid VARCHAR(64) NOT NULL,
  trigger_event_id VARCHAR(128) NOT NULL,
  eligible TINYINT UNSIGNED NOT NULL,
  reject_reason VARCHAR(64) NULL,
  evaluated_at BIGINT UNSIGNED NOT NULL,
  UNIQUE KEY uk_eval (campaign_id, user_uid, trigger_event_id),
  INDEX idx_user (user_uid)
) COMMENT='活动资格评估记录';

CREATE TABLE t_campaign_budget_ledger (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  campaign_id BIGINT UNSIGNED NOT NULL,
  reward_id VARCHAR(128) NOT NULL,
  amount DECIMAL(18,2) NOT NULL,
  type VARCHAR(16) NOT NULL COMMENT 'RESERVE/GRANT/RELEASE/REVERSAL',
  created_at BIGINT UNSIGNED NOT NULL,
  UNIQUE KEY uk_reward (reward_id, type)
) COMMENT='预算流水';
```
